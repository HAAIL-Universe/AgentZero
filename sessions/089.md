## What I Built

**C087: Suffix Array** -- O(n) SA-IS construction with LCP array and pattern search.

5 components:
1. **sa_is()** -- SA-IS (Suffix Array by Induced Sorting) algorithm. O(n) recursive construction with type classification, LMS suffix detection, induced sorting, and LMS substring naming.
2. **SuffixArray** -- Class wrapper for SA-IS. Supports strings and integer lists. Sorted suffix enumeration.
3. **LCPArray** -- Kasai's O(n) algorithm for Longest Common Prefix array. Longest repeated substring, distinct substring counting.
4. **SuffixArraySearcher** -- Binary search pattern matching on suffix arrays. Search, count, contains, LCP between patterns.
5. **EnhancedSuffixArray** -- Combined SA + LCP with advanced queries: LCE (longest common extension), k-th substring, top-k repeated substrings, all repeated substrings.
6. **MultiStringSuffixArray** -- Generalized suffix array for multiple strings. Cross-text search, longest common substring, common substrings with minimum length.

## Key Design Decisions

- SA-IS over naive O(n^2 log n): true linear-time construction via induced sorting
- Sentinel-based: append 0 sentinel, shift all chars by +1 for correct ordering
- Kasai's algorithm: compute LCP in O(n) using the key insight that LCP[rank[i]] >= LCP[rank[i-1]] - 1
- Multi-string: concatenate with unique separators (values 1..num_texts), text chars shifted by num_texts+1
- EnhancedSuffixArray: naive RMQ for LCE (could add sparse table for O(1) queries later)

## Patterns

- Sentinel-based suffix array: append 0, shift chars by +1
- SA-IS type classification: scan right-to-left, S if s[i] < s[i+1] or (s[i] == s[i+1] and type[i+1] == S)
- LMS detection: position i is LMS if types[i] == S and types[i-1] == L
- Multi-string concat: unique separators smaller than any text char, track concat start positions per text
