## What Was Fixed

### 1. Strategic Routing for Short Intent Messages (reasoning_framework.py)

**Bug:** "I want a raise." routed to `casual_chat` instead of `strategic_reasoning`. The casual check caught all messages <10 words that lacked a matching strategic keyword ("ask for a raise" != "want a raise").

**Fix:**
- Added 11 new strategic keywords: "want a raise", "i want to", "i need to", "help me", "struggling with", "can't decide", "don't know how", "afraid of", "anxious about", "thinking about", "considering"
- Added STRONG_STRATEGIC keyword set -- these indicate clear strategic intent even in short messages and bypass the 2-signal requirement
- Strong keywords: "want a raise", "ask for a raise", "career move", "i'm scared", "i am scared", "afraid of", "struggling with", "can't decide", "anxious about"
- Routing now: `signal_count >= 2 or (has_strong and signal_count >= 1)`

### 2. A2 Finding: Greedy Regex in JSON Extraction (cognitive_agents.py:222)

**Bug:** `re.search(r"\{.*\}", text)` was greedy -- if model output contained multiple JSON objects or nested braces, it matched from first `{` to last `}`, producing invalid JSON.

**Fix:** Replaced regex with iterative `json.loads()` from each `{` position. This correctly handles nested braces and finds the first valid JSON object.

### 3. A2 Finding: Unsafe Dict Access (cognitive_agents.py:236)

**Bug:** `plans[1]["name"]` -- if the dict lacked a "name" key, KeyError.

**Fix:** Changed to `plans[1].get("name", "")`.

### 4. A2 Finding: Exception Detail Leakage (tool_runtime.py:107)

**Bug:** Raw exception details (including file paths, DB connection strings) returned to user in error messages.

**Fix:** Log full exception server-side, return generic "Tool execution failed. Please try again." to user.

### 5. A2 Finding: Silent Callback Failures (cognitive_runtime.py:260,275)

**Bug:** `except Exception: pass` silently swallowed `on_step` callback failures.

**Fix:** Added `print()` warnings with agent ID and exception details for debugging.
