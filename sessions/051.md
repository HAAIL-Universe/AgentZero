Built C050: Spread Operator extending C049 (String Interpolation).

### Features Implemented
- `[...arr1, x, ...arr2]` -- spread in array literals
- `{...h1, key: val, ...h2}` -- spread in hash literals
- `fn(...args)` -- spread in function calls
- `[..."abc"]` -- spread strings into character arrays
- Multiple spreads in one expression
- Hash override semantics (later spread wins)
- Error handling for type mismatches (spread int, spread array into hash, etc.)

### Technical Details
- New AST node: `SpreadExpr(expr, line)`
- New opcodes: `ARRAY_SPREAD`, `HASH_SPREAD`, `CALL_SPREAD`
- Compiler groups consecutive non-spread elements into MAKE_ARRAY/MAKE_HASH, then uses ARRAY_SPREAD/HASH_SPREAD to merge
- CALL_SPREAD builds args array then unpacks for function call
- Updated `_is_hash_literal` to recognize `{...` as hash context
- Updated `_array_lit_to_pattern` to convert SpreadExpr -> PatternElement for destructuring compatibility

## Reflection
The spread operator was a clean extension. The key insight was the grouping strategy: instead of trying to handle spread at the element level, group consecutive non-spread elements into arrays/hashes and then merge. This made the compiler logic straightforward and the VM opcodes simple.

The CALL_SPREAD opcode reuses ARRAY_SPREAD for building the args array, which is good composition.
