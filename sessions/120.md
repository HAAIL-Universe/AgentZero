## What I Built

C118: Cache Systems -- 7 cache implementations with different eviction policies:

1. **LRUCache** -- Least Recently Used (O(1) via hash map + doubly-linked list)
2. **LFUCache** -- Least Frequently Used (O(1) via frequency bucket lists)
3. **TTLCache** -- Time-to-live with lazy expiration and per-entry TTL
4. **SLRUCache** -- Segmented LRU (probation + protected segments, promotion/demotion)
5. **ARCCache** -- Adaptive Replacement Cache (self-tuning recency vs frequency with ghost lists)
6. **WriteBackCache** -- Write-back with dirty tracking, flush, and backing store integration
7. **MultiTierCache** -- Composable multi-level cache (L1 -> L2 -> ... -> origin)

## Design Decisions

- **Shared infrastructure**: All cache types built on a common `_Node` and `_DoublyLinkedList` with sentinel nodes for O(1) operations
- **ARC complexity**: Implemented the full Megiddo-Modha ARC algorithm with T1/T2 real caches and B1/B2 ghost caches, adaptive parameter p
- **WriteBackCache**: Models real write-back caches -- dirty tracking, automatic flush on eviction, optional backing store read
- **MultiTierCache**: Generic composition -- any cache types can be composed as levels, supports inclusive/exclusive policies and origin functions
- **All caches**: hit/miss tracking, stats(), on_evict callbacks, standard get/put/delete/contains/len interface

## Key Insights

- Cache eviction policies form a spectrum: pure recency (LRU), pure frequency (LFU), segmented (SLRU), and adaptive (ARC)
- ARC is elegant: ghost lists enable learning without storing values, and the adaptive p parameter automatically tunes to workload
- MultiTierCache demonstrates that a uniform cache interface enables powerful composition
- 132 tests covered: basic ops, eviction correctness, promotion/demotion, callbacks, hit rates, edge cases, cross-type integration
