## What I Built

Persistent B-tree for ordered key-value storage with path-copying structural sharing.

### Core Features
- Configurable branching factor (order >= 3, default 32)
- O(log n) insert, delete, search via binary search within nodes
- Path-copying persistence -- every mutation returns new tree, original unchanged
- Split on overflow, borrow/merge on underflow (classic B-tree invariants)

### Query Operations
- `get`, `__contains__`, `__getitem__` -- standard dict-like access
- `min()`, `max()` -- O(log n) extremes
- `floor(key)`, `ceiling(key)` -- nearest key <= or >= given key
- `nearest(key)` -- absolute nearest key
- `range(lo, hi)` -- efficient range queries with inclusive/exclusive bounds
- `rank(key)` -- count of keys < key
- `select(k)` -- k-th smallest element (0-indexed)
- `kth_largest(k)` -- k-th from the end
- `count_range(lo, hi)` -- count keys in range

### Functional Operations
- `map_values(fn)` -- transform values preserving structure
- `filter(pred)` -- subset by predicate
- `reduce(fn, initial)` -- fold over items

### Bulk & Comparison
- `from_sorted`, `from_items`, `from_dict` -- construction helpers
- `merge(other, conflict)` -- merge two trees with conflict resolution
- `diff(other)` -- compute added/removed/changed between trees
- `slice(lo, hi)` -- extract subtree as new B-tree
- `pop_min()`, `pop_max()` -- priority queue style extraction

### Iteration
- `__iter__`, `__reversed__` -- forward/reverse key iteration
- `items()`, `keys()`, `values()` -- dict-like iterators
- `to_dict()` -- convert to Python dict
- `__eq__` -- structural equality by content

## Design Decisions
- Tuples for keys/values/children (immutable, hashable, cache-friendly)
- `__slots__` on both BTreeNode and BTree for memory efficiency
- Binary search within nodes (`_search_index`) for O(log b) per-node lookup
- Predecessor replacement for internal node deletion (swap with rightmost in left subtree)
- Rebalance cascade: borrow-left -> borrow-right -> merge (in priority order)

## Reflection
B-trees are the workhorse data structure of databases and filesystems. The persistent variant adds path-copying overhead but enables time-travel (keep old versions). The implementation is clean -- the split/borrow/merge logic is the most intricate part, but methodical case handling keeps it manageable. The ordered-map operations (range, floor, ceiling, rank, select) make this significantly more useful than C076's unordered HAMT.
