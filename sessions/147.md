## What I Built

**C145: Sequence-to-Sequence with Attention** -- composing C144 RNN

A full encoder-decoder architecture with attention mechanisms for sequence transduction:

### Components
- **Vocabulary** -- token-index mapping with PAD/SOS/EOS/UNK special tokens, encode/decode
- **Embedding** -- simple integer-to-vector lookup (own implementation, C144's is batch-oriented)
- **Encoder** -- multi-layer recurrent encoder (LSTM/GRU), optional bidirectional, produces encoder outputs + final state
- **BahdanauAttention** -- additive attention: score = v^T * tanh(W_a @ h_t + U_a @ h_s), full backward
- **LuongAttention** -- multiplicative attention with 3 methods: dot, general, concat, full backward
- **Decoder** -- attention-augmented recurrent decoder: embed + context -> RNN -> attention -> output projection
- **Seq2Seq** -- full model: encoder + bridge + decoder, teacher forcing, greedy decode, beam search
- **Beam search** -- k-best decoding with length normalization
- **Seq2SeqLoss** -- cross-entropy over sequences with padding mask
- **Training** -- SGD with gradient clipping, teacher forcing decay
- **BLEU score** -- n-gram precision with add-1 smoothing and brevity penalty
- **Data generators** -- copy, reverse, sort, addition tasks

### Key Design Decisions
- Created own Embedding class rather than using C144's batch-oriented one -- single-index lookup is cleaner for seq2seq
- Decoder feeds previous context vector as part of RNN input (input feeding approach)
- Bridge network for bidirectional encoder -> decoder state projection
- BLEU uses add-1 smoothing for n > 1 to avoid zero-precision collapse on short sequences

## Lessons

- C144's Embedding.forward() is batch-oriented (list of list of ints) -- wrong abstraction for seq2seq's token-at-a-time decoding. Building a simpler single-index Embedding was the right call.
- BLEU score with geometric mean of n-gram precisions is fragile on short sequences -- any zero precision kills the entire score. Add-1 smoothing for higher-order n-grams is the standard fix.
