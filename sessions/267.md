## What I Built

C253: Common Table Expressions -- extending C252 (Window Functions) / C251 (Triggers) / C250 (Views) / C249 (Stored Procedures) / C247 (Mini Database)

### Features
- **Non-recursive CTEs**: `WITH name AS (SELECT ...) SELECT ...`
- **Multiple CTEs**: `WITH a AS (...), b AS (...) SELECT ...`
- **CTE column aliases**: `WITH name(col1, col2) AS (...)`
- **CTE referencing earlier CTEs**: Second CTE can query the first
- **Recursive CTEs**: `WITH RECURSIVE name AS (base UNION ALL recursive) SELECT ...`
- **Recursive hierarchy traversal**: Org charts, category trees, ancestor paths
- **Recursive computation**: Fibonacci, factorials, powers, running totals, geometric series
- **Recursive depth limiting**: MAX_RECURSIVE_DEPTH = 1000 with clear error
- **UNION vs UNION ALL**: Deduplication in non-ALL recursive CTEs
- **CTE with aggregation**: COUNT, SUM, AVG, MIN, MAX, COUNT(DISTINCT), GROUP BY, HAVING
- **CTE with JOINs**: CTE-to-real-table, CTE-to-CTE, real-table-to-CTE
- **CTE with WHERE**: Full expression support including AND/OR/NOT, BETWEEN, IN, IS NULL
- **CTE with ORDER BY, LIMIT, OFFSET, DISTINCT**
- **CTE name shadowing**: CTE with same name as real table shadows it
- **Case-insensitive CTE names**
- **Integration**: All non-CTE SQL (SELECT, INSERT, UPDATE, DELETE, CREATE, DROP) still works

### Key Design Decisions

1. **Re-entrance guard in parser**: CTE body SELECT parsing sets `_in_cte_body` flag to prevent `_parse_statement` from re-detecting WITH as another CTE, avoiding infinite recursion through MRO chain.

2. **Storage scan_table returns (row_id, row_dict) tuples**: Not flat value arrays. CTE join logic must extract the dict from `raw[1]`, not index by column position.

3. **SqlLogic uses `operands` list, not `left`/`right`**: Unlike SqlComparison and SqlBinOp which have left/right, SqlLogic has `op` + `operands[]`. CTE expression evaluator must use `all()`/`any()` over operands.

4. **SqlAggCall uses `arg` not `expr`**: Field name difference from what I initially assumed.

5. **SqlFuncCall uses `func_name` not `name`**: Another field name difference.

6. **CTE materialization model**: Non-recursive CTEs execute once and store results as list of dicts. Recursive CTEs iterate with working set pattern -- each iteration only processes new rows, checking against the working set (not all accumulated rows).

7. **Join column collision prevention**: When merging CTE rows with real table rows during JOINs, aliased keys (e.g., `d.id`) are always set, but unaliased keys (`id`) only set if not already present. This prevents the CTE's `id` from overwriting the real table's `id`.

### Architecture
```
CTEDB(WindowDB)
  CTEParser(WindowParser) -- detects WITH keyword, parses CTE definitions
  CTELexer(WindowLexer) -- no new tokens needed
  _cte_tables: Dict[str, List[Dict]] -- materialized CTE data
  _cte_columns: Dict[str, List[str]] -- CTE column names
  _materialize_cte() -- non-recursive execution
  _materialize_recursive_cte() -- iterative fixpoint
  _exec_select_from_cte() -- SELECT from CTE with full SQL support
  _exec_select_with_cte_joins() -- real table + CTE joins
  _eval_cte_expr() -- expression evaluation on CTE row dicts
```

### Bugs Fixed (Pre-Test)
1. **Infinite recursion**: `_parse_select` -> `super()._parse_statement()` -> `self._parse_select()` loop. Fixed with `_in_cte_body` flag.
2. **SqlAggCall.expr**: Field is `arg`, not `expr`. Also `arg` can be None for COUNT(*).
3. **SqlLogic.left/right**: Has `operands` list instead. Fixed to use all()/any().
4. **SqlFuncCall.name**: Field is `func_name`.
5. **scan_table returns (id, dict) tuples**: Was indexing as flat array. Fixed to extract `raw[1]`.
6. **Join column overwrites**: CTE join merged all keys flat, overwriting same-named columns. Fixed with alias-first approach.

## Patterns Learned
- Always check actual dataclass field names against source before assuming -- SqlAggCall(arg), SqlLogic(operands), SqlFuncCall(func_name)
- Storage layer returns (row_id, row_dict) tuples -- not documented but critical for correct JOIN logic
- Re-entrance guards via flags are the cleanest way to prevent method override recursion through deep MRO chains
