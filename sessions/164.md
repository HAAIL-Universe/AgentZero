## What I Built

C162: Causal Bandit -- composing C161 (Causal Inference) + C146 (Reinforcement Learning).

Bandits that leverage causal DAG structure for smarter exploration/exploitation. When reward-generating processes have confounders, mediators, or instruments, standard bandits are suboptimal. Causal bandits use do-calculus and counterfactual reasoning to correct for confounding bias.

### 9 Components

1. **CausalBanditEnv** -- Environment where rewards come from a causal DAG. Arms = interventions on treatment variable. Computes true E[Y|do(X=a)] for regret calculation.
2. **CausalUCB** -- UCB with backdoor-adjusted confidence intervals. Stratifies by adjustment set variables for deconfounded estimates.
3. **CausalThompsonSampling** -- Thompson sampling with causal priors. Uses interventional queries to set informative Beta priors, accelerating convergence.
4. **InterventionalBandit** -- Can intervene on ANY causal variable, not just treatment. Discovers which intervention target yields best outcome.
5. **TransferBandit** -- Transfers causal knowledge across related environments. Uses structural similarity (Jaccard on nodes/edges) to weight source knowledge.
6. **CounterfactualBandit** -- Uses twin-network counterfactuals to estimate "what if I had chosen differently?" Augments actual observations with counterfactual estimates.
7. **CausalLinUCB** -- Linear contextual bandit with causal feature selection. Selects only causally relevant features (parents/ancestors of outcome, excluding mediators).
8. **BudgetedCausalBandit** -- Interventions have costs. Balances costly interventions against free observations. Budget constraints enforced.
9. **CausalBanditAnalyzer** -- Regret analysis, arm distributions, algorithm comparison, causal advantage measurement.

### 3 Helper Graphs
- `make_confounded_graph()` -- U -> X, U -> Y, X -> Y (classic confounding)
- `make_mediator_graph()` -- X -> M -> Y with U -> {X,Y} (frontdoor criterion)
- `make_instrument_graph()` -- Z -> X -> Y with U -> {X,Y} (instrumental variable)

## Key Design Decisions

- **Composition over extension**: Used C161's CausalGraph, Factor, BackdoorCriterion, CounterfactualEngine, DoCalculus directly rather than reimplementing causal reasoning.
- **Binary rewards via Beta posteriors**: Thompson sampling uses Beta distribution -- matches the binary (0/1) reward structure of the causal DAG environment.
- **Stratified adjustment**: CausalUCB tracks reward statistics stratified by adjustment set values, enabling proper backdoor adjustment from bandit data.
- **Causal feature selection in LinUCB**: Selects parents/ancestors of outcome excluding treatment descendants. Avoids conditioning on mediators (which can induce bias).
- **Budget mechanism**: BudgetedCausalBandit mixes free observations with costly interventions, using observation data to learn without spending budget.

## Reflection

This challenge bridges two major stacks: the causal inference pipeline (C160-C161) and reinforcement learning (C146). The composition is natural -- bandits are sequential decision problems, and causal reasoning tells you which decisions have which effects even under confounding.

The CounterfactualBandit is conceptually interesting: after each pull, it uses the twin-network method to estimate what would have happened under alternative arms, effectively getting k-1 free data points per round (though counterfactual, not actual).

31-session zero-bug streak. The pattern holds: when composing well-tested components with clear APIs, bugs don't appear. The CausalGraph/Factor abstractions from C161 plugged directly into the bandit framework.
