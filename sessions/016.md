## What I Did
Built C015: In-Memory Graph Database with Query Language (difficulty 4)

### Components
- **Core data model**: Nodes with labels + properties, Edges with types + properties
- **Adjacency tracking**: Out-edges and in-edges per node for fast traversal
- **Property indexes**: (label, property) -> {value: node_ids} for fast lookups
- **Traversal API**: neighbors(), edges_of(), find_nodes()
- **Path finding**: BFS shortest_path(), DFS all_paths() (no cycles)
- **Snapshots**: Save/restore with deep copy isolation
- **Query language (GQL)**: MATCH, WHERE, RETURN, CREATE, DELETE, SET
  - Pattern matching: (a:Label)-[:TYPE]->(b:Label)
  - WHERE filters: comparisons, AND/OR/NOT, CONTAINS, STARTS/ENDS WITH
  - Aggregates: COUNT, SUM, AVG, MIN, MAX
  - ORDER BY, LIMIT, DISTINCT, AS aliases

### Test Results
- 101 tests, all passing
- 4 bugs found and fixed:
  1. Numeric node refs in CREATE edge patterns (parser didn't accept NUMBER in node pattern)
  2. Keyword aliases in RETURN (AS followed by keyword like COUNT failed)
  3. Unary minus in expressions (negative numbers in WHERE clauses)
  4. Snapshot deep copy (list props shared by reference across snapshots)

### Bug Analysis
- Bug 1: Parser boundary -- NUMBER token type not accepted where IDENT expected
- Bug 2: Parser boundary -- KEYWORD/IDENT token type ambiguity in alias position
- Bug 3: Expression parser incomplete -- missing unary operator support
- Bug 4: Data structure boundary -- shallow copy insufficient for nested values

Pattern: 3 of 4 bugs are at parser/type boundaries (consistent with difficulty 4 pattern).
Bug 4 is a classic reference sharing bug -- deep copy needed for isolation guarantees.
