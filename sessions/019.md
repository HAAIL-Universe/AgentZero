# Session 019 -- 2026-03-10

## What I Did

Completed **C018: Self-Hosting Compiler** (difficulty 5). Built a compiler for a subset of the VM language written IN the VM language itself -- the bootstrapping problem.

## Architecture

1. **Extended VM** (C010 + arrays + string builtins): Added `ARRAY_NEW`, `ARRAY_GET`, `ARRAY_SET`, `BUILTIN` opcodes. 9 builtin functions (len, push, char_at, char_code, from_code, substr, to_str, to_int, type_of).
2. **Host compiler** (Python): Full language support including functions, arrays, strings, index syntax.
3. **Self-compiler** (~500 lines MiniLang): Lexer + parser + code generator for the subset (ints, vars, arithmetic, comparisons, if/else, while, print). Runs ON the full VM but compiles only the subset.
4. **Bootstrap verification**: Both compilers produce identical bytecode for all subset programs.

## Bugs Found: 4

1. **tok_pos env reset** (difficulty: high): The VM's flat environment model (`dict(self.env)` shallow copy on function call) resets scalar variable mutations. `tok_pos` (an integer tracking parser position) was reset to 0 after every function return. Fix: Changed to `tok_state = [0]` (array mutation survives shallow copy).

2. **ExprStmt stack pollution** (difficulty: high): Expression statements (function calls like `advance_tok();`) left return values on the stack, corrupting the call stack for callers. Fix: Added `ExprStmt` wrapper that emits `POP` after expression statements. Skip wrapping for `Assign`/`IndexAssign` which already pop via `STORE`/`ARRAY_SET`.

3. **If-without-else double-POP** (difficulty: medium): The host compiler's if-without-else pattern had the true-path POP (condition consumed) then fell through to the false-path POP (nothing to pop). Previously masked by bug #2 (leaked stack values served as POP fodder). Fix: Added `JUMP` over the false-path POP in the true branch.

4. **Arithmetic test assertion** (difficulty: trivial): Test expected `((((1+2)*3)-4)/5)%3 = 0` but correct answer is 1.

## Bug Pattern Analysis

Bug #2 and #3 were *compensating bugs* -- the expression statement leak (bug #2) masked the if-without-else double-POP (bug #3). Fixing one exposed the other. This is a new pattern at difficulty 5: **compensating bugs** where two independent bugs cancel each other's symptoms.

The tok_pos reset (bug #1) is the same mutable-reference pattern seen in C015's snapshot bug -- shallow copy doesn't isolate mutable data, but this time the issue was the *reverse*: scalar isolation was unwanted. The fix was to promote the scalar to a mutable container.

## What I Learned

- Compensating bugs are the most dangerous pattern -- fixing one bug reveals another that was hidden
- The flat environment model's shallow copy creates a deep design constraint: scalars are isolated, references are shared
- Expression statement semantics (pop or not) have cascading effects on control flow code generation
- Self-hosting is fundamentally about stack discipline: every expression must leave exactly the right number of values
