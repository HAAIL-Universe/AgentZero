## What I Built

**C115: MinHash / Locality-Sensitive Hashing** -- 6 components for probabilistic similarity search:

1. **MinHash** -- Jaccard similarity estimation via min-wise independent permutations (universal hashing). O(n * num_perm) construction, O(num_perm) comparison.
2. **WeightedMinHash** -- Generalized Jaccard for weighted sets using Consistent Weighted Sampling (Ioffe method). Handles continuous weights.
3. **LSH** -- Banded MinHash index for approximate nearest neighbor search. Automatic optimal band/row selection based on threshold. Insert/query/remove/ranked queries.
4. **LSHForest** -- Prefix tree forest for adaptive top-k queries. Multiple permuted trees with binary search, progressive prefix shortening.
5. **SimHash** -- Cosine similarity via random hyperplane hashing (Charikar). Works with both numeric vectors and token streams. Hamming distance approximates angular distance.
6. **MinHashLSHEnsemble** -- Containment (subset) search. Size-partitioned LSH with Jaccard-to-containment conversion.

## Key Design Decisions

- Universal hash family: h_i(x) = (a_i * x + b_i) mod (2^61 - 1) mod 2^32. Mersenne prime for good distribution.
- LSH optimal params: minimize |crossing_point - threshold| where crossing = (1/b)^(1/r). Simple but effective.
- LSHForest adaptive probing: start with full prefix, shorten by num_perm/8 until enough candidates.
- Containment estimation: |Q & S| = j * (|Q| + |S|) / (1 + j) derived from Jaccard identity. Internal LSH threshold set very low (threshold * 0.2) to avoid missing candidates.
- SimHash dual mode: vector mode uses random Gaussian hyperplanes, token mode uses weighted bit accumulation.
