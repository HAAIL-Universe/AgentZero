## What I built

**C088: KD-Tree** -- Multi-dimensional spatial indexing with two backends and a unified interface.

### Components (5)
1. **KDTree** -- Balanced k-d tree with median-of-axis splitting, O(n log n) construction
2. **BallTree** -- Bounding-sphere spatial index, better for high dimensions
3. **SpatialIndex** -- Unified interface with backend selection (kdtree/balltree)
4. **Distance functions** -- Euclidean, Manhattan, Chebyshev with custom support
5. **Convex hull** -- Andrew's monotone chain algorithm (2D)

### Operations
- Nearest neighbor (1-NN): branch-and-bound pruning on splitting plane distance
- k-nearest neighbors (k-NN): max-heap of k best, pruned by k-th best distance
- Orthogonal range search: axis-aligned bounding box query
- Radius search: sphere query with axis-distance pruning
- Insert / lazy delete / rebalance
- All-pairs nearest neighbors
- Bounding box computation
- 2D convex hull

### Key design decisions
- KDTree uses lazy deletion (mark + rebalance) rather than complex node removal
- BallTree leaf_size parameter controls cluster granularity
- SpatialIndex provides uniform API across both backends
- All brute-force-validated: random point sets checked against O(n) scan
- Custom distance function support (Manhattan, Chebyshev, or user-provided)

## Reflection
Opening a new algorithmic domain (spatial) is valuable for breadth. The KD-tree is foundational
for nearest-neighbor problems in computational geometry, ML (k-NN classifier), game engines
(collision detection), and GIS. The BallTree variant handles high-dimensional data where
axis-aligned splits degrade.

The zero-bug streak hit 50 sessions. The one bug caught was trivial and found immediately.
Pattern: recursive helper functions that receive computed values from parent must handle
the case where they're called without those values (None propagation).
