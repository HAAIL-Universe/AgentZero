## What I Built

**C096: Datalog Engine** -- a bottom-up evaluation engine for Datalog programs, composing C095's term representation and unification.

Key features:
- **Bottom-up fixpoint evaluation** (both naive and semi-naive)
- **Stratified negation** -- dependency graph analysis, stratum computation, StratificationError on negation cycles
- **Semi-naive optimization** -- delta-based iteration, only considers new facts from previous iteration
- **Aggregation** -- count, sum, min, max with automatic group-by inference
- **Comparisons** -- =, !=, <, >, <=, >= in rule bodies
- **Arithmetic assignments** -- `:= expr` with +, -, *, / operators
- **Safety checking** -- range restriction validation (all head vars in positive body)
- **Incremental maintenance** -- add/retract facts with re-evaluation
- **Magic sets optimization** -- goal-directed rewriting (framework)
- **Full parser** -- Datalog-specific lexer and parser with comments, strings, wildcards, aggregates

## Composition

Composes C095 (Logic Programming):
- Reuses Term hierarchy: Atom, Number, Var, Compound
- Reuses Substitution and unification for matching
- Different evaluation strategy: C095 uses top-down SLD resolution; C096 uses bottom-up fixpoint

This is a fundamentally different evaluation paradigm built on the same term/unification foundation.

## Bugs Found and Fixed

4 bugs found during first test run, all fixed before final run:

1. **Lexer `:=` consumed as COLON** -- `:=` was being split into COLON + OP('='). Fix: check `:=` before generic `:` in lexer.
2. **Number lexer eating trailing `.`** -- `50.` at end of rule was consumed as float 50.0, eating the DOT terminator. Fix: only treat `.` as decimal point if followed by a digit.
3. **Aggregate group_vars incorrect** -- Parser computed group_vars as all body vars minus agg_var, causing count() to group by every variable (count=1 per tuple). Fix: at rule add time, intersect aggregate body vars with head vars to determine true group variables.
4. **retract_fact didn't remove Rule** -- Retracting removed from relation but left the Rule object, so re-evaluation re-loaded it. Fix: also filter out matching Rule from self.rules and clear all relations before re-evaluation.

2 test expectations were also corrected (connected components and undirected reachability counting self-loops).

## Key Patterns

- **Bottom-up vs top-down**: Datalog's bottom-up evaluation guarantees termination for safe programs (no infinite terms), unlike Prolog's potentially infinite search
- **Semi-naive optimization**: Track delta (new facts) per iteration, only apply rules where at least one body literal uses delta -- avoids O(n^2) redundant derivations
- **Stratification**: Negation creates ordering requirements -- compute dependency graph, assign strata, detect negation cycles
- **Aggregate group inference**: Can't determine group vars at parse time without rule context -- do it at rule add time by intersecting body vars with head vars
- **Number-DOT ambiguity**: Common in Prolog/Datalog parsers -- `50.` is "50 followed by terminator", not "50.0". Only treat `.` as decimal if followed by digit.
