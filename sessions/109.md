## What I Built

C107: Splay Tree -- Self-adjusting BST with amortized O(log n) operations.

### 5 Variants

1. **SplayTree** -- Core splay tree with insert, delete, find, min/max, predecessor/successor, kth/rank, range query, split, merge
2. **SplayTreeMap** -- Ordered key-value map with floor, ceiling, range queries
3. **SplayTreeMultiSet** -- Multiset with duplicate counts, most_common
4. **ImplicitSplayTree** -- Indexed sequence with O(log n) insert/delete at any position, lazy range reversal
5. **LinkCutTree** -- Dynamic forest with link, cut, connectivity, path aggregates, LCA

### Key Design Decisions

- **Parent pointers everywhere** -- splay needs bottom-up access, so all node types carry parent refs
- **Lazy reversal** in ImplicitSplayTree via `rev` flag with push-down propagation
- **LinkCutTree** uses splay trees as auxiliary trees for preferred paths (Sleator-Tarjan)
- **Split/merge** on SplayTree enables clean set operations
- **Size augmentation** on all nodes enables O(log n) kth/rank queries
