## What I Built

### 1. Dynamic Context Budgeting (cognitive_agents.py)

Replaced the flat `max_chars=6000` context trimming with per-agent budget profiles.

**Key changes:**
- `AGENT_CONTEXT_BUDGETS`: 7 agents get calibrated budgets (2000-10000 chars) and per-agent response token limits (300-700 tokens)
- `AGENT_FIELD_PRIORITIES`: Per-agent must_have/important/optional field tiers
- `_trim_agent_context` now uses 5-phase trimming:
  1. Structural trimming (shadow_profile lists, episodes, rules capped at 5)
  2. Check budget -- return if fits
  3. Drop optional fields
  4. Truncate important list fields to 2
  5. Drop important fields, then generic fallback for unknown agents
- `_execute_model_backed_agent` now passes `agent_id` and uses per-agent `max_tokens`

**Impact:**
- Shadow retains 5 episodes/rules instead of 3 (67% more context for pattern detection)
- Clarifier budget reduced from 6000 to 2000 (saves ~4000 chars per call)
- Pineal gets 700 response tokens (was 500) for richer synthesis
- Budget utilization logged for future calibration

### 2. Consolidation Rule Quality Gate (consolidator.py)

Added three-stage quality validation between rule creation and injection.

**Key changes:**
- `compute_rule_quality`: Multi-dimensional scoring (structural, outcome, data sufficiency, drift)
- `_generate_quality_warnings`: Human-readable warnings for quality issues
- `get_relevant_rules` now has `apply_quality_gate=True` parameter -- filters rules by quality score >= 0.3
- Quality scores multiplied into rule ranking (topic_overlap * confidence * quality)
- Quality metadata stored on rules during `run_consolidation`
- Quality gate logging via `agent_zero.quality_gate` logger

**Quality dimensions:**
- Structural (25%): Are agent calibrations internally consistent?
- Outcome (35%): Does the rule have good outcome data?
- Data sufficiency (25%): Enough episodes? (8+ for full score)
- Confidence (15%): Statistical confidence
- Drift penalty (-0.2): Applied when outcome_decay detected

**Impact:**
- Rules with insufficient data (< 5 episodes) get warnings
- Rules with contradictory calibration or high ignore rates get blocked
- Rules showing outcome decay get penalized automatically
- Backward compatible: existing callers get quality gating automatically
