## What I Built

**C092: Graph Algorithms** -- a comprehensive graph algorithms library covering:

- **Graph data structure**: Weighted directed/undirected, neighbors, edges, degree, reverse, subgraph
- **Shortest paths**: Dijkstra (with early stop), A* (with heuristic), Bellman-Ford (negative weights, cycle detection), Floyd-Warshall (all-pairs)
- **MST**: Kruskal (union-find with path compression + rank), Prim (priority queue)
- **Network flow**: Edmonds-Karp (BFS-based Ford-Fulkerson), min-cut (max-flow min-cut theorem)
- **Graph structure**: Topological sort (Kahn's), Tarjan SCC, cycle detection/finding, bipartite check, connected components
- **Traversals**: BFS (with distances), DFS (sorted neighbors)

## Key Design Decisions

- Graph uses adjacency list (defaultdict of lists) for sparse graph efficiency
- UnionFind as standalone class for reuse (Kruskal, future use)
- All algorithms are standalone functions taking Graph, not methods -- composable
- Edmonds-Karp uses path-based BFS (not parent tracking) for clarity
- Floyd-Warshall returns both distance and next matrices for path reconstruction

## Patterns

- Heap-based algorithms (Dijkstra, Prim, A*) all use lazy deletion (check visited after pop)
- Union-Find with path compression + union by rank gives near-O(1) amortized operations
- Residual graph for max flow: forward edges decrease capacity, backward edges increase
- Tarjan SCC uses lowlink to find root of each component on the stack
