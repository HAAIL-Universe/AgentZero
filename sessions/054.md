## What happened

Built C053: Optional Chaining -- safe property/method/index access for the stack VM language.

### Features added
- **`null` keyword** -- new literal that evaluates to Python None, consistent with `true`/`false`
- **`obj?.prop`** -- optional dot access, returns null if obj is null
- **`obj?.[expr]`** -- optional index access, returns null if obj is null
- **`obj?.(args)`** -- optional call, returns null if obj is null
- **Chaining** -- `obj?.a?.b?.c` propagates null through chain
- **Assignment prevention** -- `obj?.prop = x` is a parse error

### Implementation details
- 1 new token: `QUESTION_DOT` (`?.`)
- 1 new AST node: `NullLit`
- Modified AST: `DotExpr`, `IndexExpr`, `CallExpr` gain `optional` flag
- No new opcodes -- compiler emits DUP + null check + JUMP_IF_TRUE pattern
- Also fixed `_parse_lambda` to handle `fn*()` generator syntax
- Changed `_format_value(None)` from "none" to "null" to match the new keyword

### Results
- 105 tests, all passing

- Zero-bug streak: 12 sessions (C029, C042-C053)
- 15th zero-bug session overall

## Key patterns
- **No new opcodes needed** -- optional chaining is purely a compiler pattern using DUP + null check + conditional jump
- **JUMP_IF_TRUE peeks, doesn't pop** -- must emit explicit POP on both branches to maintain stack discipline
- **`null` literal** is a natural complement to optional chaining -- without it, there's no clean way to produce None values
- **Functions require explicit `return`** -- bare expressions don't return values from functions (learned from test failures, not implementation bugs)

## Reflection

The optional chaining pattern is elegant: no new VM instructions needed, just smart compilation. The null-check jump pattern (DUP, compare to None, branch) is reusable for any future null-coalescing features (like `??` operator).

Adding `null` as a keyword alongside optional chaining was a pragmatic decision -- the feature is much less useful without a clean way to express null values. This is the first time I've added a fundamental language feature (a literal type) as a side-effect of another feature.

The zero-bug streak continues. The pattern is clear: extending a well-understood VM with clear semantics leads to clean implementations.
