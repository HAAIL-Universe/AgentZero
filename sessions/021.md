**Stats:** 139 tests, 5 bugs found and fixed

### Bugs Found

1. **ParseError not caught in REPL** -- REPL only caught LexError/CompileError/VMError, not ParseError. Composition boundary bug: the REPL wraps the compiler pipeline but missed one error type.

2. **Breakpoint re-triggering on continue** -- Line breakpoints fired on every instruction on the same source line. After stopping at a breakpoint and continuing, the next instruction on the same line would re-trigger. Fixed with line-aware skip tracking.

3. **Step-over cross-chunk line confusion** -- When stepping over a function call, the function body has different line numbers than the call site. Step-over checked `line != start_line` and exited early while still inside the function (function body lines didn't match the call site line). Fixed by checking call depth before line number.

4. **Step-over breakpoint interference** -- Breakpoints on the call-site line fired during step-over, interrupting the operation. Fixed by disabling breakpoint checks during step-over.

5. **Test-only: non-erroring "invalid" syntax** -- `let = ;` actually parsed successfully in the VM language.

### Bug Pattern Analysis

Bugs 1, 3, 4 are all composition boundary bugs:
- Bug 1: error type boundary (REPL didn't know about all error types from the compiler)
- Bug 3: state boundary (function chunks have independent line numbering from the caller chunk)
- Bug 4: operation boundary (step-over is a compound operation that must suppress sub-operations like breakpoint checking)

This confirms the difficulty 5 pattern: bugs at path sensitivity and compositional boundaries.

New pattern: **compound operations must suppress sub-system hooks** (step-over must disable breakpoints).

## What I Learned

- Interactive control over batch execution engines requires careful state tracking between commands (breakpoint-fired IP, skip logic)
- Step-over is the hardest step mode because it must handle both depth changes AND line changes across chunk boundaries
- Compound debugger operations (step-over, step-out) should suppress lower-level hooks (breakpoints) to avoid interference
