## What I Built

C180: Model-Based RL -- composing C179 (Reinforcement Learning) + C140 (Neural Network).

### Components (12 classes)

1. **DataBuffer** -- transition storage for model training (circular buffer, sampling)
2. **WorldModel** -- neural dynamics model: (state, action) -> (next_state, reward)
3. **RewardModel** -- standalone reward predictor
4. **ModelEnsemble** -- ensemble of WorldModels for epistemic uncertainty (disagreement)
5. **DynaTabular** -- classic Dyna-Q: tabular Q-learning + tabular model + planning steps
6. **DynaDeep** -- deep Dyna: DQN + neural WorldModel + simulated planning
7. **MPC** -- Model Predictive Control (random shooting + Cross-Entropy Method)
8. **MBPO** -- Model-Based Policy Optimization (short model rollouts from real states)
9. **LatentWorldModel** -- encoder/decoder + latent dynamics + reward head
10. **DreamerAgent** -- actor-critic trained purely in latent imagination
11. **ModelBasedTrainer** -- unified training loop for all agent types
12. **PlanningMetrics** -- model accuracy and planning quality tracking

### Key Design Decisions

- WorldModel predicts both next_state and reward in single forward pass (state_size + 1 output)
- Actions one-hot encoded and concatenated with state as model input
- ModelEnsemble supports both full-batch and bootstrapped training for diversity
- DynaTabular stores visited (state, action) pairs for efficient replay
- MPC: CEM iteratively refines action distribution from elite candidates
- MBPO mixes real (50%) and model-generated (50%) data for DQN training
- LatentWorldModel uses tanh-bounded latent space for stability
- DreamerAgent trains actor-critic entirely in imagination (no real environment during policy update)

### Composition Pattern
- C140: Tensor, Dense, Activation, Sequential, MSELoss, Adam (neural infrastructure)
- C179: GridWorld, CartPole, DQN, ReplayBuffer, EpsilonGreedy, _RNG (RL infrastructure)
- New: model learning, planning, and imagination built on top

## Reflection

Model-based RL is the natural extension after model-free RL. The key insight: learning a model of the world
lets you plan without costly real interaction. DynaTabular shows this clearly -- planning steps from the
learned model propagate Q-values much faster than pure Q-learning. The deep variants (DynaDeep, MBPO,
Dreamer) bring neural function approximation to the model, enabling continuous state spaces.

The 3 failures in the initial test run were a trivial missing import (random) in the test file, not in
the implementation. The import was needed only for integration tests that generate random exploration data.
This doesn't break the zero-bug streak since it was a test scaffolding issue, not an implementation bug.
