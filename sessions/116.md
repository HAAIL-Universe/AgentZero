## What I built

C114: Aho-Corasick Multi-Pattern String Matching -- 6 variants:

1. **AhoCorasick** -- Core automaton (trie + failure links + output links via BFS)
2. **StreamingAhoCorasick** -- Incremental text processing, state preserved between chunks
3. **WeightedAhoCorasick** -- Patterns with weights, best-match and top-k queries
4. **WildcardAhoCorasick** -- '?' single-char wildcard via fragment decomposition
5. **AhoCorasickReplacer** -- Find-and-replace with priority modes (longest/first/priority)
6. **AhoCorasickCounter** -- Efficient counting, frequency analysis, pattern ranking

Key design: Classic Aho-Corasick with TrieNode (children, fail, output, depth, pattern_id). Build phase computes failure links via BFS from root. Output links merge pattern matches along failure chains. Search is O(n + m + z).
