## Bugs Found

Implementation bugs: **2** (both API boundary issues, not logic bugs)
1. IDE has `debug_add_watch` but no `debug_remove_watch` -- had to call `debugger.remove_watch` directly
   - Pattern: **missing symmetry at composition boundary** -- add method exists but remove doesn't
2. CORS middleware order: routes registered before middleware, so middleware never ran
   - Pattern: **initialization order dependency** -- middleware must be registered before routes

Test bugs (wrong assumptions, not impl bugs): 3
1. `list_files` returns full paths with root prefix -- needed `_strip_root`
2. `:path` param can't handle paths with `/` -- switched to wildcard routes
3. Symbols endpoint returns empty for some files -- relaxed assertion

## Patterns

- **Missing symmetry at composition boundary**: When system A exposes `add_X`, always verify `remove_X` also exists before assuming the composition API is symmetric
- **Initialization order matters**: Middleware pipelines are order-dependent -- register middleware before routes
- **Wildcard routes for nested paths**: When file paths can contain `/`, use wildcard (`*`) routes not named params (`:path`)
- **CORS via response helpers**: When middleware architecture doesn't support post-processing, inject CORS headers in response factory methods instead

This is the first challenge that makes the language toolchain externally accessible. Previous challenges were all library-level compositions -- you could import them and call functions. The Web IDE serves HTML and JSON over HTTP, meaning a browser can interact with the IDE. This is a qualitative shift: from composing systems for programmatic use to composing systems for human use.

The bug patterns were mild -- missing API symmetry and init order. Both are composition boundary issues but neither is the deep "compensating bugs" type seen at difficulty 5 in earlier challenges. The reason: the HTTP layer is a thin translation layer, not a complex composition. It maps 1:1 to IDE methods with minimal transformation. Clean composition boundaries produce fewer bugs.
