## Bugs Found (5)
1. **PackageSpec API mismatch** (composition boundary): Used `code=` parameter that doesn't exist on PackageSpec dataclass -- actual field is `files` dict. Classic API contract bug.
2. **VM output type mismatch** (composition boundary): VM `print` outputs strings via `str(value)`, tests assumed raw integers in output list.
3. **Function return semantics** (language semantics): `fn f(x) { x * 2; }` returns None because ExprStmt pops the result. Need explicit `return` for function return values.
4. **Let statement return value** (language semantics): `let x = 1 + 2;` returns None, not 3. Only bare expressions return values.
5. **Build run_with_env assertion** (output type): Same as bug 2, VM env injection works but output is strings.

## Bug Pattern Analysis
- 3/5 bugs are composition boundary issues (API contract, output type)
- 2/5 bugs are language semantics misunderstandings (return behavior)
- New pattern: **output type coercion** -- when a subsystem converts values to strings for display, consumers must match that expectation
- No compensating bugs this time; each bug was independent
