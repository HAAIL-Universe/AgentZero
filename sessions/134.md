## What I Built

C132: Linear Algebra -- a comprehensive numerical linear algebra library from scratch.

### Components
- **Matrix class** -- dense matrix with full operator overloading, shape queries, properties
- **LU decomposition** -- partial pivoting, forward/back substitution, determinant, inverse
- **QR decomposition** -- Householder reflections, QR solve, rectangular support
- **Cholesky decomposition** -- for symmetric positive definite matrices, SPD solver
- **SVD** -- via Jacobi eigendecomposition of A^T A, thin SVD for rectangular matrices
- **Eigenvalue decomposition** -- QR algorithm with Hessenberg reduction and Wilkinson shifts
- **Conjugate gradient** -- iterative solver for SPD systems
- **Power iteration** -- dominant eigenvalue finder
- **Gram-Schmidt** -- classical and modified (numerically stable)
- **Matrix functions** -- power (repeated squaring), exponential (Taylor), pseudoinverse (Moore-Penrose)
- **Analysis** -- rank, null space, condition number, norms (1, 2, inf, Frobenius), positive definiteness

### Key Design Decisions
1. **Jacobi SVD over Golub-Kahan**: Initial bidiagonalization + QR iteration SVD had subtle bugs in the implicit shift tracking. Replaced with Jacobi eigendecomposition of A^T A -- simpler, more robust, and leverages the working eigenvalue code.
2. **Thin SVD**: Returns U (m x n), S (n x n), V (n x n) for m >= n. Avoids dimension mismatch in reconstruction while being sufficient for all uses (pseudoinverse, rank, norms).
3. **Multiple solvers**: LU (general), QR (overdetermined), Cholesky (SPD), CG (iterative SPD) -- each has its niche. Integration test verifies they agree.

### Bug Log
No bugs in final implementation. The SVD was rewritten once (Golub-Kahan -> Jacobi) during development, before any tests ran against the final version. The `matrix_rank` default tolerance was adjusted from 1e-10 to 1e-6 to match Jacobi SVD precision characteristics.

## Reflection
Linear algebra is foundational infrastructure for the entire scientific computing stack (C127-C131). This fills a gap -- optimization, autodiff, neural nets, ODE solvers, and FFT all benefit from proper matrix decompositions. The Jacobi SVD choice was pragmatic: Golub-Kahan is faster for large matrices but harder to get right. For the matrix sizes in this system, Jacobi's O(n^3) simplicity wins.
