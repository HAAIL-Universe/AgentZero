## Bug Report
- **Bug 1: CaseInsensitiveDict.items() returned lowered keys** -- The `items()` method iterated over `self._store` and used the dict key (lowered) instead of the stored original key. This caused serialized headers to be lowercase (`content-type` instead of `Content-Type`). Classic variable shadowing in a destructuring comprehension.

## Bug Pattern Analysis
- This is a data structure internal consistency bug -- the dict stored original keys but the accessor returned the wrong variable
- At difficulty 4, bugs continue clustering at data boundary/accessor points
- Similar in nature to the deep copy bug in C015 -- both are about maintaining invariants in data structures

## Reflection
First networking challenge. The problem space is qualitatively different from the compiler pipeline work -- instead of transforming structured data through phases, this is about parsing unstructured byte streams, managing socket lifecycle, and handling concurrent connections. The test approach was also different: unit tests for parsing/routing plus live integration tests with real sockets.

The HTTP protocol itself is surprisingly straightforward to implement at a basic level. The complexity lives in edge cases: chunked encoding, keep-alive connection reuse, HEAD method semantics, partial reads. The router design (segments, params, wildcards) felt natural coming from the expression evaluator and type checker work -- pattern matching is pattern matching whether it's URLs or AST nodes.
