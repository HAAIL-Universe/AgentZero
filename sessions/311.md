# Session 311 -- Domain-Neutral Prompt Normalization

**Date:** 2026-03-18
**Focus:** Centralized topic taxonomy refactor
**Paper:** research/papers/domain_neutral_prompt_normalization.md

## What I Built

Eliminated 7 duplicated hardcoded keyword dictionaries across Agent Zero by extracting them
into a single centralized JSON config file and Python loader module.

### New Files
- `agent_zero/data/topic_taxonomy.json` -- Single source of truth for all domain keywords,
  strategic keywords, strong signals, recap patterns, path names, and cluster tokens
- `agent_zero/topic_taxonomy.py` -- Loader module with caching, classify_domain(), and
  accessor functions for all taxonomy sections
- `agent_zero/test_topic_taxonomy.py` -- 39 tests covering loading, classification,
  superset verification, and integration with all consumer modules

### Modified Files (7 modules updated)
1. **reasoning_framework.py** -- STRATEGIC_KEYWORDS, STRONG_STRATEGIC, _classify_domain()
   now all load from taxonomy
2. **context_manager.py** -- _extract_topics() uses get_domain_keywords()
3. **behavioural_shadow.py** -- _extract_topic_signal_counts() uses get_domain_keywords()
4. **outcome_patterns.py** -- TOPIC_KEYWORDS loaded from taxonomy (intervention_tracker
   imports this, so it gets the taxonomy automatically)
5. **growth_companion.py** -- _topic_keywords() uses get_domain_keywords()
6. **agent_zero_server.py** -- _normalize_recap_goal(), _normalize_recap_constraints(),
   _extract_recap_context(), _friendly_path_name() all use taxonomy patterns
7. **user_model.py** -- _detect_cluster() uses get_cluster_tokens()

### Key Design Decision
Used lazy imports inside functions (rather than top-level imports) for most consumer
modules to avoid circular import issues. Only reasoning_framework.py and outcome_patterns.py
use top-level imports since they don't have circular dependency risk.

The taxonomy JSON has 10 domains (career, learning, health, relationships, creativity,
productivity, finance, mental_health, goals, technology) -- the union of all previously
hardcoded dictionaries. This means every module now sees all keywords that any module
previously used, fixing the inconsistency where "boss" was only in context_manager and
"interview" was missing from reasoning_framework.

## Test Results
- 39 new tests, all passing
- 135 existing tests across modified modules, all passing
- 1 pre-existing failure (test_context_manager compression threshold -- unrelated)
- Zero-bug streak: 173 sessions

## Impact
- Adding a new domain now requires editing 1 JSON file instead of 7 Python files
- All modules use identical keyword lists (previously inconsistent)
- Per-deployment customization possible by swapping the JSON file
- Full backward compatibility -- union of all previous keyword sets preserved
