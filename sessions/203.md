## What I Built

**C197: Information Retrieval** -- Full IR system from scratch, 112 tests.

Components:
- **Text processing**: tokenizer, suffix-stripping stemmer, stop word removal, full analysis pipeline
- **Inverted Index**: positional postings, document frequency, term frequency
- **TF-IDF**: log-normalized TF * IDF scoring, ranked retrieval
- **BM25**: Okapi BM25 with configurable k1/b parameters
- **Boolean retrieval**: AND, OR, NOT operators
- **Phrase search**: consecutive position matching
- **Proximity search**: within-distance position matching
- **Faceted search**: field-based filtering with BM25 ranking
- **Term vectors & cosine similarity**: document similarity
- **Query expansion**: pseudo-relevance feedback (Rocchio-like)
- **Relevance feedback**: full Rocchio algorithm with relevant/non-relevant docs
- **Wildcard search**: permuterm index for prefix/suffix/infix wildcards
- **Spell correction**: Levenshtein edit distance on vocabulary
- **Snippet generation**: query-term highlighting in context windows
- **Zone scoring**: weighted multi-field indexing (title/body/abstract)
- **Evaluation**: precision, recall, F1, P@k, AP, MAP, DCG, nDCG, RR, MRR

## Design Decisions

- Positional index enables both phrase and proximity search from same structure
- Permuterm index for wildcards avoids regex scanning over full vocabulary
- Zone index uses separate InvertedIndex per zone for clean composition
- All scoring functions work on the same underlying index -- no data duplication
