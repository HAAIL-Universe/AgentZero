## Bugs Found and Fixed

7 total bugs, 4 real composition boundary bugs:

1. **API import mismatch** (x3): `Const` vs `IntConst`, `App` arity, no `~` operator -- standard API surface bugs
2. **NOT(EQ) vs NEQ in DPLL(T)** -- the SMT solver handles NEQ correctly but NOT(EQ) returns UNSAT incorrectly. Fixed via complement operator optimization in `smt_not()` that converts NOT(GT) -> LE, NOT(EQ) -> NEQ, etc.
3. **Function fork propagation** -- function calls with internal branching silently dropped fork paths. Required ForkSignal mechanism to propagate function-internal forks to caller.
4. **Function paths marked COMPLETED prematurely** -- `_execute_if` marks the original state as COMPLETED when forking, which prevented continuation after function return. Fixed by resetting status to ACTIVE.
5. **completed_paths leaking into function scope** -- `_execute_if` adds to global `completed_paths` even during function body execution. Fixed by save/restore of completed_paths.

## New Bug Patterns

- **NOT(EQ) asymmetry in DPLL(T):** Direct complement operators (NEQ, LE, etc.) are handled correctly by theory solvers, but wrapping in NOT() creates Tseitin variables that don't get proper theory propagation. Prevention: always use direct complement operators when negating comparisons.
- **Function fork propagation:** When a called function contains symbolic branches, each branch produces a different return value. The caller must fork into N paths (one per function exit). This is a new class of composition boundary bug -- forking within a sub-computation.
- **Global state leakage in nested execution:** Methods that modify shared state (like completed_paths) must be scoped when called from nested contexts (function execution). Prevention: save/restore pattern.

## Composition Insight

Symbolic execution is a **deep composition** -- it uses C010's parser to build ASTs and C037's SMT solver as a constraint oracle. Unlike previous compositions where systems interact at API boundaries, here the symbolic executor *reinterprets* the AST semantically. The bug patterns are qualitatively different from API composition bugs: they arise from semantic mismatches between concrete and symbolic execution models.
