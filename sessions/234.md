## What I built

### 1. Complete Auth UI (agent_zero.html rewrite)
- Login/register screen with toggle between modes
- JWT stored in localStorage, auto-checked on page load via /auth/me
- Logout clears token and resets to auth screen
- Display name shown in toolbar + status bar
- Enter key submits auth form

### 2. JWT-wired WebSocket
- Token passed as query parameter to /ws/chat
- Server authenticates, creates session, loads shadow
- Reconnects automatically on disconnect (with token)

### 3. Tab system (Chat / Requests / Shadow)
- Chat tab: full conversation UI (unchanged aesthetics)
- Requests tab: lists all update requests, approve/reject buttons
- Shadow tab: raw JSON view of behavioural profile

### 4. Update Requests panel
- Renders all requests with status badges (pending/approved/rejected/built)
- Approve button -> POST /api/requests/{id}/approve
- Reject button -> prompt for reason -> POST /api/requests/{id}/reject
- Auto-refreshes on tab switch

### 5. Stale endpoint cleanup
- Removed references to old /api/history and /api/clear
- new session just reconnects WebSocket (server creates new session per connection)

## Architecture summary

```
Browser -> agent_zero.html
  |-> Auth screen (login/register)
  |-> JWT stored in localStorage
  |-> WebSocket /ws/chat?token=JWT
       |-> Server authenticates, creates session
       |-> Loads behavioural shadow
       |-> Chat with streaming tokens
       |-> Shadow update on disconnect
       |-> Curiosity questions injected mid-conversation
       |-> [REQUEST:] tags auto-extracted
  |-> REST endpoints (all JWT-protected):
       /api/sessions, /api/shadow, /api/requests
```
