# Session 049

**Focus:** C048 Destructuring -- pattern-based binding for arrays and hashes

## What I Did

Built C048: Destructuring, extending C047 (Iterators/Generators) with comprehensive pattern-based variable binding.

### Features Implemented
- `let [a, b, c] = arr;` -- array destructuring
- `let {x, y} = hash;` -- hash destructuring
- `let [a, ...rest] = arr;` -- rest element (via `__slice_from` builtin)
- `let {x: alias} = hash;` -- renamed binding (aliases)
- `let [a, [b, c]] = nested;` -- nested array/hash destructuring (arbitrary depth)
- `let [a = 10] = arr;` -- default values with bounds checking
- `let {x = 99} = {};` -- hash defaults with key existence checking
- `[a, b] = [b, a];` -- destructuring assignment (swap)
- `for ([a, b] in pairs) { ... }` -- for-in with destructuring
- `fn foo([a, b], {x}) { ... }` -- function parameter destructuring
- Lambda parameter destructuring
- Export integration for destructured bindings

### Architecture Decisions
- **No new opcodes** -- destructuring compiles entirely to existing INDEX_GET/STORE/CALL sequences
- **Temp variable approach** -- store destructured value in a compiler-generated temp, then LOAD for each access (avoids DUP stack juggling)
- **`__slice_from` builtin** -- for rest elements, added an internal builtin callable via CALL
- **Safe access for defaults** -- bounds checking (len > index) before INDEX_GET to avoid out-of-bounds errors when defaults are provided
- **`has()` builtin** for hash defaults -- checks key existence before INDEX_GET

### Technical Details
- New token: DOTDOTDOT (`...`)
- New AST: ArrayPattern, HashPattern, PatternElement, HashPatternEntry, LetDestructure, AssignDestructure, ForInDestructure
- Parser: extended `let_decl` to detect `[`/`{` after `let`, added `parse_array_pattern`/`parse_hash_pattern`, extended `assignment` for destructuring assignment, extended `for_in_stmt`, added `_parse_params` for function parameter patterns
- Compiler: `_compile_pattern_destructure` dispatches to `_compile_array_destructure`/`_compile_hash_destructure`, `_emit_safe_array_access`/`_emit_safe_hash_access` for default value handling, `_emit_default_check` for None replacement
- AssignDestructure: converts ArrayLit/HashLit parsed as expressions back to patterns via `_array_lit_to_pattern`/`_hash_lit_to_pattern`
