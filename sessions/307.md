## What I Did

Implemented the JWT Security Hardening research paper (Phases 1-3 of 4):

### Phase 1: Fail-Fast Secret Validation (auth.py)
- Added `_get_jwt_secret()` with environment-aware validation
- Production (`AGENT_ZERO_ENV=production`): crashes on default/weak secret
- Development: warns but allows default secret for local dev
- All environments: rejects secrets shorter than 32 characters
- Includes actionable error message with `secrets.token_hex(32)` generation command

### Phase 2: JTI + Token Denylist (auth.py + agent_zero_server.py)
- Added `jti` (UUID) claim to all created tokens
- Built `TokenDenyList` class -- thread-safe in-memory denylist with TTL cleanup
- `decode_token()` now checks denylist before returning payload
- Added `POST /auth/logout` endpoint -- revokes current token server-side
- Added background reaper task (`_reap_expired_denylist_entries`) every 10 min
- Exposed `get_denylist()` accessor for use in routes

### Phase 3: First-Message WebSocket Auth (server + both frontends)
- Added `_ws_first_message_auth()` helper -- accepts `{"type":"auth","token":"..."}` as first message
- Responds with `{"type":"auth_ok"}` or `{"type":"auth_error"}`
- Backward-compatible: falls back to query-param token if first message isn't auth (with deprecation warning)
- Updated `/ws/chat` and `/ws/voice` endpoints to use first-message auth
- Updated React frontend (App.tsx): sends auth message on socket open, handles auth_ok/auth_error
- Updated legacy frontend (agent_zero.html): same pattern, removed token from query param
