## The Bug

CZT Bluestein decomposition: swapped chirp[k] and conj(chirp[k]) in three places.
The identity nk = (n^2 + k^2 - (k-n)^2)/2 requires:
- y[n] = x[n] * A^{-n} * W^{n^2/2} = x[n] * a^{-n} * chirp[n]
- h[n] = W^{-n^2/2} = conj(chirp[n])
- X[k] = chirp[k] * (y conv h)[k]

I had conj where chirp should be and vice versa. Classic sign/conjugate confusion
in Bluestein's algorithm. This breaks the 92-session zero-bug streak.

## Key Design Decisions

1. **Iterative radix-2**: Used iterative (not recursive) FFT with bit-reversal permutation -- avoids stack depth issues for large n
2. **Bluestein for arbitrary sizes**: Rather than zero-padding all inputs to power-of-2, Bluestein preserves exact DFT semantics for any n
3. **Conjugate trick for IFFT**: IFFT(X) = conj(FFT(conj(X)))/n -- reuses forward FFT code
4. **Overlap-add ISTFT**: Window normalization by sum of squared windows at each sample
