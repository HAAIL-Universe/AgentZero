### Corpus Preparation (prepare_corpus.py)
- Read all 15 corpus files, analyzed every concept, agent role, and ethical principle
- Generated 50 high-quality instruction-following Q&A pairs covering:
  - Identity and purpose (what is Agent Zero, core values)
  - Architecture (5-layer cognitive architecture, all agent roles)
  - Ethics (EthicsGuardrail, manifesto, conflict handling)
  - Autonomy and consent (4 tiers, consent dashboard)
  - Reasoning and uncertainty (transparency, confidence thresholds)
  - Sub-agents (all 7 data harvesters + CentralHub)
  - Practical scenarios (deception, distress detection, goal conflicts)
  - Design philosophy (modularity, privacy-by-design, forward compatibility)
  - Future-proofing and emergence safeguards
- Output: `data/agent_zero_training.jsonl` in Phi-3 chat template format

### Fine-Tuning Script (finetune_phi3.py)
- Custom training loop (HF Trainer has device placement issues with split models)
- LoRA config: r=8, alpha=16, targeting qkv_proj, o_proj, gate_up_proj, down_proj
- 12.5M trainable params (0.33% of 3.8B total)
- Training on CPU (float16) -- GPU can't hold full 7.6GB model
- Cosine LR schedule with warmup, gradient clipping, AdamW optimizer
- 3 epochs, batch 1, grad accum 4, ~37 optimizer steps

### Technical Challenges
1. **HF Trainer device_map incompatibility**: When model is split across GPU+CPU via
   device_map="auto", Trainer tries to .to(device) which hits meta tensors. Solved by
   writing a custom training loop.
2. **GPU memory**: 7.6GB fp16 model doesn't fit in 6.4GB VRAM. Training runs on CPU.
3. **Output buffering**: Python stdout buffering hid training progress. Fixed with
   line-buffered stdout.
4. **Float32 OOM**: Loading model in float32 (~15GB) segfaults on 16GB system.
   Float16 (~7.6GB) fits.

## Key Insight

The gap between "I built ML from scratch" and "I can fine-tune a real model" is not about
capability -- it's about systems integration. The ML knowledge (LoRA, attention, optimization)
was all things I've built before. The hard part was getting HuggingFace's Trainer, PyTorch's
device placement, and a 3.8B parameter model to cooperate on a 6GB GPU + 16GB RAM machine.

The solution was the same principle I've applied 182 times: when frameworks fight you,
write your own training loop. Control beats convenience.
