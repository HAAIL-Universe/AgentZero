## What I Built

State-based (CvRDT) library implementing 12 CRDT types:

**Counters:** GCounter, PNCounter
**Registers:** LWWRegister (timestamp tie-break), MVRegister (vector clock, multi-value on conflict)
**Sets:** GSet, TwoPSet (permanent removal), ORSet (add-remove-add cycles), LWWElementSet (timestamp per element)
**Flags:** EWFlag (enable-wins), DWFlag (disable-wins)
**Sequences:** RGASequence (Replicated Growable Array with tombstones)
**Maps:** ORMap (observed-remove map with nested CRDT values)
**Utilities:** VectorClock, CRDTNetwork (convergence testing simulator)

All types satisfy the CRDT lattice properties: commutativity, associativity, idempotency.

## Key Decisions

- Pure state-based (CvRDT) -- full state shipped on merge, simpler than op-based
- ORSet uses UUID tags for add/remove tracking -- handles add-remove-add cycles correctly (unlike TwoPSet)
- MVRegister uses vector clocks to detect concurrent writes -- surfaces conflicts as multiple values
- RGASequence uses tree-based linearization with (counter, replica) IDs for ordering
- LWWElementSet has add-bias on timestamp ties (>= not >)
- CRDTNetwork provides full mesh sync for convergence testing

## Bug Found & Fixed

- LWWElementSet `contains()` used `>` instead of `>=` for add-bias on timestamp ties (1 test caught it, fixed immediately)
