## What I Built

C150: GAN -- Generative Adversarial Networks composing C140 (Neural Networks) + C149 (Autoencoder).

### 6 GAN Variants

1. **GAN** (Vanilla) -- Minimax adversarial training with BCE loss, label smoothing, multiple optimizer options (Adam/SGD/RMSProp)
2. **WGAN** -- Wasserstein GAN with weight clipping critic, unbounded scores, Wasserstein distance estimation
3. **LSGAN** -- Least Squares GAN replacing cross-entropy with MSE for more stable gradients
4. **ConditionalGAN** -- Class-conditional generation via one-hot label concatenation
5. **InfoGAN** -- Information-maximizing GAN with categorical + continuous latent codes, Q-network, mutual information loss
6. **BiGAN** -- Bidirectional GAN with inference encoder, learns data->latent mapping simultaneously

### Supporting Components

- **Generator** -- Reuses C149 Decoder architecture for noise->data mapping
- **Discriminator** -- Configurable sigmoid/no-sigmoid, weight clipping, leaky_relu default
- **GANMetrics** -- Distribution distance, mode coverage, discriminator score analysis
- **4 data generators** -- Gaussian mixture, line, grid, ring

## Key Design Decisions

- Generator reuses C149's Decoder class directly (true composition, not reimplementation)
- WGAN critic has no sigmoid -- outputs raw unbounded score
- InfoGAN Q-network is separate from D (shared architecture, independent weights) -- simpler gradient flow
- ConditionalGAN concatenates one-hot labels to both G input and D input
- BiGAN discriminator sees (data, latent) pairs -- real=(x, E(x)), fake=(G(z), z)
- All variants share common `fit()` API pattern for consistency

## Composition Pattern

- C140 provides: Tensor, Dense, Activation, Sequential, Dropout, optimizers
- C149 provides: Encoder, Decoder, BCE/MSE loss utilities
- C150 adds: adversarial training loops, Wasserstein distance, mutual information
