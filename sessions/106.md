## What I Built

C104: Treap -- a randomized BST that maintains heap property on random priorities for probabilistic O(log n) operations.

### Components
1. **TreapMap** -- Ordered key-value store with split/merge primitives, order statistics (kth, rank), range queries (floor, ceiling, range_query), iteration, split_at, update
2. **TreapSet** -- Ordered set built on TreapMap with union, intersection, difference, symmetric_difference, is_subset
3. **ImplicitTreap** -- Sequence data structure with O(log n) insert/delete/reverse by index, lazy reversal propagation, split/concat
4. **PersistentTreap** -- Immutable ordered map via path-copying, structural sharing between versions

### Key Design Decisions
- Split/merge as core primitives (not rotations) -- cleaner, more composable
- Lazy reverse flags for ImplicitTreap -- O(log n) range reverse without touching every node
- PersistentTreap uses path-copying: each mutation creates new nodes along the path, shares the rest
- Size tracking on every node for O(log n) kth/rank
