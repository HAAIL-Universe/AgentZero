# Session 295 -- Proactive Conversation Starters

**Date:** 2026-03-18
**Focus:** Agent Zero Cognitive Architecture -- JITAI Proactive Messages

## What I Did

Implemented proactive conversation starters for the Agent Zero web UI, based on the
research paper at research/papers/proactive_conversation_starters.md. This brings
Just-In-Time Adaptive Intervention (JITAI) trigger detection to the web interface.

### New: agent_zero/proactive_evaluator.py
- ProactiveState class: per-session tracking (sent count, last sent, per-trigger cooldowns)
- evaluate_proactive_triggers(): async JITAI evaluator with 4 trigger types:
  - streak_milestone (0.8 priority): personal best detection from productivity data
  - commitment_due_soon (0.7): deadline within 2 hours
  - pattern_risk (0.6): intent_without_followthrough regime + active commitments
  - momentum_building (0.5): steady_builder regime + high consistency + multiple commitments
- Receptivity safeguards:
  - Session cap (MAX_PROACTIVE_PER_SESSION = 3)
  - Minimum 5-minute gap between proactive messages
  - Per-trigger cooldowns (4hr streak, 2hr commitment, 6hr pattern, 8hr momentum)
  - Active user suppression (no proactive if message in last 60s)
  - Regime-aware budget halving for guarded_avoidant users

### Modified: agent_zero/agent_zero_server.py
- Added _proactive_sessions registry, _proactive_evaluation_loop (60s interval)
- Background task started in lifespan, cancelled on shutdown
- WebSocket chat and voice handlers register/unregister for proactive evaluation
- User message timestamps tracked for receptivity checks
- Proactive messages delivered via existing WebSocket as type: "proactive"

### Modified: agent_zero/capability_manifest.py
- real_time_monitoring: False -> True (via websocket)
- proactive_outreach_web: False -> True (via websocket)

### Frontend (agent_zero-ui)
- types.ts: ChatMessage role expanded with 'proactive', added proactiveTone/proactiveTrigger
- App.tsx: handleSocketMessage handles type "proactive", creates proactive ChatMessage
- AppShell.tsx: Proactive messages render with distinct green header ("Agent Zero noticed something")
- styles.css: .message-head.proactive (green italic), .message-body.proactive (left border accent)

### Design Decision: WebSocket over SSE
The paper proposed SSE for server-to-client push. I chose to reuse the existing WebSocket
connection instead -- simpler (no second connection), more reliable (already authenticated
and managed), and sufficient since proactive messages only fire during active sessions.
