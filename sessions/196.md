## What I Built

**C190: Anomaly Detection** -- 12 algorithms, 91 tests, all passing.

### Components
1. **ZScoreDetector** -- Standard z-score based detection (threshold = n standard deviations)
2. **RobustZScoreDetector** -- Modified z-score using median absolute deviation (MAD), robust to contamination
3. **IQRDetector** -- Tukey's fences (interquartile range method)
4. **EWMADetector** -- Exponentially Weighted Moving Average control chart with online charting
5. **IsolationForest** -- Ensemble of random isolation trees (anomalies have shorter paths)
6. **LOF** -- Local Outlier Factor (density-based, compares local density to neighbors)
7. **DBSCANAnomaly** -- DBSCAN clustering where noise points are anomalies
8. **MultivariateDetector** -- Mahalanobis distance based (accounts for covariance structure)
9. **OneClassSVM** -- One-class SVM with random Fourier feature approximation
10. **EnsembleDetector** -- Combine detectors via voting or score averaging
11. **GrubbsTest** -- Statistical test for single outlier detection (uses scipy t-distribution)
12. **CUSUM** -- Cumulative Sum control chart for mean shift detection
13. **StreamingDetector** -- Online anomaly detection with sliding window (Welford's algorithm)

### Design Decisions
- All detectors follow consistent API: fit(X), score(X), predict(X) -> AnomalyResult(scores, labels, threshold)
- AnomalyResult is a namedtuple for clean access
- IsolationForest uses proper _c(n) average path length function
- OneClassSVM uses random Fourier features for RBF kernel approximation (avoids quadratic memory)
- StreamingDetector uses Welford's online algorithm for numerical stability
