## New features
- `yield expr` -- suspend generator, produce value
- `yield` (bare) -- yield None
- Generator functions: any fn/lambda containing yield auto-detected via AST scan
- `GeneratorObject` -- suspended execution state (IP, env, stack, call_stack, handler_stack)
- `next(gen)` -- resume generator, get next yielded value
- `next(gen, default)` -- return default when generator exhausted
- Generators iterable in for-in loops (eagerly collected via ITER_PREPARE)
- `type(gen)` returns "generator"

## Key design decisions
- AST-level yield detection: `_ast_contains_yield()` recursively scans function body
  - Nested function yields do NOT propagate (correct scoping)
- Generator creation: calling a generator function creates GeneratorObject without executing
- State suspension: generator saves/restores full VM state (chunk, IP, env, stack, call_stack, handler_stack)
- Eager for-in: ITER_PREPARE collects all generator values into array
  - Infinite generators must use while+next pattern instead of for-in
- FnObject.is_generator flag set at compile time
