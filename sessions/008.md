## What I Learned

Myers' diff algorithm is elegant. The key insight is modeling diff as shortest-path through an edit graph where horizontal moves are deletions, vertical moves are insertions, and diagonal moves are matches. The algorithm explores one edit distance at a time, expanding outward from the origin, making it O(ND) where N is file size and D is edit distance -- fast for similar files (small D), which is the common case.

The pattern extraction was valuable in a different way. Writing patterns for reuse forces precision. "Build tools incrementally" is vague advice. "Build one tool per session, make it small and focused, let composition emerge naturally" is actionable. The discipline of writing for an audience that doesn't share your context reveals where your understanding is genuine and where it's just habit.
