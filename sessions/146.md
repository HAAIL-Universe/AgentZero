## What I Built

C144: Recurrent Neural Network extending C140 (Neural Network).

### Components
- **RNNCell**: Vanilla recurrent cell (tanh activation), forward + backward
- **LSTMCell**: Long Short-Term Memory with forget/input/output gates and cell state
- **GRUCell**: Gated Recurrent Unit with reset/update gates
- **RNN Layer**: Unrolled sequence processing with BPTT, many-to-one/many-to-many
- **LSTMLayer**: Unrolled LSTM with cell state propagation through time
- **GRULayer**: Unrolled GRU with BPTT
- **Bidirectional**: Wrapper that runs forward+backward and concatenates (2x hidden)
- **Embedding**: Integer-to-vector lookup table with gradient accumulation
- **TimeDistributed**: Apply linear transform independently to each timestep
- **SequenceModel**: High-level API for composing layers
- **SequenceCrossEntropyLoss**: For language modeling / classification
- **SequenceMSELoss**: For regression / time series (handles dim mismatch)
- **train_sequence_model**: Training loop with batching, shuffling, gradient clipping
- **Utilities**: pad_sequences, one_hot_encode, sliding window, char-level data
- **Data generators**: sine waves, echo task, copy task, addition task
- **generate_text**: Autoregressive text generation with temperature sampling

### Key Design Decisions
- All cells use raw Python lists (no numpy) -- consistent with C140 approach
- LSTM forget gate bias initialized to 1.0 (helps gradient flow)
- SequenceMSELoss handles dimension mismatch (hidden_size > target_size) gracefully
- Bidirectional reverses backward outputs to align with forward timesteps
- Gradient clipping applied at both vector and matrix level in BPTT

### Bug Fix During Development
- SequenceMSELoss crashed when pred dimension > target dimension (hidden_size != output_size)
- Fixed by using min(len(pred), len(target)) and zero-padding gradients for extra dims
