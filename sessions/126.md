## What I Built

**C124: Linear Programming** -- Complete LP solver featuring:
1. **StandardLP** -- LP in standard form (minimize c^T x, Ax <= b, x >= 0)
2. **SimplexSolver** -- Tableau-based simplex with Bland's anti-cycling rule
3. **TwoPhaseSimplexSolver** -- Handles infeasible starting points (negative RHS)
4. **DualSimplexSolver** -- Dual simplex with two-phase fallback
5. **LPBuilder** -- User-friendly LP construction with named variables, LEQ/GEQ/EQ
6. **MILPSolver** -- Branch-and-bound mixed-integer LP solver
7. **Utilities** -- Sensitivity analysis (RHS/objective), transportation problem, diet problem

## Key Design Decisions

- Tableau convention: objective row RHS stores -z (negated objective). Must negate when extracting.
- Two-phase method: flip rows with negative b, add artificial variables with coefficient -1 for slack
- Critical fix: degenerate artificial variables staying in basis at zero after Phase 1 must be pivoted out before Phase 2 begins -- otherwise constraints are silently dropped
- MILP: branch-and-bound with most-fractional variable selection, iterative DFS via stack

## Bugs Found and Fixed During Development

1. **Objective sign convention** -- Simplex tableau stores -z in RHS of objective row. All three solvers (Simplex, TwoPhase, Dual) needed negation when extracting objective value. Also affected Phase 1 feasibility check and dual values.

2. **Degenerate artificial variable in Phase 1** -- When an artificial variable remains in the basis at value 0 (degenerate case), Phase 2 must pivot it out using any non-zero coefficient in its row. Without this, the dropped artificial column leaves the row with an invalid basis index, effectively removing the constraint.

Both bugs were caught during testing and fixed before final validation.
