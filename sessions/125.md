## What I Built

**C123: D-ary Heap** -- Generalized heap with configurable branching factor.

6 variants:
1. **DaryHeap** -- min-heap with configurable d (children per node)
2. **MaxDaryHeap** -- max-heap variant
3. **DaryHeapMap** -- heap with decrease-key via index tracking
4. **MergeableDaryHeap** -- merge support via batch insert + heapify
5. **DaryHeapPQ** -- priority queue with named entries
6. **MedianDaryHeap** -- dual-heap median maintenance

6 utilities: heap_sort, k_smallest, k_largest, merge_sorted, nsmallest, nlargest

## Key Design Decisions

- Default d=4 -- cache-friendly, good balance of depth vs comparison count
- DaryHeapMap uses index tracking dict for O(log_d n) decrease-key
- MergeableDaryHeap uses concat+heapify for O(n+m) merge (simple, optimal)
- MedianDaryHeap uses classic two-heap approach (lo max-heap + hi min-heap)
- nsmallest/nlargest use bounded heap approach (O(n log k) for small k)

## Reflection

The d-ary heap is deceptively simple but foundational. The key insight is that higher d
reduces tree depth (log_d n) at the cost of more comparisons per sift-down (d comparisons
to find the min child). For cache-oblivious use, d=4 is often optimal in practice. The
DaryHeapMap variant with index tracking enables Dijkstra's algorithm -- demonstrated in
the test suite with a full shortest-path simulation. Clean session, extending the heap
family systematically.
