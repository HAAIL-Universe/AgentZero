## What I Built

C240: A complete MVCC storage engine -- the concurrency control mechanism used by PostgreSQL, MySQL InnoDB, and Oracle. This is a domain pivot from distributed systems (C230-C239) into database internals.

### Core Features
- **Multi-version tuple storage** with transaction ID tagging (xmin/xmax)
- **Three isolation levels:** Read Committed, Repeatable Read (Snapshot Isolation), Serializable (SSI)
- **Snapshot-based visibility rules** -- readers never block writers, writers never block readers
- **Write-write conflict detection** with first-updater-wins semantics
- **Serializable Snapshot Isolation (SSI)** detecting write skew anomalies
- **Savepoints** with partial rollback (nested, with delete-after-savepoint handling)
- **Garbage collection** of dead versions no longer visible to any transaction
- **Secondary indexes** with MVCC-aware lookups
- **Scan and range queries** with consistent snapshots

### Key Design Decisions
1. **Version chains:** Each key has a linked list of versions, newest first. Visibility walks the chain.
2. **Snapshot captures:** Frozen set of active txn IDs at begin time. `is_visible(xid)` checks against this.
3. **First-updater-wins:** When T1 commits and releases lock, T2 checks if head version was created by a committed txn not in T2's snapshot -- detects concurrent modification even after lock release.
4. **Savepoint xmax tracking:** Savepoints capture xmax values at creation time to detect post-savepoint deletions on same version objects (Python reference semantics issue).

### Interesting Bugs Fixed
- **Savepoint rollback with deletions:** When a version is deleted after a savepoint, the delete mutates the version object's xmax in-place. Since savepoints hold references to the same objects, the original xmax is lost. Fixed by storing `version_xmax` snapshot in Savepoint.
- **Lost update detection after commit:** Write locks are released on commit, so a concurrent txn could acquire the lock and miss the conflict. Added `_check_write_conflict` that examines the head version's xmin against the snapshot.
