### 3. BM25 Semantic Memory Retrieval (Research Paper Implementation)
Created `bm25_scorer.py` -- Okapi BM25 scoring for memory retrieval.

- IDF weighting: rare terms score higher than common terms
- TF saturation: keyword-stuffed notes don't dominate
- Length normalization: short specific notes compete fairly
- Minimal suffix stemmer: "exercising" matches "exercise"

Integrated into `retrieval_policy.py` and `memory_policy.py`:
- BM25 index built per-turn from all retrievable texts (~38 docs)
- Replaces keyword overlap in sessions, goals, requests, and notes scoring
- Falls back to keyword overlap when scorer not provided (backward compat)
