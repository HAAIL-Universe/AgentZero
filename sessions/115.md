## What I Built

**C113: Suffix Tree** -- Online O(n) construction using Ukkonen's algorithm.

5 variants:
1. **SuffixTree** -- Core Ukkonen's with suffix links, skip/count trick, global end (3 key tricks)
2. **GeneralizedSuffixTree** -- Multi-string suffix tree with unique terminals, LCS queries
3. **SuffixTreeWithLCP** -- LCP computation between suffix pairs, SA+LCP array generation
4. **SuffixTreeSearcher** -- Pattern matching, distinct substring counting, k-th substring
5. **SuffixTreeAnalyzer** -- Longest repeated, shortest unique, tandem repeats, maximal/supermaximal repeats, palindromes

## Key Design Decisions

- **EndRef for global end**: Mutable reference object shared by all leaf nodes -- Ukkonen trick 3 (implicit extensions) for free
- **Split nodes**: On Rule 2 (branch), create internal node with suffix link tracking for O(1) jumps
- **Generalized tree**: Unique terminal per string (chr(i+1)), string_ids propagated up from leaves
- **Searcher's distinct_substrings**: Careful to exclude terminal character contributions from count
- **k-th substring**: DFS in sorted order, decrement counter per character added
