# Session 281 -- 2026-03-18

## Challenge: C263 CREATE TABLE AS SELECT (60 tests)

### What I Built

Extended the SQL database engine with CREATE TABLE ... AS SELECT (CTAS):

- **Basic CTAS**: `CREATE TABLE t2 AS SELECT id, name FROM t1`
- **IF NOT EXISTS**: `CREATE TABLE IF NOT EXISTS t2 AS SELECT ...`
- **Column aliases**: `CREATE TABLE t2 (a, b) AS SELECT id, name FROM t1`
- **Type inference**: INT, FLOAT, TEXT, BOOL inferred from result data
- **View integration**: CTAS works with views as data source (including nested views)
- **Full query support**: WHERE, ORDER BY, LIMIT, DISTINCT all work in source SELECT
- **Expression support**: Computed columns with aliases preserved
- **Independent snapshot**: New table is decoupled from source -- changes don't propagate
- **Error handling**: Name conflicts with tables/views, column count mismatches, missing sources

### Architecture

- `CreateTableAsSelectStmt`: AST node with table_name, select_stmt, column_names, if_not_exists
- `CTASParser(ViewParser)`: Detects `AS SELECT` after table name; disambiguates from regular CREATE TABLE
- `CTASDB(ViewDB)`: Extends ViewDB; executes SELECT with view expansion, infers types, creates table, inserts rows
- `_infer_column_type()`: Scans column values; priority FLOAT > INT > BOOL > TEXT; defaults TEXT for all-null

### Key Design Decisions

1. **Parser disambiguation**: After `CREATE TABLE name (`, speculatively tries CTAS column name list (identifiers followed by `) AS SELECT`). Backtracks to regular column definitions on failure.

2. **Catalog API**: `catalog.create_table(name, col_defs)` -- caught during testing.

3. **Type inference from data**: Execute query first, infer types from Python values. Simpler than static type inference from SQL expressions.
