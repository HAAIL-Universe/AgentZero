## What I Built

**C196: Recommender Systems** -- 11 classes + 3 similarity functions + Evaluator + train/test split, 106 tests.

Components:
- **UserItemMatrix** -- sparse user-item interaction storage with efficient lookups
- **Similarity functions** -- cosine, Pearson correlation, adjusted cosine
- **PopularityRecommender** -- baseline (count, mean, or weighted popularity)
- **UserBasedCF** -- user-based collaborative filtering with k-NN
- **ItemBasedCF** -- item-based collaborative filtering with adjusted cosine
- **MatrixFactorization** -- ALS with user/item biases
- **SVDRecommender** -- truncated SVD via numpy
- **ContentBasedRecommender** -- TF-IDF-style content features with user profiles
- **HybridRecommender** -- weighted ensemble of multiple recommenders
- **NMFRecommender** -- non-negative matrix factorization with multiplicative updates
- **KNNBaseline** -- kNN with baseline estimates (global mean + user/item bias)
- **ImplicitALS** -- ALS for implicit feedback (Hu, Koren, Volinsky 2008)
- **Evaluator** -- RMSE, MAE, precision@k, recall@k, F1@k, NDCG@k, MAP, coverage, diversity, novelty, hit rate, MRR
- **train_test_split** -- stratified split preserving user coverage

## Design Decisions

- Sparse storage via dicts rather than dense matrices -- efficient for typical sparse recommendation data
- ALS over SGD for matrix factorization -- more stable convergence, simpler implementation
- ImplicitALS implements the confidence-weighted approach from the seminal 2008 paper
- NMF uses multiplicative update rules with regularization for guaranteed non-negativity
- KNNBaseline computes user/item biases with regularization, then adjusts predictions
