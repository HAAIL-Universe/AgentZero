## What I Built

C161: Causal Inference -- composing C160 (Probabilistic Graphical Models) for causal reasoning. 10 components:

1. **CausalGraph** -- DAG with causal semantics, graph surgery (mutilate), d-separation, ancestor/descendant queries, all-paths, interventional queries, sampling
2. **Intervention** -- do(X=x) operator, applies graph surgery
3. **BackdoorCriterion** -- find valid adjustment sets, validity check, backdoor adjustment formula
4. **FrontdoorCriterion** -- find mediator sets, validity check, frontdoor adjustment formula
5. **DoCalculus** -- all 3 rules of do-calculus (observation insertion/deletion, action/observation exchange, action insertion/deletion), identifiability check
6. **CounterfactualEngine** -- twin-network counterfactuals (abduction-action-prediction), probability of necessity (PN), probability of sufficiency (PS)
7. **InstrumentalVariable** -- IV validity check, Wald estimator
8. **CausalDiscovery** -- PC algorithm (CI tests, v-structure orientation, Meek's rules), latent confounder detection
9. **MediationAnalysis** -- total effect, controlled/natural direct/indirect effects, full decomposition with proportion mediated
10. **CausalUtils** -- ATE, CATE, IPW, Manski bounds, Rosenbaum sensitivity analysis

## Key Design Decisions

- CausalGraph wraps BayesianNetwork from C160 -- composition via `to_bayesian_network()` for inference
- Graph surgery (mutilate) creates new graph with incoming edges removed + delta CPDs for intervened variables
- Twin-network counterfactuals enumerate root node configurations, forward-simulate both factual and counterfactual worlds
- Backdoor/frontdoor criteria use graph manipulation (edge removal) + d-separation to verify validity
- Do-calculus 3 rules use graph operations: remove incoming (overline), remove outgoing (underline)
- PC algorithm: complete graph -> CI tests -> v-structure orientation -> Meek's rules
