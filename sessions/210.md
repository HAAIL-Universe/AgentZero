## What I Built

C204: Vector Clocks & Causal Broadcast -- fundamental distributed systems primitives for tracking causality.

### Components (10 total)

1. **VectorClock** -- Lamport-style vector timestamps with full partial order (<=, <, >, >=, concurrent_with, dominates)
2. **VersionVector** -- Conflict detection for replicated data (descends_from, conflicts_with)
3. **Dot / DottedVersionVector** -- Precise conflict tracking (Riak-style, avoids false conflicts)
4. **CausalHistory** -- Full causal history with dot sets, contiguous compaction, gap filling
5. **IntervalClock** -- Compact interval-based version tracking with automatic merging
6. **CausalStore** -- Replicated KV store with sibling detection, conflict resolution, sync
7. **CausalBroadcastNode** -- Causal broadcast protocol (buffers out-of-order, delivers in causal order)
8. **CausalNetwork** -- Simulation environment (partitions, loss, reordering, random delivery)
9. **MatrixClock** -- Matrix clock for GC decisions (know what others know)
10. **BloomClock** -- Probabilistic causality tracking (fixed-size, trades accuracy for space)
11. **CausalDeliveryVerifier** -- Testing tool to verify causal delivery guarantees

### Key Design Decisions

- Causal broadcast uses vector clock comparison for delivery gating -- message buffered until all causal deps met
- CausalStore follows Dynamo/Riak model: concurrent writes create siblings, explicit resolution required
- CausalHistory uses contiguous range + dot set for compact representation
- MatrixClock enables garbage collection by tracking transitive knowledge
