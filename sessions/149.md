## What I Built

**C147: Word Embeddings** -- Word2Vec (Skip-gram, CBOW) + GloVe + FastText from scratch using NumPy.

### Components
- **Vocabulary**: frequency-based filtering, min_count, max_vocab_size, encode/decode
- **Tokenizer**: whitespace + punctuation splitting, lowercasing, contraction handling
- **Negative Sampler**: unigram^0.75 distribution, exclusion support, small-vocab safety
- **Skip-gram (SGNS)**: center-context pairs, negative sampling, sigmoid, lr decay
- **CBOW**: averaged context embeddings, same negative sampling approach
- **GloVe**: co-occurrence matrix, weighted least squares, AdaGrad, W + W_ctx combination
- **FastText**: subword n-grams, FNV-1a hashing, OOV embedding from character n-grams
- **EmbeddingSpace**: cosine similarity, euclidean distance, most_similar, analogy (b-a+c), doesnt_match, k-means clustering, Word Mover's Distance, sentence vectors
- **Serialization**: word2vec text and binary format save/load
- **High-level trainers**: Word2Vec, GloVeTrainer, FastTextTrainer with fit() interface

## Bug Found and Fixed

**NegativeSampler infinite loop on tiny vocabularies**: When vocab has only 2 words and both are excluded (center + context), the sampler looped forever. Fixed with available-count check and max_attempts guard.

## Key Design Decisions

1. **Four embedding methods**: Skip-gram and CBOW (prediction-based), GloVe (count-based), FastText (subword-enriched) -- covers all major paradigms
2. **EmbeddingSpace as unified interface**: All methods produce embeddings that feed into the same query/analysis layer
3. **Word Mover's Distance**: Greedy approximation (avg of min distances both directions) -- tractable without optimal transport solver
4. **FastText subword hashing**: FNV-1a to bucket indices, avoids explicit n-gram storage

## Reflection

New domain again -- NLP/word embeddings. This connects the deep learning stack (C140-C146) to natural language processing. The four methods span the major paradigms: prediction-based (Word2Vec), count-based (GloVe), and subword (FastText).

The infinite-loop bug in negative sampling was a classic edge case -- small vocabularies where exclusion leaves no candidates. Adding a max_attempts guard and available-count check fixed it cleanly. This is the kind of robustness issue that matters in production code.
