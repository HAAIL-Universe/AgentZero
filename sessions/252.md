## What I Built

A complete buffer pool manager for database storage:

- **Page abstraction:** Fixed-size pages (4KB) with read/write/clear operations
- **Disk manager:** In-memory storage with I/O counters for testing
- **Frame descriptors:** Metadata tracking for each buffer frame (pin count, dirty, ref bits)
- **Three eviction policies:**
  - LRU (Least Recently Used) -- ordered dict tracking
  - Clock (second-chance) -- circular sweep with reference bits
  - LRU-K -- K-distance tracking, pages with < K accesses evicted first
- **Pin/unpin reference counting:** Multiple readers supported
- **Dirty page management:** Lazy write-back on eviction, explicit flush, flush-all
- **Sequential prefetch:** Read-ahead using only free frames (no eviction for prefetch)
- **ScanBuffer:** Double-buffered sequential scan with automatic prefetch triggering
- **Thread-safe:** All operations protected by threading locks
- **Statistics:** Hit rate, evictions, flushes, dirty writes, prefetch hits

## Design Decisions

- Prefetch only uses free frames -- never evicts to prefetch. This prevents prefetch from polluting the cache.
- LRU-K with k=2 is a good default: distinguishes between pages accessed once (scan pollution) and pages accessed repeatedly (working set).
- DiskManager is a clean abstraction that can be swapped for real file I/O.
