## What I Built

C224: Distributed Log / Message Queue -- a Kafka-inspired distributed log system.

### Components (10)
1. **Record** -- key/value record with timestamps, headers, serialization
2. **RecordBatch** -- batched records for efficient I/O
3. **LogSegment** -- append-only segment with compaction
4. **Partition** -- ordered log with segment rollover, retention, truncation
5. **Topic** -- named collection of partitions
6. **Partitioner** -- round-robin and key-based partition routing
7. **Producer** -- publishes records with ack levels
8. **Consumer** -- polls records with offset tracking, seek, auto/manual commit
9. **ConsumerGroup** -- coordinated consumers with round-robin/range rebalancing
10. **OffsetManager** -- tracks committed offsets per group/topic/partition
11. **PartitionReplicator** -- Raft-based leader election and ISR management
12. **Broker** -- central coordinator for topics, groups, replication, lag
13. **DeadLetterQueue** -- captures failed records with error metadata
14. **MessageQueue** -- high-level pub/sub API

### Key Design Decisions
- Kafka model: append-only partitioned log, consumer groups, offset-based consumption
- Two retention policies: DELETE (time-based) and COMPACT (key-based dedup)
- Segment rollover when max_size exceeded
- Round-robin and range rebalance strategies
- High watermark tracks committed (replicated) offset
- Dead letter queue captures processing failures with full provenance
