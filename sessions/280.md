# Session 280 -- Explicit Agree/Disagree Deliberation Protocol

**Date**: 2026-03-18
**Focus**: Agent Zero cognitive architecture -- deliberation quality

## What I Did

Implemented the Explicit Agree/Disagree Deliberation Protocol based on research paper
from the Research Agent (grounded in DCI, ConfMAD, and 4 other 2025-2026 papers).

### Changes Made

**agent_schemas.py**: Added `optional` schema section for fello, othello, shadow with
`agree_with` (list), `disagree_with` (list), `revision_reason` (str). Updated
`validate_agent_output` to type-check optional fields when present without requiring them.

**cognitive_agents.py -- _fello_reaction**: Replaced binary if/else (confidence > +0.15)
with multi-factor evaluation. Checks substantive risk keywords (irreversible, cascade,
overcommit, relapse, pattern) and high-weight opponents. Structured agree/disagree.

**cognitive_agents.py -- _othello_reaction**: Replaced binary fello-conceded check with
concrete-mitigation detection (exit criteria, bounded, reversible, probe). Checks shadow
historical data. Structured agree/disagree.

**cognitive_agents.py -- _shadow_reaction**: Shadow now actively participates -- evaluates
positions against commitment_prediction. Agrees with scoped experiments, disagrees when
commitment exceeds historical follow-through capacity. Data-grounded revision_reason.

**cognitive_agents.py -- _compute_pineal_confidence**: Uses agreement/disagreement ratio.
Agreements dominate = +0.12. Disagreements dominate = -0.10. Old converged check as fallback.

**cognitive_runtime.py**: New `_extract_key_claims` helper. `_build_reaction_context` now
passes agent_weights, key_claims, historical_accuracy. Transcript enriched with
agreements, disagreements, revision_reasons.

**pineal.md**: Updated step 4 for structured deliberation signals + minority reports.

**ThoughtBubbles.tsx**: Added labels for agree_with, disagree_with, revision_reason.

### Test Results

- 28 new tests (test_deliberation_protocol.py), all pass
- 84 existing Agent Zero tests pass
- 144-session zero-bug streak

### Key Design Decisions

1. Optional fields (backward compatible -- round-1 outputs unchanged)
2. Argument merit over confidence anchoring (per arXiv 2511.07784)
3. Shadow as active participant (addresses lazy agent problem)
