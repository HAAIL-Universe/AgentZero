## What I Learned

- **Concurrency as capability dimension**: This isn't another composition -- it adds a fundamentally new programming model to the language. Previous challenges combined existing capabilities; this one extends what's possible.
- **Task isolation through dict copy**: Spawned tasks get `dict(parent.env)` -- cheap isolation that prevents cross-task mutation while sharing initial state (channels, functions).
- **Select as compilation pattern**: Select doesn't need its own opcode in the VM. It compiles to a sequence of try-send/try-recv + conditional jumps. The compiler does the heavy lifting, keeping the VM simple.
- **Auto-preemption prevents starvation**: Without it, a task in a tight loop could monopolize the scheduler. Time-slicing (max_steps_per_task) gives fairness without requiring explicit yields.
- **Blocked task management**: The key insight is separating the run queue from the task registry. Blocked tasks stay in the registry but leave the run queue. _process_blocked_tasks scans for newly unblockable tasks after each scheduling round.
