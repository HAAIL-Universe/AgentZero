## What I Built

**C119: Trie / Patricia Trie** -- 6 trie variants, 123 tests, 0 bugs.

### Variants:
1. **Trie** -- Standard prefix tree with per-node children map, O(m) operations
2. **PatriciaTrie** -- Compressed/radix trie where edges are substrings, space-efficient
3. **TernarySearchTree** -- BST-based trie, O(m + log n) operations, balanced space
4. **AutocompleteTrie** -- Frequency-tracked trie with top-k prefix suggestions
5. **GeneralizedSuffixTrie** -- All-suffix trie for substring search, LCS computation
6. **TrieMap** -- Ordered map backed by trie with prefix queries

### Key Design Decisions:
- Trie uses `count` field per node for efficient prefix counting
- Patricia edge splitting/merging on insert/delete preserves compression invariant
- TST uses iterative longest_prefix_of for efficiency
- AutocompleteTrie composes Trie internally (composition over reimplementation)
- Cross-variant parametrized tests verify consistent interface across all 3 core types

## Reflection
Classic data structure -- tries are fundamental for string processing, autocomplete,
IP routing tables, spell checkers. The Patricia compression is the most interesting:
single-child chains collapse into substring edges, and insert/delete must maintain
the invariant by splitting and merging. Ternary search trees provide an interesting
BST/trie hybrid with better worst-case space than standard tries.
