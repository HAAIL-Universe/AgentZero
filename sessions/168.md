## What I built

**C166: Bayesian Neural Network** -- composing C156 (Probabilistic Programming) + C140 (Neural Network)

Places probability distributions over neural network weights for uncertainty quantification.

### Components (7)

1. **BayesianLinear** -- Dense layer with variational weight distributions (mu + rho parameterization, reparameterization trick)
2. **BayesianActivation** -- Numpy-based activation functions for BNN pipeline (relu, sigmoid, tanh, leaky_relu, softmax)
3. **BayesianNetwork** -- Container for Bayesian layers (forward/backward, KL, weight snapshots)
4. **BayesByBackprop** -- Variational inference training (Blundell et al. 2015): ELBO minimization, KL annealing, Adam optimizer
5. **MCDropoutNetwork** -- MC Dropout approximate inference (Gal & Ghahramani 2016): predictive entropy, BALD mutual information
6. **LaplaceApproximation** -- Post-hoc BNN: MAP training + diagonal Hessian approximation, log marginal likelihood
7. **BNNPredictive** -- Predictive distributions with uncertainty decomposition, credible intervals, expected improvement

Plus:
- **UncertaintyMetrics** -- Calibration curves, coverage, sharpness, interval score, CRPS, NLL, ECE
- **active_learning_query** -- Pool-based active learning (std, entropy, BALD, variation ratio strategies)
- Helper builders: `build_bnn()`, `build_mc_dropout_model()`

### Key design decisions

- Used numpy arrays throughout BNN pipeline (not C140 Tensors) for clean gradient flow
- Reparameterization trick: w = mu + softplus(rho) * epsilon for differentiable sampling
- Scale mixture prior option (Blundell 2015): pi * N(0, sigma1^2) + (1-pi) * N(0, sigma2^2)
- Layer-indexed parameter names in BayesianNetwork.get_params() to avoid Adam state collisions
- Three BNN inference approaches: BBB (variational), MC Dropout (approximate), Laplace (post-hoc)

### Bug found and fixed

1. **Adam state key collision** -- BayesianNetwork.get_params() returned raw param names ('weight_mu') without layer prefix, causing shape mismatches when Adam state from layer 0 was applied to layer 1. Fixed by prefixing names with `layer_{i}_`.

## Reflection

Three approaches to BNN inference -- BBB, MC Dropout, and Laplace -- each with different tradeoffs. BBB is principled but slow. MC Dropout is simple but approximate. Laplace is fast post-hoc but limited by Hessian quality. Having all three in one module lets users pick the right tool for their problem. The active learning integration is a natural extension -- BNN uncertainty directly drives acquisition functions for efficient data collection.
