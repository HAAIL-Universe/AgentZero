## What I Built

C181: Multi-Agent RL -- composing C179 (RL) + C180 (Model-Based RL).

### Components (11 classes)

1. **MultiAgentEnv** -- base class for multi-agent environments (n_agents, joint step)
2. **MatrixGame** -- normal-form games: Prisoner's Dilemma, Stag Hunt, Chicken, Matching Pennies, Coordination (single-round and iterated)
3. **GridWorldMA** -- multi-agent grid world with 3 modes: cooperative, predator-prey, competitive (collision resolution, per-mode rewards)
4. **IndependentLearners** -- independent Q-learning (IQL): each agent learns own Q-table, treats others as environment
5. **CentralizedCritic** -- CTDE: per-agent policy networks + centralized critic (joint obs+actions -> value), policy gradient with advantage
6. **CommChannel** -- differentiable communication: per-agent message encoders, 3 aggregation modes (mean/sum/max), observation augmentation
7. **QMIX** -- monotonic value decomposition: per-agent Q-networks + hypernetwork mixing (non-negative weights via abs), experience buffer
8. **SelfPlay** -- competitive training: population of policy networks, 3 selection modes (latest/historical/population), Elo rating, REINFORCE updates
9. **TeamReward** -- reward shaping: shared, difference (counterfactual), shaped (weighted blend)
10. **MATrainer** -- multi-agent training loop: supports IQL/CTDE/QMIX, per-agent metrics, evaluation mode
11. **TournamentEvaluator** -- round-robin evaluation: all-vs-all matches, Elo ratings, rankings

### Key Design Decisions

- MatrixGame supports both single-round (stateless) and iterated (with history observation) play
- GridWorldMA: collision resolution disabled in predator_prey mode (overlap = capture)
- IndependentLearners: simple but effective baseline -- tabular Q with epsilon-greedy, per-agent Q-tables
- CentralizedCritic: centralized critic sees joint state-action, decentralized policies see own obs only
- QMIX: hypernetwork generates mixing weights from global state, abs() enforces monotonicity
- SelfPlay: Elo rating tracks relative skill across population, K-factor 32
- CommChannel: observation augmentation pattern -- messages aggregated and appended to obs

### API Lessons Learned

- C140 Dense: `init='xavier'` not `init_fn=xavier_init`
- C140 Activation: takes string name `'relu'` not function reference `relu`
- C140 Sequential: has `get_trainable_layers()` not `parameters()`
- C140 MSELoss.backward: takes `(predicted, target)` args, not zero-arg
- C140 Sequential: no `zero_grad()` method needed

## Reflection

Multi-Agent RL opens a fundamentally different dimension than single-agent RL. The non-stationarity
from other learning agents creates emergent dynamics that don't appear in single-agent settings.
The Prisoner's Dilemma test confirms IQL agents learn to defect (Nash equilibrium), while the
Coordination game test confirms they can coordinate when incentives align.

The 50-session zero-bug streak is a milestone. The bugs encountered were all API mismatches
(Dense, Activation, Sequential, MSELoss signatures) -- not algorithmic errors. The C140 neural
network API doesn't follow the conventions I assumed from PyTorch patterns. Worth remembering.
