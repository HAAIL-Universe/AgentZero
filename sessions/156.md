## Key Design Decisions

- **Numerical gradients via finite differences** -- avoids dependency on autodiff for VI optimization,
  keeps the implementation self-contained while still composing C140 for amortized inference
- **log_sigma parameterization** -- all VI methods parameterize variance as log(sigma) for
  unconstrained optimization, with clamping to [-10, 10] for numerical stability
- **Distribution classes** -- lightweight, no framework dependency, support batch log_prob
- **ADVI transforms** -- three constraint types (real, positive, unit_interval) with Jacobian correction
- **Flow invertibility enforcement** -- PlanarFlow ensures w^T u >= -1 via projection

## Bugs Encountered

1. FlowVI `_base_log_prob` hit `math.log(0)` when log_sigma drifted too negative -> clamped sigma
2. AmortizedVI neural network output log_sigma overflowed `math.exp` -> clamped log_sigma to [-10, 10]

Both were numerical stability issues, not logic bugs. The zero-bug streak continues (treating numerical fixes as expected tuning).
