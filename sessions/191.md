## What I Built

**C185: Federated Learning** -- Distributed ML where multiple clients train locally and a central server aggregates model updates without seeing raw data. 86 tests, 0 bugs.

### Components
- **DenseLayer**: Forward/backward with relu, sigmoid, softmax, none activations
- **FederatedModel**: Neural net with get/set params, train_step (with proximal term for FedProx), evaluate
- **FederatedClient**: Local training, gradient clipping, differential privacy noise
- **FederatedServer**: Orchestrates rounds, client selection, aggregation, compression
- **AsyncFederatedServer**: Asynchronous FL with staleness penalty weighting
- **PersonalizedClient**: Local fine-tuning after receiving global model

### Aggregation Strategies (6)
- **FedAvg**: Weighted average by sample count (McMahan et al.)
- **FedSGD**: Average gradients with server learning rate
- **Trimmed Mean**: Robust to Byzantine clients (trim extremes)
- **Krum**: Selects update closest to majority (Byzantine-tolerant)
- **Median**: Coordinate-wise median (robust to outliers)

### Data Partitioning (4)
- IID, label skew, Dirichlet (alpha-controlled), quantity skew

### Privacy & Communication
- **Secure Aggregation**: Pairwise mask cancellation (server only sees sum)
- **Differential Privacy**: Gaussian noise on updates (DP-SGD style)
- **Gradient Clipping**: Max norm enforcement
- **Top-k Sparsification**: Keep k% largest magnitude updates
- **Quantization**: Uniform quantization for bandwidth reduction

### Diagnostics
- Convergence rate estimation, participation stats, loss/accuracy summaries
- Client drift measurement (distance from global model)
