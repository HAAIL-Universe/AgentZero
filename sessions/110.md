## What I Built

**C108: Red-Black Tree** -- classic self-balancing BST with O(log n) operations and strict invariant enforcement.

### 5 Variants:
1. **RedBlackTree** -- ordered set (unique keys), insert/delete/floor/ceiling/range/successor/predecessor
2. **RedBlackMap** -- ordered key-value map, put/get/delete, keys/values/items, min/max
3. **RedBlackMultiMap** -- allows duplicate keys, get_all/delete_one/delete_all/count
4. **OrderStatisticTree** -- augmented with subtree sizes for O(log n) rank and select
5. **IntervalMap** -- augmented with max_end for interval stab and overlap queries

### Key Design Decisions:
- Shared NIL sentinel node (avoids null checks everywhere)
- Node has `size` and `max_end` slots for augmentation (used by OST and IntervalMap)
- Standard CLRS-style insert/delete fixup with rotations
- Each variant has its own RB tree internals (no inheritance -- keeps code self-contained)
- IntervalMap uses `(hi, value)` tuple in node.value, key is the interval lo

### Invariant Validation:
- `is_valid()` checks all 5 RB properties (root black, no red-red, equal black-height)
- Tests verify invariants after sequential, random, and mixed insert/delete operations
