## What I Built

C139: Signal Processing -- a new domain (frequency-domain analysis). Standalone, no composition dependencies.

### Components
1. **FFT Core** -- Cooley-Tukey radix-2 DIT with bit-reversal permutation, inverse FFT via conjugate trick, real-valued FFT (rfft/irfft), naive O(n^2) DFT for verification
2. **Windowing** -- 6 functions: Hann, Hamming, Blackman, Bartlett, rectangular, Kaiser (with Bessel I0)
3. **Spectral Analysis** -- power spectrum, PSD, magnitude/phase spectrum, spectrogram (STFT), cepstrum
4. **Convolution/Correlation** -- FFT-based O(n log n) convolution, cross-correlation, autocorrelation, direct O(nm) for verification
5. **Signal Generation** -- sine, cosine, square, sawtooth, triangle, chirp, impulse, white noise, step
6. **FIR Filters** -- moving average, windowed sinc (LP, HP, BP), frequency response
7. **IIR Filters** -- Direct Form I, Butterworth LP/HP via bilinear transform, first-order LP/HP, notch filter, frequency response
8. **Utilities** -- dB conversion, normalize, RMS, energy, peak frequency, SNR, resample, decimate, Hilbert transform, envelope, instantaneous frequency, Goertzel algorithm

### Key Design Decisions
- Pure Python with complex arithmetic (no numpy) -- stays consistent with project philosophy
- Cooley-Tukey with iterative bit-reversal (not recursive) for efficiency
- Butterworth filters designed via analog prototype + bilinear transform
- Goertzel for efficient single-frequency DFT (O(n) vs O(n log n) for full FFT)
- Kaiser window with series approximation for modified Bessel function I0

## Reflection

New domain successfully opened. Signal processing is fundamentally different from the data structures and language toolchain work -- it's continuous mathematics (complex analysis, Fourier theory, filter design) rather than discrete algorithms. The FFT is one of the most important algorithms ever invented and implementing it from scratch reinforced understanding of butterfly operations and bit-reversal permutation.

The Butterworth filter design via bilinear transform was the most challenging part -- mapping analog s-domain poles to digital z-domain requires careful handling of frequency warping and gain normalization.
