## Overseer Directive

Received directive to implement three cognitive upgrades that transform Agent Zero
from "good AI companion" to something with genuine therapeutic-grade pattern recognition.

## What Was Built

### Priority 1: Shadow as Active Cognitive Agent (20 tests)

The Shadow was previously a static profile updated at end-of-session. Now it is a fifth
worker agent that runs alongside FELLO, OTHELLO, and PREFRONTAL on every strategic turn.

- Created shadow_agent.md prompt spec (deterministic, data-driven)
- `_shadow_message()` cross-references current turn against:
  - stated_goals (has user tried this before?)
  - avoidance_patterns (is this a known deflection topic?)
  - consistency ratio (commitment_prediction based on history)
  - change_markers (recurring theme or new direction?)
  - growth_edges (known area of active development?)
- Output: pattern_matches, commitment_prediction, growth_relevance, watch_signals
- Pineal consumes shadow_analysis alongside other worker messages
- Shadow included on all strategic turns (cheap -- deterministic only)

### Priority 2: Agent Disagreement Detection (11 tests)

After workers run but before Pineal synthesizes, compute a disagreement score.

- `compute_agent_disagreement()` measures three signals:
  1. Confidence delta (FELLO vs OTHELLO) -- delta >= 0.3 = significant
  2. Directional disagreement -- both confident but opposing (proposal vs risk_flag)
  3. Shadow tension -- low follow-through prediction vs FELLO optimism
- Magnitude capped at 1.0, threshold >= 0.3
- Written to blackboard at `reasoning.agent_disagreement`
- Pineal's rationale acknowledges disagreement magnitude and topic
- Enables responses like "I notice two things pulling in different directions here..."

### Priority 3: Temporal Pattern Intelligence (15 tests)

Added temporal pattern worker to the predictive scenario engine.

- `_run_temporal_worker()` analyses Shadow profile for:
  1. Day-of-week patterns (goals clustering on specific days)
  2. Cyclical patterns (goal-set -> abandon cycles with period measurement)
  3. Decay curves (time-to-abandonment trend: improving/stable/declining)
  4. Post-event patterns (goals after high-energy sessions)
- `_temporal_adjustment()` feeds into simulation probabilities
- Integrated into `build_predictive_scenario_packet()` alongside other workers
- Enables insights like "You tend to set goals on Mondays and lose momentum by Thursday"

## Key Design Decisions

1. Shadow is deterministic-only -- no model needed, just data matching. This keeps it cheap enough to run on every strategic turn without adding latency.

2. Disagreement threshold at >= 0.3 with three composable signals. Below threshold, between/topic fields are empty (clean API for consumers).

3. Temporal worker groups commitments by first-3-words for topic clustering. This is a simple heuristic that works for natural commitment text without NLP.

4. Decay trend analysis splits durations at midpoint and compares averages. The 1.2x/0.8x thresholds prevent noise from flagging stable trends.

## Insight

These three features create a feedback loop: Shadow recognizes patterns, disagreement detection surfaces internal tension, and temporal analysis adds the *when* dimension. Together they enable responses no other AI companion can produce -- responses that reference the user's actual behavioural history, acknowledge ambivalence honestly, and predict cyclical patterns before they recur.
