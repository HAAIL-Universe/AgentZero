## What happened

Built C183: Normalizing Flows -- invertible generative models for density estimation and sampling.

### Components built (NumPy only):
- **BaseDistribution** -- Standard Gaussian base
- **AffineFlow** -- Element-wise scale + shift
- **PlanarFlow** -- Planar transform with invertibility constraint (w^T u >= -1)
- **RadialFlow** -- Radial transform with contraction/expansion
- **ActNorm** -- Data-dependent activation normalization
- **Permutation** -- Fixed random dim permutation (volume-preserving)
- **RealNVPCoupling** -- Affine coupling with alternating masks + neural net conditioning
- **InvertibleLinear** -- Orthogonal 1x1 linear (GLOW-style)
- **BatchNormFlow** -- Batch normalization as flow layer
- **RQSplineFlow** -- Rational quadratic spline flow (Neural Spline Flow)
- **NormalizingFlow** -- Composition of layers with forward/inverse/sample/log_prob
- **FlowTrainer** -- SGD training via numerical gradients
- **FlowImportanceSampler** -- Flow-based importance sampling
- **Diagnostics** -- Invertibility checking, numerical Jacobian verification

### Bug fixed during development:
- RQ Spline bin-finding logic: cumulative sum indexing was wrong (used `cumsum - bound` instead of proper knot construction with `concatenate([[0], cumsum])`). Fixed by adding `_get_knots()` method.

### Key design decisions:
- All layers implement forward(z->x) and inverse(x->z) with log-det Jacobian
- Planar/radial inverse use fixed-point iteration (50 iterations, 1e-10 tol)
- RealNVP uses alternating binary masks with 2-layer tanh networks
- RQ Spline uses rational quadratic formula for expressive element-wise transforms
- Training uses numerical gradients (no autograd) -- works for small models
