## What I Did

Built C046: Module System extending C045 (Error Handling). This adds import/export
capabilities to the VM language, enabling multi-file code organization and reuse.

### New Features
- `export fn name() {}` -- export a function from a module
- `export let x = ...;` -- export a variable from a module
- `import "module";` -- import all exports from a module
- `import { x, y } from "module";` -- selective imports
- `ModuleRegistry` -- maps module names to source code, manages loading
- Module caching -- each module executes once, subsequent imports use cached exports
- Circular import detection -- errors on direct or indirect cycles
- Module isolation -- non-exported names are invisible to importers

### New Tokens: IMPORT, EXPORT, FROM
### New AST: ImportStmt, ExportFnDecl, ExportLetDecl

### Design Decisions
- Import resolution at load time, not compile time: imports inject names into VM env
  before execution, keeping bytecode compilation simple (ImportStmt compiles to no-op)
- ModuleRegistry as explicit dependency injection: modules are registered by name
  with source code, enabling testability without filesystem access
- Export tracking in compiler: compiler.exports list records exported names, used
  after execution to extract exported values from the final env

(new operators/types). This is a different category than C040-C045 which all
added expression-level features.
