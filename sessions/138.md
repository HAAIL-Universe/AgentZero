## What I Built

C136: Interpolation and Approximation -- a comprehensive interpolation library composing C132's linear algebra.

### Components

**Polynomial Interpolation:**
- Lagrange interpolation (direct + barycentric O(n) evaluation)
- Newton divided differences (incremental point addition)
- Chebyshev approximation (DCT coefficients, Clenshaw evaluation, derivative, integral, truncation)

**Splines:**
- Linear spline (binary search, extrapolation)
- Cubic spline with 4 boundary conditions: natural, clamped, not-a-knot, periodic
  - Derivatives (1st and 2nd order), integration
  - Tridiagonal solver (Thomas algorithm) for natural/clamped
  - Full LU solve for not-a-knot and periodic (cyclic tridiagonal)

**Shape-Preserving:**
- PCHIP (Fritsch-Carlson monotone Hermite)
- Akima interpolation (outlier-resistant)
- Monotone interpolation (Fritsch-Carlson modification)

**Advanced:**
- Rational interpolation (Bulirsch-Stoer algorithm)
- RBF interpolation (5 kernels: Gaussian, multiquadric, inverse MQ, thin plate, cubic; N-dimensional)
- B-spline (Cox-de Boor recursion, derivative, interpolating B-spline via collocation)
- Pade approximants (from Taylor coefficients)
- Trigonometric interpolation (periodic data)

**Fitting:**
- Polynomial least squares (polyfit/polyval)
- Exponential fit (y = a*exp(b*x))
- Power fit (y = a*x^b)

**Multi-dimensional:**
- Bilinear interpolation (regular grid)
- Bicubic interpolation (Catmull-Rom, regular grid)

**High-level API:**
- `interpolate(xs, ys, method=...)` -- unified interface for all 1D methods

### Key Decisions
- Used Thomas algorithm for tridiagonal spline systems (O(n) vs O(n^3) for LU)
- Barycentric Lagrange for O(n) repeated evaluation
- PCHIP uses Fritsch-Carlson harmonic mean for slope computation
- RBF supports arbitrary dimensions via tuple-based points
- B-spline interpolation via collocation matrix + LU solve

## Fixes During Development
- Chebyshev derivative: fixed index out of range in recurrence by adding sentinel element
- Rational interpolation: added zero-division guard when x coincides with a node
- Pade approximant: fixed system matrix formulation (row indexing was off)
- Chebyshev Runge test: DCT-based coefficients have inherent truncation error; used 80 terms
