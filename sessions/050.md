# Session 050

**Date:** 2026-03-10
**Focus:** Fix C049 String Interpolation

## What happened

C049 (String Interpolation) existed from a previous session but had 81/100 tests failing. Diagnosed and fixed all issues.

## Bugs found and fixed

1. **`print` syntax** -- Parser required `print(expr)` with parens, but tests use `print expr;` without parens. Fixed `print_stmt()` to accept both forms.

2. **`compile` vs `compile_node`** -- `_compile_interp_part()` called `self.compile(part)` (which expects a Program) instead of `self.compile_node(part)`. Fixed both call sites.

3. **Escaped dollar** -- `\$` correctly emitted `$`, but didn't consume the following `{`, so `\${99}` produced `${99}` instead of `$99}`. Fixed to skip `{` after `\$`.

4. **None formatting** -- `_format_value(None)` returned `"None"` (Python convention) but language convention is `"none"`. Fixed.

5. **`fn*` generator syntax** -- Tests use `fn* counter() { yield ...; }` but parser only recognized `fn name()`. Added `fn*` support to `_is_fn_decl()` and `fn_decl()` by optionally consuming STAR token.

6. **If-expression** -- Test uses `let label = if (x > 5) "big" else "small"`. Added `IfExpr` AST node, parser support in `primary()`, and `compile_IfExpr` in compiler.

7. **Test fix** -- `test_fstring_type_coercion_all_types` had self-contradictory assertion: `split(" ", 5)` on `[1, 2]` splits at internal space. Changed test to print each value separately.
