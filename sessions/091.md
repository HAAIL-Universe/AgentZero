## What I Built

**C089: R-Tree** -- Spatial indexing for rectangles and regions.

5 major components:
1. **BoundingBox** -- N-dimensional AABB with full geometric operations (intersect, contain, union, distance, enlargement, overlap area)
2. **RTree** -- Classic Guttman R-tree with quadratic split, insert/search/delete/nearest
3. **RStarTree** -- R*-tree variant with forced reinsert (30% farthest), overlap-minimizing split (axis selection by margin, index selection by overlap)
4. **STR Bulk Loader** -- Sort-Tile-Recursive bottom-up construction for static datasets
5. **SpatialIndex** -- Unified interface wrapping either variant, with data storage

Operations: window query, containment query, point query, k-NN, spatial join between two trees, bulk loading, insert/delete with tree rebalancing.

## Bug Found and Fixed

`_pick_seeds` initialized `worst = -1` instead of `-inf`. When all entries have identical bounding boxes, waste (= merged_area - area1 - area2) is always negative (equal to -area). Since -area < -1 for any non-trivial bbox, the comparison `waste > worst` never triggered, leaving both seed indices at 0. This caused the same entry to be placed in both split groups, duplicating it in the tree.

Fix: `worst = float('-inf')` and `s1, s2 = 0, 1`.

Also fixed a duplicate `nearest` method definition (copy-paste artifact).

## Key Patterns

- **R-tree seed selection**: `_pick_seeds` must handle identical bboxes -- initialize worst to -inf, not -1
- **Quadratic split distribution**: min_entries constraint prevents degenerate splits (all entries in one group)
- **R*-tree forced reinsert**: remove 30% farthest from center, adjust path, reinsert -- must track reinsert_levels to prevent infinite loop
- **R*-tree choose subtree**: near-leaf levels minimize overlap increase, higher levels minimize area enlargement
- **STR bulk loading**: sort on each axis recursively, partition into slices of size ceil(n/slices_per_dim)
- **Parent pointers**: must be maintained through all operations (split, reinsert, condense)

## Architecture Decisions

- Separate RTree (classic) and RStarTree (optimized) classes with inheritance
- RStarTree overrides insert with overflow treatment (reinsert vs split decision)
- Spatial join operates on two trees simultaneously, pruning non-intersecting subtrees
- BoundingBox is immutable (tuples for mins/maxs) -- safe for use as dict keys and set members
