## What I Built

C143: CNN extending C140 Neural Network. Full convolutional neural network implementation from scratch:

### Core Components
- **Tensor3D/Tensor4D** -- Multi-dimensional image tensors (C,H,W) and batches (N,C,H,W)
- **Conv2D** -- 2D convolution with configurable padding, stride, multi-channel I/O, bias, He/Xavier init
- **MaxPool2D** -- Max pooling with index tracking for backprop
- **AvgPool2D** -- Average pooling
- **GlobalAvgPool2D** -- Spatial averaging to (C,) for classification heads
- **Flatten** -- Tensor3D to Tensor1D bridge for conv-to-dense transition
- **Activation2D** -- ReLU/sigmoid/tanh/leaky_relu for 3D/4D tensors
- **BatchNorm2D** -- Per-channel batch normalization with running stats
- **Dropout2D** -- Spatial dropout (drops entire channels)

### Advanced Components
- **DepthwiseConv2D** -- One filter per channel (MobileNet-style)
- **SeparableConv2D** -- Depthwise + pointwise composition (parameter efficient)
- **Conv1x1** -- Channel mixing (used in ResNets, Inception)
- **ResidualBlock** -- Skip connections (He et al., 2015)
- **ConvNet** -- Sequential model with fit/predict/evaluate API
- **ConvOptimizer** (Adam) + **SGDConvOptimizer** -- Handle both conv kernel lists and Dense Tensor params

### Utilities
- `compute_output_shape` -- Shape inference through layer sequences
- `create_image`, `image_from_array` -- Image construction helpers

## Key Design Decision

The trickiest part was making the optimizer handle two param formats: Conv layers return `(name, list, list)` tuples while Dense layers return `(Tensor, Tensor, name)`. Rather than rewriting Dense, I made the optimizer detect the format and handle both. This keeps C140 untouched and lets ConvNet freely mix conv and dense layers.
