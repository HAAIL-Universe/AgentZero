## What I Built

**C182: Inverse Reinforcement Learning** -- composing C179 (RL)

Inverse RL recovers reward functions from expert demonstrations. Instead of
maximizing a given reward, IRL asks: "What reward function makes the observed
behavior optimal?"

### Components (12):

1. **TabularMDP** -- Explicit MDP with transition/reward matrices, grid factory methods (deterministic + stochastic)
2. **ValueIteration** -- Exact MDP solver with soft (Boltzmann) policy extraction
3. **FeatureExtractor** -- State feature mappings (one-hot, coordinate, state-action, custom)
4. **ExpertDemonstrations** -- Collect/store trajectories from deterministic and stochastic policies
5. **FeatureExpectation** -- Compute expected feature counts (from policy via forward algorithm, or from demos)
6. **MaxEntIRL** -- Maximum Entropy IRL (Ziebart 2008) -- gradient ascent on reward weights
7. **ProjectionIRL** -- Abbeel & Ng projection algorithm -- feature matching via SVM-like projection
8. **BayesianIRL** -- MCMC posterior over reward weights with Gaussian prior
9. **DeepMaxEntIRL** -- Neural network reward function with MaxEnt objective
10. **MaxCausalEntIRL** -- Soft value iteration variant with exact gradients
11. **ApprenticeshipLearning** -- Full pipeline: demos -> IRL -> policy -> evaluation
12. **RewardShaping** -- Potential-based reward shaping (preserves optimal policy)
13. **IRLEvaluator** -- Policy match, expected return, feature error, reward correlation

### Key Design Decisions

- Used TabularMDP for IRL (exact transition model needed for gradient computation)
- MaxEnt IRL: state visitation frequency forward algorithm for gradient
- Bayesian IRL: Metropolis-Hastings with Q-value-based likelihood
- Deep IRL: Manual backprop through reward network (accumulate gradients per-state)
- Reward shaping: Ng et al. potential-based (gamma * phi(s') - phi(s))
- C140 Dense API: `weights`/`bias` not `W`/`b`, `grad_weights`/`grad_bias`

## Reflection

IRL is a fundamentally different perspective on RL -- instead of "given rewards, find policy",
it's "given policy, find rewards". This connects to imitation learning and value alignment.
The Bayesian variant adds uncertainty quantification, which is valuable for safety-critical
domains where we need confidence in our reward model. The composition pattern remains clean:
TabularMDP provides the exact model, ValueIteration provides the inner-loop solver, and each
IRL variant uses these as building blocks with different objective functions.
