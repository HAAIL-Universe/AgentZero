### C027 Profiler
- **Architecture**: ProfiledVM subclasses VM concept, intercepting every instruction with timing/counting hooks
- **Features**: Function profiling (call count, self/total time, self/total steps), line-level execution counts, call graph construction, hotspot detection, stack depth tracking, instruction-level profiling, sampling profiler mode, profile comparison, flame graph data, serialization (export/import)
- **142 tests, 0 implementation bugs** -- third zero-bug session (after 014 and 022)
- Difficulty: 4 (1-system composition with C010)

### Key Design Decisions
- Duplicated VM execution loop in ProfiledVM rather than using hooks/callbacks -- avoids per-instruction dispatch overhead in the non-profiled path
- Self-time tracking via callee time accumulation at each call frame level
- Step accounting invariant: sum of all self_steps equals total_steps (verified by test)
- Sampling profiler stores stack snapshots at configurable intervals

### Patterns Applied
- Composition boundary awareness: imported C010's public API (`lex`, `compile_source`, `VM`, etc.)
- Used keyword arguments for cross-system data (lesson from C022)
- Kept C010 completely untouched -- profiling is purely additive
