## What I Built

**C199: Search Engine** -- composing C197 (Information Retrieval) + C016 (HTTP Server)

A complete web search engine with 12 components:

1. **HTMLParser** -- extracts title, meta, text, links, headings from HTML
2. **RobotsParser** -- robots.txt compliance (user-agent rules, crawl-delay, sitemaps)
3. **SiteMapParser** -- sitemap XML parsing (URLs, lastmod, priority, changefreq)
4. **URLFrontier** -- priority-based crawl scheduling with deduplication and politeness
5. **PageRank** -- iterative PageRank algorithm with dangling node handling and convergence
6. **WebCrawler** -- simulated-network crawler (robots compliance, link discovery, PageRank)
7. **SearchResult** -- rich result objects with URL, title, snippet, combined score
8. **AutoComplete** -- trie-based query suggestion with frequency ranking
9. **SearchIndex** -- unified index combining BM25 zone-weighted relevance + PageRank scores
10. **QueryProcessor** -- query parsing (phrase detection, spell correction, enhancement)
11. **CrawlScheduler** -- crawl job management (create, run, status tracking)
12. **SearchAPI** -- REST API over HTTP (search, suggest, index, crawl, stats, CORS)

## API Endpoints

- `GET /api/search?q=query&limit=10` -- search with combined text+PageRank ranking
- `GET /api/suggest?q=prefix` -- autocomplete suggestions
- `POST /api/index` -- index a single page (url + html)
- `POST /api/crawl` -- start crawl job from seed URLs
- `GET /api/stats` -- index statistics
- `GET /api/jobs` -- list crawl jobs
- `GET /api/health` -- health check

## Key Design Decisions

- **Zone-weighted BM25**: title=3x, headings=2x, meta=1.5x, body=1x
- **Combined ranking**: 70% text relevance + 30% PageRank (configurable)
- **Simulated network**: for testability without real HTTP
- **Defensive JSON parsing**: handles None from empty body edge case in HTTP stack
