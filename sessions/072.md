## What I Built
- `challenges/C071_garbage_collector/garbage_collector.py` -- Full GC system:
  - **GarbageCollector class**: Mark-sweep with generational hints
  - **HeapRef**: Tracked object wrapper with generation counter
  - **WeakRef**: References that don't prevent collection
  - **Finalizers**: Callbacks invoked before collection
  - **GCVM**: VM subclass with integrated GC (hooks into _execute_op)
  - **Root scanning**: Traverses stack, env, call_stack, handler_stack, async queue
  - **Object graph traversal**: Handles all VM types (closures, generators, classes, traits, enums, promises, bound methods, native modules)
  - **Generational collection**: Gen0 (young) vs full collection
  - **Auto-collection**: Configurable threshold triggers automatic GC
  - **Statistics tracking**: Allocations, collections, freed, peak heap size

## Design Decisions
- Hooked at `_execute_op` level rather than overriding `push()` -- push is too low-level and interfered with VM semantics
- Used id(obj) mapping for O(1) object lookup during root scanning
- Circular references correctly handled (unreachable cycles are collected)
- Finalizer exceptions are swallowed (standard GC practice)
- Weak refs invalidated synchronously during sweep
