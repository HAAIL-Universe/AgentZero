## What I Did

### Security Fixes (all 4 from NEXT.md)

1. **CORS wildcard + credentials** (agent_zero_server.py:104-119)
   - Replaced `allow_origins=["*"]` with explicit origin list
   - Configurable via `AGENT_ZERO_CORS_ORIGINS` env var
   - Default: localhost:8888, localhost:3000 (both IPv4)

2. **Singleton inference race** (agent_zero_inference.py:323-340)
   - Added `get_inference_async()` with `asyncio.Lock` double-check pattern
   - Updated all async handlers (WebSocket + HTTP) to use async version
   - Sync `get_inference()` preserved for startup-time calls

3. **Unbounded message accumulation** (agent_zero_server.py:944-947)
   - Added `_MAX_SESSION_MESSAGES = 200` cap
   - Trims oldest messages when exceeded (slice assignment)
   - Applied at the single entry point after user message append

4. **Prompt injection via interpolation** (cognitive_agents.py:15-30)
   - Added `_sanitize_user_value()` -- truncates to 300 chars, strips injection markers
   - Applied to all user-derived values interpolated into agent messages:
     desired_state, blocker, best_path, next_step, fallback
   - Also applied in cognitive_hub.py clarifying question builder
   - 4 new tests covering injection, truncation, clean passthrough, and end-to-end

### Test Results

## Design Decisions

- CORS: env-var configurable rather than hardcoded, because deployment origins will differ
- Sanitizer uses regex-based marker detection rather than heavy NLP -- fast, deterministic, covers the known attack patterns
- Message cap uses 200 rather than a token estimate -- simpler, and context compression already handles the token budget
