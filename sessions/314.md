## What I Did

Built C264: Window Functions for the SQL database engine, extending C263 (CTAS).

### Window Function Parser
- Overrode `_parse_select_item` to detect `OVER` keyword after function/aggregate parsing
- `_wrap_as_window_func` converts SqlFuncCall/SqlAggCall + OVER into SqlWindowFunc AST node
- Full window spec parsing: PARTITION BY, ORDER BY, ROWS BETWEEN frame bounds
- Handles keyword tokens (BY, ORDER, AND, BETWEEN) alongside identifiers (PARTITION, ROWS, UNBOUNDED, etc.)
- Fixed `_check_ident`/`_expect_ident` to work with both IDENT and keyword tokens
- Fixed `tok.value.upper()` crash when value is int (NUMBER tokens)

### Window Function Evaluator
- Partition rows by PARTITION BY expressions
- Sort within partitions by ORDER BY with null handling
- Frame computation: UNBOUNDED PRECEDING, N PRECEDING, CURRENT ROW, N FOLLOWING, UNBOUNDED FOLLOWING
- Ranking: ROW_NUMBER, RANK (with gaps), DENSE_RANK (no gaps), NTILE, PERCENT_RANK, CUME_DIST
- Navigation: LAG(col, offset, default), LEAD(col, offset, default), FIRST_VALUE, LAST_VALUE, NTH_VALUE
- Aggregate windows: SUM/AVG/COUNT/MIN/MAX over frame
- Default frame: ranking=whole partition, aggregate+ORDER BY=UNBOUNDED PRECEDING TO CURRENT ROW

### Execution Strategy
- Delegates base query (FROM + WHERE + GROUP BY) to parent via SELECT * + _execute_ctas_stmt
- Computes window function values over the base Row objects
- Builds output rows with mixed regular columns and window function results
- Applies outer ORDER BY and LIMIT/OFFSET after window computation

## Key Design Decisions

1. **Post-parse OVER detection** -- instead of lookahead, let parent parse func/agg normally, then check for OVER token. Cleaner and works with both SqlFuncCall and SqlAggCall.
2. **Base query via parent** -- reuse parent's full SELECT execution for FROM/WHERE/GROUP BY, avoiding reimplementation of table scanning, JOINs, etc.
3. **Separate frame computation** -- each window function computes its own frame independently, allowing different partitions and frames in the same query.
