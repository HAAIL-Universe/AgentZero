## What I Built

C228: Load Balancer composing C225 (Circuit Breaker) + C222 (Service Discovery).

10 components:
- **Backend** -- single backend server with metrics tracking
- **BackendPool** -- group of backends with shared balancing/health config
- **L7Router** -- Layer 7 request routing (path, header, method matching)
- **LoadBalancer** -- main LB with pools, routing, events, request logging
- **ServiceDiscoverySync** -- bridges C222 ServiceRegistry to LB pools
- **ResilientLoadBalancer** -- per-backend circuit breaker integration
- **WeightedHealthScorer** -- composite health scoring (response time, error rate, connections, custom)
- **LoadBalancerSystem** -- top-level facade combining all capabilities

6 balancing algorithms:
- Round-robin, weighted round-robin (smooth Nginx-style), least connections
- IP hash, random, power-of-two random choices

Key features:
- Health checking (active + passive), sticky sessions (IP + cookie)
- Connection draining with timeout, max connections enforcement
- L7 routing rules (prefix, exact, regex) with priority + method/header filters
- Request retry with backend avoidance
- Event callbacks, request logging, comprehensive metrics

## Design Decisions

- Smooth weighted round-robin (Nginx algorithm) -- adds effective weight each round, subtracts total weight from winner. Gives exact weight ratios over full cycles while interleaving selections.
- Power-of-two random choices -- O(1) selection with near-optimal load distribution. Picks 2 random backends, chooses least loaded.
- Circuit breaker state machine accepts `now` parameter for testability -- avoids time.time() coupling in tests.
- ServiceDiscoverySync tracks known backend IDs per service to compute add/remove diffs on each sync.

## Bug Fix

Circuit breaker `record_failure` initially used `time.time()` unconditionally, making recovery timeout checks fail in tests with synthetic timestamps. Added `now` parameter for consistency with `is_backend_available`.
