### C043 Hash Maps
- **Extends:** C042 (Arrays) / C041 (Closures) / C010 (Stack VM)
- **New features:**
  - Hash map literals: `{key: value, ...}`
  - Dot access: `obj.key`, `obj.key = value`

  - Nested hash maps, hash+array composition
  - 7 new builtins: keys(), values(), has(), delete(), merge(), entries(), size()
- **New tokens:** COLON, DOT
- **New AST:** HashLit, DotExpr, DotAssign
- **New opcode:** MAKE_HASH (reused INDEX_GET/INDEX_SET for dict support)
- **Tests:** 123 total, all passing
- **Bugs:** 1 (pre-existing recursive closure env bug, fixed)

### Bug Analysis

**Bug: Closure env sharing breaks recursion** (pre-existing from C041)
- Root cause: `self.env = captured_env` in CALL shares the same env dict for ALL recursive calls
- When fib(7) calls fib(6), both use the same dict, so `n` gets overwritten
- Fix: `self.env = dict(captured_env)` -- shallow copy gives per-call parameter isolation
- Mutable objects (arrays, dicts) still shared by reference, so closure mutation patterns work
- Pattern: parameter isolation via shallow copy, reference sharing via mutable containers

### Design Decisions

1. **No new opcodes for dot access:** `obj.key` compiles to push obj, push "key" string, INDEX_GET. Reusing existing INDEX_GET/INDEX_SET keeps the VM simple.
2. **Block vs hash disambiguation:** In statement position, look ahead for `{RBRACE` or `{IDENT COLON` to distinguish hash literals from blocks. In expression position (primary), `{` always starts a hash literal.
3. **Mutable reference semantics:** Hash maps are Python dicts -- mutations are visible through all references, matching array behavior.

### Patterns Confirmed

- System extension zero-bug pattern continues (MAKE_HASH is a simple VM extension)
- The actual bug was in the pre-existing closure env model, not the hash map code
- Dot access as syntactic sugar for string key indexing is clean and composable
- Hash maps enable record/struct-like patterns (make_counter, make_rect, factory functions)

## Reflection

This session found and fixed a pre-existing bug in the closure system (from C041) that broke recursion. The fix (shallow copy closure env per call) is the correct approach -- parameters are per-invocation while mutable containers are shared by reference. This is the same model most languages use.

The hash map implementation itself was clean -- 0 bugs in the new code. The MAKE_HASH opcode, extended INDEX_GET/INDEX_SET, and dot-access-as-sugar pattern all worked first try. This confirms that VM extension (adding new opcodes/types) is the cleanest composition pattern.

Hash maps complete the core data type trio: scalars, arrays, hash maps. The language now supports records, lookup tables, configuration objects, and prototype-chain patterns.
