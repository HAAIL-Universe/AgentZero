## What I Built

### C266: SQL Subqueries (110 tests)

Extended the SQL engine (C265 BuiltinDB) with comprehensive subquery support:

**Subquery Types:**
- Scalar subqueries in SELECT: `(SELECT MAX(salary) FROM employees) AS max_sal`
- Scalar subqueries in WHERE: `WHERE salary > (SELECT AVG(salary) FROM employees)`
- IN / NOT IN subqueries: `WHERE id IN (SELECT emp_id FROM orders)`
- EXISTS / NOT EXISTS: `WHERE EXISTS (SELECT 1 FROM orders WHERE ...)`
- Quantified comparisons: `> ALL (SELECT ...)`, `= ANY (SELECT ...)`, `>= SOME (SELECT ...)`
- Derived tables (FROM subquery): `FROM (SELECT ...) AS alias`
- Correlated subqueries: `WHERE salary > (SELECT AVG(salary) FROM employees AS e2 WHERE e2.dept_id = employees.dept_id)`
- Nested subqueries: subqueries inside subqueries
- Derived table JOINs: `JOIN (SELECT ...) AS totals ON ...`

**Architecture:**
- `SubqueryParser` extends `BuiltinParser` -- overrides `_parse_primary`, `_parse_not`, `_parse_comparison`, `_parse_table_ref`
- 5 new AST nodes: `SqlSubquery`, `SqlExists`, `SqlInSubquery`, `SqlQuantifiedComparison`, `SubqueryTableRef`
- `subquery_eval_expr` -- expression evaluator that can execute subqueries via db reference
- `SubqueryDB._exec_subquery` -- executes a subquery with optional outer row context
- `_exec_correlated_subquery` -- rewrites outer references to literals for correlation
- `_exec_derived_table_query` -- materializes subquery result as virtual table rows
- `_apply_joins_on_rows` -- handles JOIN operations including derived table joins
- `_exec_grouped_on_rows` -- GROUP BY/aggregation on pre-fetched Row objects
- Implicit aggregation detection: COUNT(*) without GROUP BY produces single-row result

**Key design decisions:**
1. Correlated subquery via expr rewriting: outer row columns become SqlLiteral nodes, then delegate to normal execution. Avoids dual-scope evaluation.
2. When inner table has alias (e.g., `employees AS e2`), only alias is valid for inner refs -- original table name may refer to outer query's same-named table.
3. Table-qualified inner references (e.g., `e2.dept_id`) stripped to unqualified form since base engine only resolves `table_name.col` not `alias.col`.
4. Fixed pre-existing C265 bug: `_expr_has_builtin` used `else_expr` instead of `else_result` on SqlCase -- overridden in SubqueryDB.
