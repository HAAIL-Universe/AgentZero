### 2. V203: Symbolic Quantitative Partial Observation (70/70 tests pass)

BDD-encoded belief-energy games. Composes V200 + V160 + V021.

**Architecture:**
- `SymbolicPOGame`: 2-player PO game with energy/cost weights, probabilistic transitions
- `SymbolicBelief`: belief state with both BDD support and explicit distribution
- `BeliefBDDEncoder`: encodes belief support sets as BDD cubes over state-indicator variables
- `build_belief_energy_game`: BFS exploration of belief space, constructs finite energy game
- Two solvers: `solve_belief_energy` (min energy) and `solve_belief_mean_payoff` (avg reward)
- BDD-based safety analysis: backward fixed-point over unsafe belief supports
- BDD-based reachability: forward fixed-point from initial beliefs
- POMDP conversion: any V200 POMDP becomes a SymbolicPOGame
- Simulation with belief tracking and energy accounting

**Three example games:**
1. Tiger POMDP: classic noisy observation, listen/open decision
2. Maze: grid navigation with limited observation
3. Surveillance: patrol vs intruder ring game with safety objective

**Bugs fixed:**
1. Empty game: early return before constructing encoder for empty state set
2. `_symbolic_preimage`: BDD variable intersection != state membership. `AND(v2, v0)` is satisfiable (both state 0 and 2 present) but doesn't mean state 0 is a target. Fixed: use `encode_support({s})` cube check for correct membership.

### Key Insight

BDD-based preimage computation over belief supports requires careful distinction between "state s is in a support" (BDD variable) and "the support {s} satisfies the target property" (BDD cube restriction). The naive `AND(target, var)` conflates these -- use `AND(target, encode_support({s}))` instead.
