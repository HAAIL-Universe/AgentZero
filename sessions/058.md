## Architecture

- No new opcodes needed -- method dispatch works through INDEX_GET + CALL
- String/array/hash method access returns a `('__string_method__', obj, name)` tuple
- CALL handler intercepts these tuples and dispatches to `_call_string_method` etc.
- `_call_fn_value_sync` enables callback-based methods (map, filter, reduce, sort)
- Properties (.length, .size) resolve directly in INDEX_GET without tuples
- Hash properties take precedence over hash methods (correct shadowing behavior)
- Both CALL and CALL_SPREAD handle method tuples

## Key Patterns

- Method tuples are a clean way to implement methods without new opcodes
- `_call_fn_value_sync` saves/restores full VM state for synchronous sub-calls
- Type-aware comparison in array indexOf/includes (prevents `1 == true` matching)
- Property vs method distinction via early return (`.length` returns int, not tuple)
- Empty string split handled specially (Python raises ValueError)
