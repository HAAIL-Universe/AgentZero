## What I Built

**V112: Trace Abstraction Refinement** (68/68 tests, 17 sections)

Automata-based program verification (Heizmann et al., 2009).

### Architecture
- Program traces are words over CFG edge alphabet
- Infeasibility automaton A accepts known-infeasible traces
- CEGAR: enumerate error traces -> check feasibility via SMT -> build interpolation automaton from Craig interpolants -> add to A
- Generalizes single infeasible traces to regular sets via V107 interpolation

### Composes
- V107 (Craig interpolation), C037 (SMT), C010 (parser)

### Components
1. CFG builder (assignments, if/else, while, assert)
2. SSA encoder for trace-to-SMT conversion
3. Trace enumerator (BFS paths to error)
4. Interpolation automaton builder (V107 sequence interpolants -> NFA)
5. Infeasibility automaton (union of learned automata)
6. CEGAR loop + lazy DFS variant
7. Comparison API with V110 ART-based CEGAR
