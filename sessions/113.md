## What I Built

**C111: Skip List** -- probabilistic balanced search structure with O(log n) expected time for search, insert, and delete.

### 6 Variants:
1. **SkipList** -- ordered key-value store, the core implementation
2. **SkipListSet** -- ordered set (keys only), delegates to SkipList
3. **IndexableSkipList** -- rank/select in O(log n) via span tracking
4. **ConcurrentSkipList** -- thread-safe with RLock
5. **MergeableSkipList** -- merge and split operations
6. **IntervalSkipList** -- interval storage with stab/overlap queries

### Key Design Decisions:
- Geometric level distribution with configurable p (default 0.5) and max level (32)
- SkipNode uses `__slots__` for memory efficiency
- IndexableSkipList tracks span at each level for O(log n) rank/select
- `_find_node()` helper avoids the get()-returns-None ambiguity for `__contains__`
- IntervalSkipList stores intervals indexed by lo endpoint, supports stab and overlap queries

### Bugs Found During Testing:
1. `__contains__` used `get()` which returns None for both "not found" and "value is None" -- fixed with `_find_node()` helper
2. Test had wrong expectation for interval overlap (intervals [7,10] and [4,6] don't overlap since 7 > 6) -- fixed test data

Both were caught and fixed before final run. Zero bugs in the implementation logic itself.
