## What I Built

C189: Survival Analysis -- a comprehensive time-to-event analysis library built from scratch using NumPy.

### Components (12 classes)
1. **SurvivalData** -- Container with censoring, covariates, groups, subsetting
2. **KaplanMeier** -- Non-parametric survival estimator (product-limit), Greenwood CI, median, RMST
3. **NelsonAalen** -- Cumulative hazard estimator, S(t) = exp(-H(t))
4. **LogRankTest** -- Multi-group survival comparison (standard, Wilcoxon, Tarone-Ware, Peto-Peto weights)
5. **CoxPH** -- Cox proportional hazards via Newton-Raphson on partial likelihood (Breslow ties), concordance, baseline hazard, prediction
6. **ExponentialModel** -- Constant hazard parametric model (MLE, AIC)
7. **WeibullModel** -- Shape/scale parametric model (Newton-Raphson for shape, closed-form scale)
8. **LogNormalModel** -- Log-normal parametric model (EM-style with censoring)
9. **LogLogisticModel** -- Log-logistic parametric model (gradient ascent on logistic parametrization)
10. **LifeTable** -- Actuarial life table analysis (actuarial adjustment)
11. **SchoenfeldTest** -- PH assumption testing (Schoenfeld residuals, rank correlation)
12. **CumulativeIncidence** -- Competing risks (Aalen-Johansen CIF estimator)

### Utility functions
- Normal PDF/SF, chi-squared SF, t-distribution SF, Spearman correlation -- all implemented from scratch (no scipy for core)

## Key Decisions
- Logistic parametrization for log-logistic MLE: mu = log(alpha), s = 1/beta, with numerically stable sigmoid computation
- Cox PH uses ascending-sort risk set accumulation for O(n) partial likelihood computation
- Competing risks uses overall KM for S(t-) estimation in Aalen-Johansen formula
- All statistical tests use custom approximations (erfc, Wilson-Hilferty) to avoid scipy dependency

## Bug Fix
- LogLogisticModel MLE initially diverged due to numerical overflow in (t/alpha)^beta. Fixed by reparametrizing to logistic location-scale (mu, s) and using stable sigmoid computation with clipped steps.
