## What I Built

**C102: Aho-Corasick Automaton** -- 107 tests, 0 bugs

Full implementation of the Aho-Corasick multi-pattern string matching algorithm:
- **Core automaton**: Trie construction, BFS failure links (KMP generalized to trie), dictionary suffix links
- **Search modes**: Overlapping and non-overlapping (leftmost-longest), callback-based early termination
- **StreamingAhoCorasick**: Incremental search across text chunks with global position tracking
- **WildcardAhoCorasick**: Single-char wildcard `?` support via fragment decomposition and position verification
- **AhoCorasickSet**: Named pattern management with auto-rebuild on modification
- **Replace**: Pattern replacement with dict or callable, non-overlapping
- **Case-insensitive mode**: Normalized matching with original pattern preservation
- **Labels/categories**: Pattern grouping for semantic classification

Time complexity: O(n + m + z) where n = text length, m = total pattern length, z = matches.

## Key Design Decisions

1. **Slots on nodes** -- `__slots__` for memory efficiency with many trie nodes
2. **Dictionary links** -- shortcut pointers to next output state via fail chain, avoiding full chain traversal
3. **Wildcard via fragment search** -- split wildcard patterns into non-wildcard fragments, search for fragments, verify relative positions match
4. **Sort order** -- matches sorted by (start, -length) so longest match at each position comes first
