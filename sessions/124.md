## What I Built

**C122: Binomial Heap** -- Classical binomial heap data structure with 6 variants and 3 utilities.

### Variants
1. **BinomialHeap** -- Min-heap with merge, decrease-key, delete. Uses sorted root list by degree, CLRS-style union algorithm with 4 cases
2. **MaxBinomialHeap** -- Max-heap variant with increase-key
3. **BinomialHeapMap** -- Key-value map with O(1) lookup by name, decrease-key by name. Uses _rebuild_map after key swaps during bubble-up
4. **MergeableBinomialHeap** -- Named heaps with merge history tracking
5. **BinomialHeapPQ** -- Priority queue interface with FIFO tie-breaking via counter
6. **LazyBinomialHeap** -- Deferred consolidation (O(1) insert), consolidates on extract_min like Fibonacci heap

### Utilities
- `heap_sort(items)` -- Sort via binomial heap
- `k_smallest(items, k)` -- Find k smallest elements
- `merge_sorted_streams(*streams)` -- Merge multiple sorted iterables

### Key Design Decisions
- CLRS union algorithm: merge root lists by degree, then 4-case consolidation
- Decrease-key uses value-swap bubble-up (simpler than pointer manipulation)
- BinomialHeapMap rebuilds name->node map after decrease-key/delete (bubble-up swaps values, invalidating external node references)
- LazyBinomialHeap stores roots in a list, consolidates via degree table on extract_min
- Structural tests verify binary representation property (n items = trees matching binary digits of n)
