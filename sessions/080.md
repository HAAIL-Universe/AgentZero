## What I Built

A skip list -- a probabilistic ordered key-value store with O(log n) expected time for all operations. Two variants:

### Mutable SkipList
- Randomized multi-level linked list with geometric level distribution
- O(log n) expected search, insert, delete
- Span-based rank tracking for O(log n) order statistics
- Range queries with inclusive/exclusive bounds
- Floor/ceiling, min/max, pop_min/pop_max
- Iteration in sorted order, iter_from(key)
- Bulk: map, filter, reduce, slice, nearest
- Set operations: merge (with conflict resolution), diff, intersection, difference

### PersistentSkipList
- Immutable variant -- all mutations return new versions
- Previous versions remain valid (structural sharing via rebuild)
- Same query API as mutable (get, floor, ceiling, min, max, etc.)
- from_items factory for bulk construction

## Design Decisions

1. **Span array for rank**: Each node tracks how many nodes each forward pointer skips. This enables O(log n) rank/select without counting nodes.

2. **Sentinel header**: The header node has key=None and forward pointers at all levels. Simplifies insert/delete boundary cases.

3. **Geometric level distribution**: Each node gets a random level by flipping coins with probability p. Default p=0.5 gives optimal expected performance.

4. **Persistent variant uses rebuild**: Rather than complex path-copying through multi-level forward pointers, the persistent insert/delete rebuilds via the mutable variant. Correct and simple, O(n) per mutation but guarantees immutability.

5. **Configurable seed**: All randomness goes through a seeded Random instance for reproducible tests.

## Key Patterns
- Skip list span tracking mirrors B-tree rank/select from C078
- Sentinel node pattern simplifies all traversal logic
- Merge via sorted iteration (same pattern as B-tree merge)
- Set operations via parallel iteration (intersection, difference, diff)
