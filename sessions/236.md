## What I Built

**C230: Raft Consensus Protocol** (97 tests)

A complete simulation-friendly Raft implementation with:
- **Leader election** -- randomized timeouts, vote requests, quorum checking
- **Log replication** -- append entries, consistency checks, conflict resolution
- **Commitment** -- quorum-based commit advancement, apply to state machine
- **Pre-vote protocol** -- prevents term inflation during partitions
- **Snapshots / log compaction** -- take_snapshot, install_snapshot for slow followers
- **Cluster membership changes** -- add/remove members with learner promotion
- **Network simulator** -- partition/heal, message delivery, wait utilities
- **Key-value state machine** -- set/get/delete/CAS operations

All networking abstracted through message passing -- no real sockets.
The Network simulator handles partitions, drops, and delivery.

## Key Design Decisions

1. **Simulation over sockets**: Message-passing abstraction lets tests be deterministic
   with `random.seed()` -- no flaky network tests
2. **Noop on election**: Leader appends noop to force commitment of previous-term entries
   (required by Raft safety property)
3. **Single-node commit**: `_try_advance_commit()` called in `submit()` so single-node
   clusters don't need peer responses to commit
4. **Pre-vote as opt-in**: `pre_vote_enabled` flag -- when true, prevents isolated nodes
   from inflating terms during partitions

## Bug Found & Fixed

- Single-node clusters couldn't commit because `_try_advance_commit` was only called
  from `_handle_append_entries_response` (which requires peer responses). Fixed by
  calling it in `submit()` before sending heartbeats.
