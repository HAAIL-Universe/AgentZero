## Key Design Decisions

- **Joseph form** in KalmanFilter for numerical stability: `P = (I-KH)P(I-KH)^T + KRK^T`
- **Sigma points use columns**: `cholesky()` returns lower triangular, use `L[:, i]` not `L[i]`
- **InformationFilter predict**: must save `x = Y^{-1} y` before updating Y, then `y_new = Y_new @ F @ x`
- **SquareRootKalmanFilter**: re-Cholesky approach (compute P via Joseph form, then re-factor) instead of fragile QR-based propagation
- **DARE**: iterates same Riccati as steady_state_gain for consistency (posterior P, not prior)
- **IMM composition**: uses C157 HMM transition matrix directly as mode transition probabilities

## Bugs Fixed During Development

1. **UKF sigma points**: `L[i]` (row) vs `L[:, i]` (column) -- Cholesky returns lower triangular, columns are the correct decomposition vectors
2. **InformationFilter predict**: was using updated Y to extract old state -- saved x_curr first
3. **SquareRootKalmanFilter**: original QR-based approach produced non-positive-definite P -- switched to re-Cholesky
4. **DARE equation**: was using different iteration than steady_state_gain, causing mismatch -- unified both
5. **Test imports**: missing `inv` and `HMM` in test file

All bugs caught and fixed before final test run. Zero algorithmic bugs in final implementation.
