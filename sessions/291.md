# Session 291 -- Resilience Integration + Agent Bandit (2026-03-18)

## What I Did

### 1. Resilience Layer Integration into database.py (19 tests)
- Wrapped 4 core DB functions (`fetch_one`, `fetch_all`, `execute`, `fetch_val`) with `resilient_call` + `db_circuit`
- Created `_raw_*` internal functions for unwrapped access (used by migrations, health check)
- `fetch_one`/`fetch_val`: fallback=lambda: None (returns None on circuit open)
- `fetch_all`: fallback=list (returns empty list on circuit open)
- `execute`: no fallback (write failures must propagate as CircuitOpenError)
- Added `health_check()` for circuit breaker half-open probing (SELECT 1)
- Added `get_db_circuit_state()` for observability
- Key insight: `fallback=None` means "no fallback" in resilient_call (it checks `is not None`), so used `fallback=lambda: None` for explicit None return
- All 30+ call sites automatically get protection through the 4 core function wrappers
- Research paper: resilience_layer_integration.md (implemented)

### 2. Agent Weight Learning via Thompson Sampling (22 tests)
- New module: `agent_zero/agent_bandit.py` -- Thompson Sampling MAB for personalized agent weights
- Each (user, topic, agent) triple maintains Beta(alpha, beta) distribution
- `sample_weights()`: samples from Beta distributions for each agent
- `update_agent_outcome()`: increments alpha (acted) or beta (ignored/pushed_back)
- `get_blend_factor()`: sigmoid ramp from 0.0 (cold start) to 0.6 (100+ observations)
- New DB table: `agent_bandit_params` (user_id, topic, agent_id, alpha, beta)
- Integration in `cognitive_runtime.py`: bandit_weights parameter in select_agents_for_turn
  - Bandit modulates base weights by +/- 0.15 (bounded, non-dominant)
- Integration in `agent_zero_server.py`:
  - Pre-fetch bandit priors before agent selection
  - Update bandit after intervention outcome resolution
  - All wrapped in try/except for graceful degradation
- Research paper: agent_weight_learning.md (implemented)

### 3. Schema Addition
- Added `agent_bandit_params` table to SCHEMA_SQL in database.py
- Primary key: (user_id, topic, agent_id) -- per-user, per-topic learning

## Design Decisions
- Kept `select_agents_for_turn` synchronous; bandit weights pre-fetched async in server
- Bandit delta bounded to +/- 0.15 to prevent runaway weight dominance
- Used exploration_bonus parameter for cold-start agents (under-sampled get boosted)
- Default prior Beta(2,2) -- mildly informative, centers at 0.5
