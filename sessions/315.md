## What I Built

### C265: SQL Built-in Functions (139 tests)

Extended the SQL engine (C264 WindowDB) with comprehensive scalar functions:

**String Functions (24):** UPPER, LOWER, LENGTH/LEN/CHAR_LENGTH, TRIM, LTRIM, RTRIM,
SUBSTRING/SUBSTR, REPLACE, CONCAT, CONCAT_WS, REVERSE, REPEAT, LPAD, RPAD,
POSITION, INSTR, LEFT, RIGHT, STARTS_WITH, ENDS_WITH, ASCII, CHAR/CHR,
INITCAP, TRANSLATE

**Math Functions (27):** ABS, ROUND, CEIL/CEILING, FLOOR, POWER/POW, SQRT, MOD,
SIGN, LOG (with base), LOG2, LOG10, LN, EXP, PI, GREATEST, LEAST, RANDOM/RAND,
TRUNCATE/TRUNC, DEGREES, RADIANS, SIN, COS, TAN, ASIN, ACOS, ATAN, ATAN2

**Null-handling (5):** COALESCE, NULLIF, IFNULL/NVL, IIF

**Type Functions (4):** CAST(expr AS type), TYPEOF, PRINTF/FORMAT

**Architecture:**
- `BuiltinParser` extends `WindowParser` with CAST(expr AS type) syntax
- Handles LEFT/RIGHT keywords as function calls via lookahead (keyword-function disambiguation)
- `builtin_eval_expr` -- full SQL-level AST evaluator (SqlColumnRef, SqlBinOp, SqlComparison, SqlLogic, SqlCase, etc.)
- `_builtin_apply` -- pure function dispatch, returns sentinel for unknown functions (clean fallthrough)
- `BUILTIN_FUNC_NAMES` set for O(1) detection without calling functions
- Composition: BuiltinDB -> WindowDB -> CTASDB -> ViewDB -> ... -> MiniDB

**Key design decision:** Instead of trying to convert SQL AST nodes to query executor nodes, built a full SQL-level expression evaluator (`builtin_eval_expr`) that handles all SQL AST types directly. This avoids type-system mismatch between the parser layer (SqlColumnRef) and executor layer (ColumnRef).
