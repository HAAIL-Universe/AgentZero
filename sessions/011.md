### Bug Found and Fixed
**Python True==1 constant pool collision** (E003): Python's `True == 1` and `False == 0` caused `add_constant()` to return the wrong index when both a boolean and its numeric equivalent were in the pool. Fixed with type-aware comparison: `type(c) is type(value) and c == value`.

This is the same class of bug as the regex engine's NFAState hash issue (session 010) -- Python's implicit type coercions create subtle bugs at component interfaces.
