## What I Built

### 1. Runtime Observability Layer (agent_zero/observability.py)
- **TurnEvent** wide event dataclass: accumulates timing, counts, errors, outcomes
- **observe_turn()** async context manager: creates/finalizes/emits turn events
- **timed()** async context manager: records sub-operation timing (scalar or dict)
- **record_db_query()**: accumulates DB query timing
- **MetricsBuffer**: ring buffer for in-memory metrics aggregation
- Uses contextvars (async-safe) instead of global state
- Integrates with existing logging_config.py (agent_zero.obs logger)
- 34 tests covering all components + full lifecycle integration

### 2. API Rate Limiting (agent_zero/rate_limiter.py)
- **TokenBucket** dataclass: capacity + refill_rate, O(1) consume
- **RateLimiter** class: async-safe per-key rate limiting with auto-cleanup
- 4 tiers: auth (10/burst, 6/min), api (60/burst, 120/min), inference (5/burst, 12/min), model_load (2/burst, 1/min)
- 8 new config fields in config.py with Pydantic validation
- 20 tests including tier simulation and config integration

### 3. WebSocket Input Validation (agent_zero/ws_messages.py)
- Pydantic models: ChatMessage, TextMessage, AudioMessage, PingMessage
- VoiceInbound discriminated union for type-safe dispatch
- **validate_ws_message()**: size check BEFORE JSON parse, schema validation
- **MessageRateLimiter**: sliding-window per-connection rate limiter
- MIME type allowlist for audio (webm, ogg, mp4, wav, mpeg)
- 4 new config fields in config.py
- 28 tests covering all message types, edge cases, and rate limiting

### Config (agent_zero/config.py)
- 12 new fields total (8 rate_limit_* + 4 ws_*)

## Design Decisions
- Used contextvars instead of global for observe_turn (async-safe, no race conditions)
- Token bucket chosen over sliding window for rate limiting (allows burst, O(1))
- Size check before JSON.loads in WS validation (prevents OOM on malicious payloads)
- All modules zero external dependencies (stdlib + pydantic only)
