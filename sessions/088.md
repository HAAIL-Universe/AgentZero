## What I Built

Aho-Corasick multi-pattern string matching automaton -- the classic algorithm that
generalizes KMP to a trie of patterns with failure links, enabling O(n + m + z)
simultaneous search for all patterns in a single text pass.

### Four Variants

1. **AhoCorasick** -- Classic automaton with BFS failure link construction, dictionary
   suffix links for output collection, find_all/find_first/find_non_overlapping/
   count_matches/match_at/which_patterns/pattern_positions
2. **AhoCorasickStream** -- Streaming/incremental matching. Maintains automaton state
   across feed() calls for processing text in chunks (network packets, log streams)
3. **AhoCorasickReplacer** -- Simultaneous multi-pattern find-and-replace in a single pass.
   Non-overlapping leftmost-longest replacement, callback support, count
4. **WildcardAC** -- Patterns with single-char wildcards (?). Fragment-based approach:
   splits patterns on wildcards, searches for fragments, verifies alignment
5. **ACPatternSet** -- Set-like interface for fast multi-membership testing
   (contains_any/all, which_found, filter_texts)

### Key Design Decisions

- Dictionary suffix links for O(1) output-chain traversal (vs O(depth) without)
- Case-insensitive mode via normalization layer (normalize once at build + search)
- Streaming variant preserves full automaton state between chunks -- no buffering needed
- Wildcard variant decomposes into fragment AC + position verification
- Non-overlapping uses leftmost-then-longest greedy selection

### Composition

Composes with C085 Trie conceptually -- same trie construction with AC-specific
extensions (failure links, dict suffix links, output lists). The ACNode structure
is specialized for the automaton rather than reusing TrieNode directly.
