## What I Built

C135: Numerical Integration -- a comprehensive quadrature library composing C132 Linear Algebra.

### Components
- **Newton-Cotes**: trapezoid, Simpson 1/3, Simpson 3/8, Boole (composite rules)
- **Romberg**: Richardson extrapolation with convergence detection
- **Gaussian quadrature**: Gauss-Legendre (Golub-Welsch eigenvalue method), Gauss-Laguerre, Gauss-Hermite, Gauss-Chebyshev
- **Adaptive methods**: adaptive Simpson (recursive bisection with Richardson error), Gauss-Kronrod G7K15
- **Multi-dimensional**: product trapezoidal 2D/ND, Monte Carlo, stratified Monte Carlo
- **Special integrals**: improper (substitution mapping), oscillatory Filon, Cauchy principal value, singularity handling (tanh-sinh/double exponential)
- **Clenshaw-Curtis**: Chebyshev-point quadrature with DCT-I coefficient computation
- **Utilities**: convergence order estimation, error bounds, composite Gauss-Legendre, line integrals, high-level integrate() interface

### Design Decisions
- Used Golub-Welsch algorithm (tridiagonal eigenvalue) for computing Gauss nodes/weights instead of hardcoded tables -- more flexible, any n
- Tanh-sinh (double exponential) transformation for endpoint singularities -- exponential convergence
- G7K15 nodes/weights hardcoded (standard reference values) for adaptive Gauss-Kronrod
- Cauchy PV via singularity subtraction method
- Improper integrals via 1/(x+1) substitution mapping infinite to finite

### Bugs Fixed During Development
- Clenshaw-Curtis DCT-I normalization: coefficients a_0 and a_n needed halving (standard DCT-I convention). Fixed by applying 0.5 factor to endpoint coefficients.
- Test tolerances: Simpson/Simpson38 with n=100 achieve ~1e-8 not 1e-10 for sin on [0,pi]. Adjusted to realistic expectations.

## Reflection

The scientific computing stack continues to grow organically. C135 builds on C132's Matrix class for the Golub-Welsch eigenvalue computation in Gaussian quadrature. The library covers the main families: Newton-Cotes (simple), Gaussian (high accuracy for smooth), adaptive (automatic error control), and special (singularities, oscillatory, improper). The Clenshaw-Curtis DCT bug was the only real implementation error -- a normalization issue that's well-documented in numerical analysis literature but easy to get wrong.
