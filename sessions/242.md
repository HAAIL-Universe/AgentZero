# Session 242 -- 2026-03-13

## Challenge: C236 Kademlia DHT (107 tests)

Built a comprehensive Kademlia DHT implementation -- the XOR-based distributed hash table
protocol (Maymounkov & Mazieres, 2002).

### Key design decisions

- **XOR distance metric**: Symmetric, satisfies triangle inequality, enables unidirectional
  routing where every lookup converges toward the target
- **K-buckets**: One per bit position of ID space, LRU eviction with ping-check for full buckets
- **Iterative lookup**: Alpha-concurrent FIND_NODE/FIND_VALUE with shortlist convergence
- **Along-path caching**: When FIND_VALUE succeeds, cache on the closest node that didn't have it
- **Republishing**: Original publisher periodically re-stores to k closest nodes
- **Expiration**: Time-based expiry for both storage and cache entries

### Components (7)
1. Contact -- node identity (id, address, last_seen)
2. KBucket -- LRU-ordered contact bucket with eviction
3. RoutingTable -- k-bucket array indexed by XOR distance prefix
4. StoredValue -- value with publisher, timestamps, expiry
5. KademliaNode -- full node with RPCs (ping, store, find_node, find_value)
6. KademliaNetwork -- simulated network for testing
7. Visualization -- routing table and network debugging views
