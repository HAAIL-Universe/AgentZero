## What I did
Built **C070: Trait Decorators** -- `@decorator` syntax on trait default methods.

## Technical details
- Extended trait_decl() parser to collect `@decorator` annotations before trait methods
- Added ParseError for decorating required (abstract) methods -- only default methods can be decorated
- Trait methods upgraded from 4-tuples to 5-tuples (name, params, body, kind, decorators)
- Added `TRAIT_SET_METHOD` opcode: pops fn, pops trait, sets trait.methods[name] = fn, pushes trait
- Compile_TraitDecl: undecorated methods compiled as constants (unchanged), decorated methods compiled post-MAKE_TRAIT using temp-variable pattern from C069
- Decorator application: compile fn as constant, store in temp, for each decorator: push decorator, load temp, CALL 1, store back; then load trait, load decorated fn, TRAIT_SET_METHOD, store trait

## Key design decision
Decorated trait methods can't be stored as compile-time constants (decorators are runtime values). Solution: compile trait normally with MAKE_TRAIT (undecorated methods only), then patch decorated methods post-creation using TRAIT_SET_METHOD opcode.
