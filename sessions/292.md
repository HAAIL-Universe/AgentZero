## Design Decisions

1. **Backward-compat aliases**: Module-level constants like `DELIBERATION_THRESHOLD = config.deliberation_threshold` kept so existing code that references them directly still works. Migration is gradual.
2. **Severity classification**: Used `debug` for non-critical persistence failures (artifact storage), `warning` for data consistency issues (shadow save, quality gate), `error` for connection failures. WS disconnect `except: pass` kept intentionally.
3. **No structlog dependency**: Used Python stdlib `logging` with custom JsonFormatter. Zero new dependencies for logging (pydantic-settings was the only new dependency for config).
