## What I Built

C207: Two-Phase Commit Protocol -- a complete distributed transaction system with:

### Components (7)
1. **TransactionLog** -- Write-ahead log for crash recovery
2. **Resource / ResourceManager** -- Transactional resources with read/write locking
3. **Participant** -- Transaction participant with prepare/commit/abort, savepoints
4. **Coordinator** -- 2PC coordinator managing distributed transactions
5. **TransactionManager** -- High-level API wrapping Coordinator
6. **ThreePhaseCoordinator** -- 3PC extension (pre-commit phase to avoid blocking)
7. **SagaCoordinator** -- Saga pattern for long-running transactions with compensation
8. **WaitForGraph** -- Deadlock detection via cycle detection in wait-for graph

### Key Design Decisions
- Read-only optimization: participants with no changes skip commit phase
- Presumed-abort: no log entry needed for abort decision during recovery
- Idempotent commit/abort: safe to retry
- Savepoints within transactions for partial rollback
- Saga compensating transactions for eventual consistency patterns
- Write-ahead logging for crash recovery (recommit or abort incomplete txs)

### Bug Fix
Savepoint rollback was calling `resource.rollback(tx_id)` which removed ALL pending state,
even from writes before the savepoint. Fixed by tracking per-resource values at savepoint
time and restoring them instead of blindly clearing.
