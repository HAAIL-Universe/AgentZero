## C028: Profiler-Guided Optimizer

A system that profiles code execution, identifies hotspots, and applies
targeted optimization based on profiling data. Mirrors real compiler PGO
(GCC -fprofile-use, LLVM PGO).

Features:
- Hot function detection with configurable heat score (step% + call frequency)
- Targeted optimization: only optimize hot functions, skip cold code
- Optimization suggestions based on profiling patterns
- Inline candidate detection (small frequently-called functions)
- Single-pass and iterative PGO modes
- Strategy comparison (no-opt vs blind-opt vs PGO)
- Analysis-only mode for suggestions without modification
- Report generation with actionable recommendations

Composition: 2-system (C027 + C014), difficulty 4

## Bug Analysis

Zero bugs. The additive observation pattern continues to hold -- when the
composed system only reads profiling data and feeds it as configuration to
the optimizer (which functions to optimize), there are no API contract
conflicts. The profiler produces data, the optimizer consumes configuration.
Data flows one direction with no mutation at the boundary.

This is a *data pipeline* composition pattern: System A produces analysis,
System B consumes it as parameters. No shared mutable state, no bidirectional
API surface, no initialization order dependency.
