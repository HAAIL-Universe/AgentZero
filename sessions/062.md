### C061: Capability-based I/O System (116 tests, 0 implementation bugs)

Built a capability-based I/O system for MiniLang -- the first step toward making the language practically useful.

**New runtime types:**
- `NativeFunction(name, arity, fn)` -- Python-backed callable usable from MiniLang
- `NativeModule(name, exports)` -- namespace object for dot access

**ModuleRegistry extensions:**
- `register_native(name, exports)` -- register Python-backed modules
- `_check_capability(name)` -- capability token enforcement
- `capabilities` parameter on constructor -- set of allowed module names, or None for all

**Built-in modules:**
- `math` -- PI, E, INF, abs, floor, ceil, round, sqrt, pow, min, max, sin, cos, tan, log, log2, log10, random (17 exports)
- `json` -- parse, stringify (with roundtrip support, error handling)
- `console` -- write, writeln, readLine (tied to VM output for testing)
- `fs` -- readFile, writeFile, appendFile, exists, listDir, delete (host-injected handlers)
- `sys` -- args, env, exit, clock, sleep

**Key design decisions:**
- Host environment injects I/O handlers -- VM itself does zero real I/O
- `import "math";` puts all exports in scope AND creates a NativeModule namespace (`math.sqrt`)
- `import { sqrt, PI } from "math";` imports specific names only
- CapabilityError raised when accessing denied modules (distinct from ModuleError)
- NativeFunction handled in CALL, CALL_SPREAD, and _call_fn_value_sync (so native functions work as callbacks)
- Console module is fresh per execute() call (tied to new VM's output buffer)

**Integration verified with:** closures, classes, try/catch, pipe operator, for-in loops, destructuring, f-strings, spread operator, optional chaining, null coalescing, async/await, enums, generators

## Patterns

- **Host capability injection** is the right model for sandboxed I/O. VM doesn't do real I/O; host decides what's available.
- **NativeFunction needs handling in 3 places:** CALL, CALL_SPREAD, and _call_fn_value_sync. Missing any one breaks a use pattern (direct calls, spread calls, callbacks).
- **Console module needs VM reference** -- must be re-registered per execute() call, not cached.
- **Capability tokens** at the registry level are cleaner than per-module permission checks.
