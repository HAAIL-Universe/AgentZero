## What I Built

**C120: Fibonacci Heap** -- 5 variants + 2 graph algorithms:

1. **FibonacciHeap** -- Min-heap with O(1) amortized insert/decrease_key, O(log n) extract_min. Full implementation: circular doubly-linked root list, lazy consolidation (binomial-tree merging), cascading cuts for decrease_key.

2. **MaxFibonacciHeap** -- Max-heap via key negation. Wraps FibonacciHeap internally. O(1) insert/increase_key.

3. **FibonacciHeapMap** -- Key-value priority queue with O(1) name lookup. Supports insert/update/delete/pop by name. Handles both decrease and increase (delete+reinsert for increase).

4. **MergeableFibonacciHeap** -- Named heap collection with O(1) merge between any two named heaps.

5. **FibonacciHeapPQ** -- Clean priority queue interface. Supports both min and max mode. push/pop/peek/update/delete/contains.

6. **dijkstra()** -- Shortest path using FibonacciHeapPQ. O(V + E log V) with Fibonacci heap.

7. **prim_mst()** -- Minimum spanning tree using FibonacciHeapPQ.

## Key Design Decisions

- **Circular doubly-linked lists** for O(1) splice/merge -- separate helper functions keep node manipulation clean
- **FibNode uses __slots__** for memory efficiency (no per-instance dict)
- **MaxFibonacciHeap stores negated keys** internally rather than duplicating the entire structure with reversed comparisons
- **FibonacciHeapMap handles increase_key** via delete+reinsert since Fibonacci heaps only support decrease natively
- **Consolidation uses dynamic degree table** sized by log2(n) -- extends if needed rather than over-allocating
