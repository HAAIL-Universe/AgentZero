## What I Built

A comprehensive probabilistic data structures library implementing 7 structures:

1. **Standard Bloom Filter** -- optimal m/k calculation, FPR estimation, Swamidass-Baldi count estimation, union/intersection/Jaccard similarity, serialization
2. **Counting Bloom Filter** -- 8-bit counters supporting deletion, count-of queries
3. **Partitioned Bloom Filter** -- k partitions for better cache locality
4. **Scalable Bloom Filter** -- auto-grows with geometric FPR tightening
5. **Cuckoo Filter** -- fingerprint-based with add/query/delete, power-of-2 buckets, cuckoo eviction
6. **HyperLogLog** -- cardinality estimation with small/large range corrections, merge support
7. **Count-Min Sketch** -- frequency estimation, heavy hitters, merge
8. **TopK (Space-Saving)** -- streaming top-k tracking

Supporting infrastructure:
- MurmurHash3 and FNV-1a hash functions (pure Python)
- Kirsch-Mitzenmacher double hashing for k positions
- Compact BitArray with set operations and serialization

## New Domain

This is the first purely probabilistic DS challenge. Previous work was deterministic (B-trees, skip lists, ropes). Probabilistic structures trade exactness for space/time efficiency -- a fundamentally different design philosophy.

## Key Design Decisions

- **Double hashing** (Kirsch-Mitzenmacher): Generate k hash positions from just 2 hash functions. Simpler than k independent hashes with negligible FPR increase.
- **Pure Python hashes**: MurmurHash3 + FNV-1a implemented from scratch rather than using hashlib. Gives full control over seeding.
- **Scalable BF growth**: Geometric FPR tightening (each new filter gets r^level of the FPR budget) ensures overall FPR converges.
- **Cuckoo power-of-2 buckets**: Enables XOR-based alternate indexing -- key property of cuckoo hashing.
