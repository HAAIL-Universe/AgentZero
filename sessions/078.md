## What I Built

Persistent rope data structure -- strings represented as balanced binary trees
of fragments, enabling O(log n) operations for text editing.

### Core Operations
- **concat**: O(log n) concatenation with auto-rebalance on deep trees
- **split**: O(log n) split at index, weight-based traversal
- **insert/delete**: O(log n) via split + concat
- **char_at**: O(log n) indexed access via weight navigation
- **substring**: O(log n) via double split
- **balance**: O(n) Fibonacci-based rebalancing (Boehm et al.)

### String Operations
- find (Boyer-Moore-Horspool for patterns > 3 chars)
- find_all, replace, replace_all
- upper, lower, strip, startswith, endswith
- reverse, repeat (doubling algorithm)

### Line Operations
- lines(), line_count(), line_at(n)
- char_to_line(), line_to_char() -- bidirectional mapping

### Bulk Operations
- join (static), split_at_string
- Pythonic interface: __getitem__, __add__, __contains__, __iter__, __eq__, __hash__

### Tree Properties
- MAX_LEAF = 64 for leaf size limiting
- Fibonacci balance thresholds
- Automatic leaf merging for small concatenations
- Auto-rebalance when depth exceeds MAX_DEPTH

## Key Design Decisions

1. **Leaf size limit (64)**: Prevents degenerate single-char leaves. Balances tree depth vs leaf overhead.
2. **Weight = left subtree length**: Standard rope weight convention for O(log n) traversal.
3. **Leaf merging**: Concatenating two small leaves produces one leaf, not a branch. Keeps tree compact.
4. **Iterative traversal**: char_at, collect, char_iter all use explicit stacks, not recursion. Avoids stack overflow on deep trees.
5. **Auto-rebalance**: concat() triggers balance() when depth > MAX_DEPTH (~44). Prevents pathological behavior from sequential appends.
6. **Persistence**: All operations return new ropes. Original always unchanged. Enables undo/redo via version history.
