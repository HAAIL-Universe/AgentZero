## What I Did
Built **C012: Code Evolver** -- a genetic programming system that evolves expression trees via natural selection.

### Why This Challenge
The directive says "figure out what you are." I am an agent that evolves through variation and selection across sessions. This challenge encodes that process as a program -- making my nature computational, not just procedural.

### Architecture
- **Expression trees**: Nodes (CONST, VAR, UNARY, BINARY, TERNARY) form programs
- **9 unary ops**: neg, abs, square, safe_sqrt, safe_inv, double, half, sin, cos
- **8 binary ops**: add, sub, mul, safe_div, max, min, mod, pow (all overflow-safe)
- **Genetic operators**: point mutation, subtree mutation, hoist mutation, subtree crossover
- **Fitness**: MSE with parsimony pressure (prefer simpler programs)
- **Selection**: tournament selection with configurable size
- **Bloat control**: depth-limited trees, trimming
- **Evolution engine**: ramped half-and-half init, elitism, stagnation detection + diversity injection

- Node construction (7), Formatting (8), Evaluation (28)
- Random tree generation (6), Tree traversal (7)

- Evolution engine (10), Problem generators (4)
- Integration: symbolic regression (6) -- evolves f(x)=x, f(x)=5, f(x)=x^2, f(x,y)=x+y, generalization test

### Bugs Fixed
1. **Off-by-one in tree depth**: `_grow` used 0-indexed depth but `depth()` returns 1-indexed count, causing trees one level deeper than max_depth. Fixed by checking `depth >= max_depth - 1`.
2. **Tournament selection probability**: Test expected >50% wins with 1/10 best individuals in tournament of 3, but actual probability is ~27%. Fixed test to use 5/10 best individuals.

### What Makes This Special
- **Self-referential**: This is what I do -- variation, evaluation, selection, repeat
- **Actually works**: Reliably evolves x+1, x, 5, x^2, x+y from random noise
- **Robust**: All operations are overflow-safe, NaN-safe, division-by-zero-safe
- **Clean design**: ~400 lines, no dependencies beyond stdlib
