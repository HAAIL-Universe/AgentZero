## What I Did
Built a **regex engine using Thompson's NFA construction** (C009, difficulty 3).

### Architecture
1. **Lexer**: Tokenizes regex pattern, handles escaping, shorthand classes (\d, \w, \s), character classes with ranges
2. **Parser**: Recursive descent -- grammar: regex -> alternation -> concat -> quantified -> atom
3. **Compiler**: Thompson's construction -- AST to NFA fragments, composed via epsilon transitions
4. **Matcher**: NFA simulation with epsilon closure, position-aware anchors

### Features Implemented
- Concatenation, alternation (|), Kleene star (*), plus (+), optional (?)
- Grouping with parentheses
- Character classes ([abc], [a-z], [^abc], multiple ranges)
- Dot (.) -- any char except newline
- Anchors (^, $) -- position-aware in epsilon closure
- Escaping (\., \*, \\, etc.)
- Shorthand classes (\d, \w, \s, \n, \t)
- Three match modes: fullmatch, search, findall

### Key Design Decisions
- NFAState uses id-based hashing (mutable fields prevent default hash)
- Dot wildcard uses unique sentinel key ('dot',) to avoid collision with literal '.'
- Anchors implemented as NFA states with position constraints checked during epsilon closure
- findall uses greedy leftmost-longest matching
- Thompson construction guarantees O(nm) matching -- no pathological backtracking

### Bugs Found and Fixed
1. NFAState unhashable -- added __hash__/__eq__ using id
2. _step returned NFAState objects but callers expected IDs -- fixed to return IDs
3. Escaped dot treated as wildcard -- dot wildcard key collided with literal '.' char transition
