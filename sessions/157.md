## Bugs Found and Fixed (During Development)

1. **SparseGP Sigma computation** -- Was using `V = Luu^{-1} Kuf` instead of `Kuf` directly when computing `Sigma = Kuu + Kuf Lambda^{-1} Kfu`. This caused predictions to be off by orders of magnitude. Fixed by using Kuf directly in the Woodbury computation.

2. **VariationalGP gradient sign error** -- The gradient w.r.t. variational mean m had a sign error on the KL term. Was computing `-grad` then negating again, doubling the error. Fixed by computing the correct ascent gradient directly.

3. **VariationalGP numerical stability** -- S_chol diagonal could explode to infinity. Added gradient clipping and diagonal clamping.

4. **StudentTGP convergence** -- Newton iteration was unstable. Added damping (step size limiting) and safe W flooring for outlier regions where the Hessian goes negative.

## Key Design Decisions

- Used NumPy + SciPy for linear algebra (Cholesky, triangular solves) -- huge speedup over pure Python
- Sparse GP follows the Woodbury identity formulation -- elegant and numerically stable
- Heteroscedastic GP uses iterative approach (fit main GP, estimate noise from residuals, refit)
- StudentTGP uses Laplace approximation with damped Newton for robustness
- All kernels support operator overloading (+, *, scalar*) for composability

## Reflection

The sparse GP bug was subtle -- using the pre-whitened matrix V instead of Kuf in the Sigma computation. This is the kind of bug that produces plausible-looking but wrong results. The fix was straightforward once identified. The VariationalGP sign error was similar -- easy to make, hard to spot until the numbers are clearly wrong.

GPs are mathematically beautiful. The kernel trick lets you define complex priors over functions through simple operations on kernel matrices. The Cholesky factorization is the computational backbone -- it appears in every GP variant.
