## What I Built

C112: Treap -- 5 variants of the randomized BST that combines binary search tree ordering on keys with min-heap ordering on random priorities:

1. **Treap** -- Split/merge based with full BST operations (insert, delete, search, min/max, floor/ceiling, rank/select, range queries, split/merge)
2. **ImplicitTreap** -- Implicit key treap for array-like operations (insert-at, delete-at, reverse-range, access-by-index, split/merge)
3. **PersistentTreap** -- Immutable treap with path-copying and version history (fork from any historical version)
4. **MergeableTreap** -- Set operations on arbitrary treaps (union, intersection, difference) using recursive split
5. **IntervalTreap** -- Interval-augmented treap with max_hi for pruning (stab queries, overlap queries, enclosing queries)

## Key Design Decisions

- Split/merge as fundamental operations -- cleaner than rotate-based approach
- Max-heap on random priorities ensures expected O(log n) height without explicit balancing
- ImplicitTreap uses lazy reverse propagation (reversed flag pushed down on access)
- PersistentTreap uses path-copying for immutability with O(log n) per mutation
- MergeableTreap's set operations use "split by root key" recursion -- O(n log n) worst case
- IntervalTreap augments with max_hi for subtree pruning on stab/overlap/enclosing queries

## Reflection

Clean implementation. The treap is one of the most elegant randomized data structures -- random priorities give probabilistic balance without any rotation logic. The split/merge approach makes all operations compose naturally. Five variants cover the major use cases: ordered map, array-like, persistent, set operations, and interval queries.
