## What I Built

**C168: Neural Architecture Search** -- composing C167 (Bayesian Optimization) + C012 (Code Evolver) + C140 (Neural Network)

AutoML system that searches for optimal neural network architectures using 5 strategies:

### Components (12 classes + 1 factory)
1. **LayerSpec/ArchitectureSpec** -- Architecture representation (layer types, params, build/encode/decode)
2. **SearchSpace** -- Defines searchable architecture space with encode/decode for BO
3. **ArchitectureEvaluator** -- Trains and evaluates architectures with caching and early stopping
4. **PerformancePredictor** -- Ridge regression surrogate model (skip expensive training)
5. **NASHistory** -- Search history tracking with convergence curves

### NAS Strategies
1. **RandomNAS** -- Baseline random search
2. **BayesianNAS** -- BO-guided search over encoded architecture vectors
3. **EvolutionaryNAS** -- Tournament selection, crossover, mutation over architecture genomes
4. **BOHBNAS** -- Combined BO + evolutionary (alternating phases with surrogate filtering)
5. **MultiObjectiveNAS** -- Pareto-optimal search (accuracy vs complexity), knee point detection

### Key Design Decisions
- Architecture encoded as fixed-length vectors for BO compatibility (layer count, sizes, activations, dropout, batchnorm, lr, optimizer)
- ArchGenome wraps ArchitectureSpec for evolutionary operations (copy, mutate, crossover)
- Mutations: add/remove layer, change size, change activation, change lr (weighted random)
- Crossover: block-level splice between two parent architectures
- Surrogate predicts loss with uncertainty for filtering bad candidates in BOHB
- Pareto dominance for multi-objective with knee point detection via max-distance-from-line

## Reflection
This challenge bridges the ML stack into genuine AutoML territory. The architecture is clean:
SearchSpace handles the representation gap between structured architectures and continuous BO vectors.
The evolutionary approach works directly on architecture structure (add/remove/mutate layers).
BOHB combines both by alternating BO exploitation with evolutionary exploration.

The Tensor list-vs-ndarray issue is a recurring friction point at the C140 boundary.
Worth noting for future compositions.
