## What I Built

C198: Advanced Time Series Forecasting -- 15 components, 108 tests.

Components:
- **GARCH(p,q)**: Volatility modeling via MLE, forecast conditional variance, persistence/unconditional variance
- **VAR(p)**: Vector autoregression, OLS fitting, impulse response functions, Granger causality, FEVD
- **KalmanFilter**: Full linear Gaussian state space (predict/update/filter/smooth/forecast), log-likelihood
- **LocalLevelModel**: Random walk + noise via Kalman, MLE parameter estimation
- **LocalLinearTrend**: Level + trend state space model via Kalman
- **ProphetDecomposition**: Piecewise linear trend + Fourier seasonality + changepoints, ridge regression
- **ThetaMethod**: Theta-line decomposition (M3 competition winner approach)
- **CrostonMethod**: Intermittent demand forecasting (separate demand size/interval smoothing)
- **HoltWinters**: Triple exponential smoothing with additive/multiplicative seasonality, grid optimization
- **EnsembleForecaster**: Multi-model combination (equal/inverse-error/stacking weights)
- **Backtester**: Walk-forward cross-validation (5 metrics: MSE, RMSE, MAE, MAPE, sMAPE)
- **PredictionInterval**: Parametric, bootstrap, and conformal prediction intervals
- **ForecastCombiner**: Static combination (average, median, trimmed mean, variance-weighted)
- **diebold_mariano_test**: Statistical test for comparing forecast accuracy
- **Data generators**: Seasonal, intermittent, GARCH, and VAR series generation

This extends C188 (basic time series analysis) with the full forecasting toolkit: volatility models, multivariate systems, state space methods, ensemble methods, backtesting, and statistical tests.

## Key Design Decisions

- Used scipy.optimize.minimize for GARCH MLE (L-BFGS-B with bounds ensures stationarity)
- Kalman filter is general-purpose -- LocalLevelModel and LocalLinearTrend are thin wrappers
- Prophet uses ridge regression for stability (not full Bayesian like real Prophet)
- EnsembleForecaster uses train/validation split for weight estimation
- Backtester is model-agnostic via factory pattern
