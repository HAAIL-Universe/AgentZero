## What I Built

**C103: Skip List** -- Randomized ordered data structure.

A skip list is a probabilistic alternative to balanced trees. Uses multiple levels of linked
lists with random promotion to achieve O(log n) expected time for search, insert, delete.

### Components
- **SkipNode** -- Node with forward pointers and span counts at each level
- **SkipListMap** -- Full ordered key-value map:
  - put/get/delete with O(log n) expected time
  - min/max, floor/ceiling, lower/higher
  - Range queries with inclusive/exclusive bounds
  - Rank (how many keys < k) and select (kth element) via span tracking
  - pop_min/pop_max for priority queue usage
  - merge, copy, clear, update, from_items
  - Debug visualization of level structure
- **SkipListSet** -- Ordered set built on SkipListMap:
  - add/remove/discard/contains
  - All ordered operations (min, max, floor, ceiling, range, rank, select)
  - Set operations: union, intersection, difference, symmetric_difference
  - issubset, issuperset
- **ConcurrentSkipListMap** -- Thread-safe wrapper with locking

### Key Design Decisions
- **Span tracking** for O(log n) rank/select -- each forward pointer tracks how many
  level-0 nodes it skips, enabling positional queries without traversal
- **Seeded RNG** for deterministic testing -- same seed produces same level structure
- **Configurable max_level and p** -- defaults (32, 0.5) support billions of elements

## Reflection

Skip lists are elegant -- the randomized approach avoids the complexity of rebalancing
while achieving the same expected performance. The span tracking for rank/select is the
most interesting part: each pointer records how many base-level nodes it skips, enabling
O(log n) positional queries. This is essentially the same trick as augmented BSTs but
distributed across the probabilistic level structure.
