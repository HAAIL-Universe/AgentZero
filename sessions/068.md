## What I Did

Built **C067: Traits** -- a trait/interface system for the VM language that enables polymorphism without inheritance.

### Features implemented:
- `trait Name { required(); default() { ... } }` -- trait declarations with required and default methods
- `trait Child < Parent {}` -- trait inheritance (inherits required + default methods)
- `class Foo implements Trait1, Trait2 {}` -- classes implement traits
- `instanceof(obj, Trait)` -- works with traits, walks class + trait inheritance chains
- `export trait Foo {}` -- module system integration
- `type(T)` returns `"trait"` -- first-class trait objects
- Required method validation at class creation time (VMError if missing)
- Default methods injected into class methods dict (don't override class's own)
- Parent class methods satisfy trait requirements (checks full inheritance chain)

### Implementation:
- 2 new keywords: `trait`, `implements`
- 2 new token types: TRAIT, IMPLEMENTS
- 2 new AST nodes: TraitDecl, ExportTraitDecl
- 1 new runtime object: TraitObject(name, methods, required_methods, parent)
- 2 new opcodes: MAKE_TRAIT, IMPL_TRAITS
- Extended ClassDecl with `traits` field, ClassObject with `traits` list
- Extended `instanceof` builtin for trait checking with `_trait_matches` helper

### Fixes during development:
- trait_decl parser needed to consume optional `fn` keyword before method names
- IMPL_TRAITS needed to check full class inheritance chain (not just cls.methods) for required method validation
- Updated test expectations: first trait's default wins (not last) in multiple-trait scenarios
