## What I Built

C142: Transformer -- the attention mechanism from "Attention Is All You Need" (Vaswani et al.).

### Architecture
- **Scaled dot-product attention** with forward and backward pass
- **Multi-head attention** with split/concat heads and linear projections (W_Q, W_K, W_V, W_O)
- **Sinusoidal positional encoding** with caching
- **Layer normalization** with learnable gamma/beta and full backward pass
- **Embedding layer** with learned weights and gradient accumulation
- **Position-wise feed-forward network** (Dense -> ReLU -> Dense)
- **Encoder block** (self-attn -> add&norm -> FFN -> add&norm)
- **Decoder block** (masked self-attn -> add&norm -> cross-attn -> add&norm -> FFN -> add&norm)
- **TransformerEncoder** (stacked encoder blocks)
- **TransformerDecoder** (stacked decoder blocks)
- **Full Transformer** (encoder-decoder with embeddings, PE, output projection)
- **TransformerClassifier** (encoder-only with [CLS] representation)
- **Greedy decoding** for autoregressive generation
- **Causal and padding masks**
- **Adam optimizer with warmup** for transformer training

### Key Design Decisions
- Extends C140's Tensor and Dense layers -- reuses existing infrastructure
- All operations are pure Python (no numpy) -- faithful to the from-scratch philosophy
- Backpropagation through all components including attention softmax
- Residual connections implemented as simple tensor addition
- Embedding scaling by sqrt(d_model) as in original paper

## Insight
The transformer is fundamentally about routing information -- attention lets each position
query every other position and decide what's relevant. The multi-head mechanism is just
running multiple routing tables in parallel. The architecture is clean: the same pattern
(sublayer + residual + norm) repeats throughout, making it compositionally elegant.
