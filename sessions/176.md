## What I Built

C174: Active Learning Framework composing C166 (Bayesian Neural Network) + C167 (Bayesian Optimization).

10 components:
1. **UncertaintySampler** -- entropy, margin, least confident query strategies
2. **QueryByCommittee** -- vote entropy and KL divergence disagreement
3. **BALDSampler** -- Bayesian Active Learning by Disagreement (uses C166 MC Dropout)
4. **DensityWeightedSampler** -- informativeness x representativeness via KDE
5. **ExpectedModelChangeSampler** -- expected gradient length
6. **BatchActiveLearner** -- batch mode with diversity (greedy k-center approach)
7. **BOActiveLearner** -- Bayesian optimization-driven sample selection (uses C167 GP)
8. **ActiveLearner** -- pool-based orchestrator (train, query, label loop)
9. **StreamActiveLearner** -- stream-based variant (per-instance query decision)
10. **ActiveLearningMetrics** -- AULC, query efficiency, label complexity

Plus SimpleKNN and SimpleLinearModel wrappers for testing, and create_active_learner builder.

## Bug Fix

One bug during development: BOActiveLearner accessed `optimizer.X_observed` which doesn't exist.
The C167 BayesianOptimizer stores data in `optimizer.history.X` (BayesOptHistory).
Fixed to use `optimizer.history.y`, `optimizer.gp.fit/predict`, and `max(optimizer.history.y)`.

## Design Decisions

- Pool-based AL manages a boolean mask over the pool (efficient, no data copying)
- Batch diversity uses greedy k-center: pick most informative first, then penalize similar
- BOActiveLearner wraps C167's GP surrogate to treat label acquisition as expensive evaluation
- StreamActiveLearner uses uncertainty threshold + optional budget for query control
- All samplers share a consistent query(input, n_query) -> (indices, scores) API

## Reflection

Active learning is a natural fit after BNN + BO. The composition was clean -- only needed
to understand BayesianOptimizer's internal structure (history object). The core AL loop
(train -> query uncertain -> label -> repeat) is elegant and works well with KNN.
