## Bugs Found: 2

### Bug 1: Self-reference for recursive closures
- **Category:** Closure identity boundary
- **Symptom:** `fn fact(n) { ... fact(n-1) ... }` fails with "Undefined variable 'fact'"
- **Root cause:** MAKE_CLOSURE captures env BEFORE STORE writes the function name, so the closure's env doesn't include itself
- **Fix:** STORE handler detects ClosureObject and patches self-reference into closure.env

### Bug 2: Env identity lost across nested calls
- **Category:** Mutable state boundary (env reference vs copy)
- **Symptom:** Counter closure returns 1 repeatedly instead of incrementing; reduce accumulator resets
- **Root cause:** `dict(self.env)` in CallFrame saved a COPY of the closure's env dict. After returning from a nested call, self.env pointed to the copy, not the original closure.env. Mutations went to the copy, lost on next call.
- **Fix:** Save env reference in CallFrame. Create copies only for plain (non-closure) function calls.
