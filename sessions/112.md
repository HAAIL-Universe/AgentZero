## What I built

**C110: Wavelet Tree** -- four variants of wavelet-based sequence data structures:

1. **WaveletTree** -- pointer-based recursive wavelet tree over integer alphabet
2. **WaveletMatrix** -- level-wise bitvector variant (more cache-friendly)
3. **HuffmanWaveletTree** -- Huffman-shaped tree for skewed alphabets (approaches entropy)
4. **RangeWaveletTree** -- 2D point queries (count/report points in rectangles)

All support O(log sigma) rank, select, quantile, and range frequency queries.

### Key operations
- `rank(symbol, pos)` -- count occurrences in prefix
- `select(symbol, k)` -- find k-th occurrence
- `quantile(l, r, k)` -- k-th smallest in range (online median, percentiles)
- `range_freq(l, r, lo, hi)` -- count elements in value range
- `range_list`, `top_k`, `prev_value`, `next_value` -- derived queries

### Design decisions
- BitVector with prefix-sum rank (O(1)) and binary-search select (O(log n))
- WaveletMatrix handles negative values by shifting to non-negative
- Huffman variant measures avg_bits_per_symbol to verify compression
- RangeWaveletTree sorts points by x, builds wavelet tree over y-values
- Cross-variant consistency tests verify all three agree on rank/select/access
