## What I Built

C126: Network Flow Algorithms -- a comprehensive flow algorithms library covering:

### Core Algorithms (4)
1. **Edmonds-Karp** -- Ford-Fulkerson with BFS augmenting paths, O(VE^2)
2. **Dinic's Algorithm** -- Level graph + blocking flows with current-arc optimization, O(V^2 E)
3. **Push-Relabel** -- FIFO with discharge/relabel, O(V^3)
4. **Min-Cost Max-Flow** -- Successive shortest paths via SPFA, O(V^2 E * C)

### Specialized Algorithms (1)
5. **Hopcroft-Karp** -- Bipartite matching via layered augmentation, O(E sqrt(V))

### Applications (5)
6. **Min-Cut** -- Via max-flow-min-cut theorem (on all 3 max-flow algorithms)
7. **Edge-Disjoint Paths** -- Via unit-capacity max-flow
8. **Node-Disjoint Paths** -- Via node-splitting reduction
9. **Circulation with Demands** -- Lower-bound feasibility via super-source/sink
10. **Assignment Problem** -- Via min-cost flow reduction
11. **Project Selection** -- Via min-cut with dependencies
12. **Vertex Cover / Independent Set** -- Via Konig's theorem on bipartite matching

### Infrastructure
- `FlowNetwork` -- Graph with residual edges, undirected edges, copy, reset
- `Edge` -- Slots-optimized edge with cap/flow/cost/rev

## Key Design Decisions

- **Residual graph via reverse edges**: Each add_edge creates a forward edge (cap=C) and reverse edge (cap=0). Augmenting adjusts both. Clean, standard approach.
- **Current-arc optimization in Dinic**: iter_map tracks progress through adjacency lists to avoid re-scanning dead ends. Critical for O(V^2 E) guarantee.
- **Source==sink guard**: All max_flow methods return 0 immediately when source==sink. Without this, BFS finds "path" of length 0 with infinite bottleneck, causing infinite loop. Caught during development.

## Reflection

Network flow is a foundational algorithms domain. The three max-flow algorithms (Edmonds-Karp, Dinic, Push-Relabel) represent three distinct paradigms: augmenting paths, blocking flows, and preflow-push. The applications demonstrate the remarkable versatility of max-flow reductions -- matching, cuts, circulation, assignment all reduce to flow problems.

The source==sink bug was interesting: an empty augmenting path with infinite bottleneck creates a silent infinite loop. Simple guard, but easy to miss.
