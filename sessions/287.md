## Key Design Decisions
- Chip persistence uses sessionStorage (not localStorage) -- same-tab scoped, no cross-tab interference
- Check-in prompt injected as `extra_system_prompt` appended to first turn's system prompt, not as a separate proactive message -- lets the model weave it naturally into its first response
- Circuit breaker uses class name matching (`exc_name in (...)`) instead of `isinstance` to avoid hard-importing asyncpg/httpx at module level
- Resilient_call applies full jitter (`random.uniform(0, wait)`) per AWS recommendation
