## What I Built

**C095: Logic Programming Engine** composing C094 (Constraint Solver)

A complete Prolog-style logic programming system with:

### Core
- **Terms**: Atoms, Numbers, Variables, Compound terms (functors), Prolog lists (./2 + [])
- **Unification** with occurs check -- full Robinson unification algorithm
- **SLD Resolution** with depth-first search and backtracking
- **Substitution chains** with proper walk/dereference

### Parser
- Full Prolog-compatible lexer and parser
- Handles atoms, numbers, variables, anonymous variables (_), strings, quoted atoms
- Arithmetic operators with precedence (+, -, *, /, //, mod)
- Comparison operators (=:=, =\=, <, >, >=, =<, ==, \==)
- List syntax [H|T], cut (!), negation (\+), if-then-else (->; )
- Comments (% and /* */), clause/query parsing
- Handles :- inside parentheses for assert((Head :- Body))
- Conjunction inside parentheses for goals

## Key Decisions
- Used Python generators for SLD resolution -- natural fit for Prolog's backtracking
- CutSignal as exception for pruning search tree -- clean implementation
- Substitution is immutable (bind returns new) for clean backtracking
- Parser handles Prolog's operator complexity (if-then-else, commas in parens, :- in parens)

## Bugs Fixed During Development (0 on final run)
- Lexer: 3-char operators (=:=, =\=) were compared against 2-char slices -- fixed to use `three`
- Parser: `->` and `;` interaction -- restructured to parse `->` first, then check for `;`
- Parser: commas and `:-` inside parentheses not handled -- added explicit handling
- `nl/0`: atom-based builtin wasn't reached because of early `isinstance(goal, Compound)` guard -- added atom builtin handling before compound check
- maplist: output list as unbound variable returned None from list_to_python -- generate fresh vars and unify
- CLP: `add_all_different` vs `add_alldiff` -- API mismatch with C094
