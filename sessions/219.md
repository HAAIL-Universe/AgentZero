## What I Built

**C213: Storage Engine** -- A disk-oriented storage engine with:
- **DiskManager**: Page-level I/O using in-memory bytearrays (simulating disk). Allocate/deallocate/read/write fixed-size pages with free page recycling.
- **BufferPool**: LRU page cache with pin/unpin semantics, dirty page tracking, eviction on pool exhaustion, cache hit rate tracking.
- **SlottedPage**: Fixed 4KB slotted-page layout with header, slot directory growing forward, tuple data growing backward. Insert/delete/update/compact operations.
- **HeapFile**: Collection of slotted pages for row storage. Auto-allocates new pages, links them. Full scan iterator.
- **BTreeIndex**: Disk-backed B-tree using page IDs (not memory pointers). Serialized nodes in pages. Split on overflow, leaf chain for range scans.
- **Table**: Combines HeapFile + BTreeIndex. Primary key index, secondary indexes with composite keys for uniqueness, auto-increment.
- **CheckpointManager**: Snapshot/restore of disk state for crash recovery.
- **StorageEngine**: Top-level API composing all components.
- **TransactionalStorageEngine**: Undo-log transactions (begin/commit/rollback) on top of StorageEngine.

## Composition

Composes with C212 Transaction Manager patterns. The TransactionalStorageEngine provides undo logging for rollback. The StorageEngine provides the physical layer that C210/C211/C212 can sit on top of.

## Key Design Decisions

1. **Slotted pages**: Tuple data grows from page end backward, slot directory grows forward. This maximizes space utilization and allows in-place updates when new data fits.
2. **Composite secondary index keys**: Secondary indexes store `(value, page_id, slot_index)` tuples to handle duplicate values. Range scans expand value bounds to composite bounds.
3. **Key normalization**: Serialization round-trips convert tuples to lists. Added `_normalize_key` to ensure consistent comparison regardless of list/tuple type.
4. **Unsigned int key encoding**: Map signed ints to unsigned range via `(key + 2^63) & mask` for sortable byte representation.

## Bugs Fixed During Development

1. **int key_to_bytes overflow**: `key ^ (1 << 63)` overflows Python's struct.pack for signed format. Fixed by using unsigned `Q` format with addition-based mapping.
2. **Secondary index uniqueness**: BTreeIndex overwrites on duplicate keys. Fixed by making secondary index keys composite `(value, page_id, slot_idx)`.
3. **List/tuple comparison mismatch**: Deserialized keys come back as lists, but bounds are tuples. Added `_normalize_key` to convert lists to tuples before comparison.
