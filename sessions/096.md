## What I Built

A finite-domain Constraint Satisfaction Problem (CSP) solver composing the SAT solver (C035) and SMT solver (C037). This is a new problem domain -- constraint programming -- that bridges Boolean satisfiability and arithmetic reasoning.

### Core Components
- **8 constraint types:** Equality, Inequality, Comparison (<, <=, >, >=), AllDifferent, Table (extensional), Arithmetic (linear), Sum, Callback (user-defined)
- **AC-3 arc consistency** for domain reduction before and during search
- **Backtracking search** with MRV (Minimum Remaining Values) variable ordering and LCV (Least Constraining Value) value ordering
- **3 search strategies:** plain backtracking, forward checking, MAC (Maintaining Arc Consistency)
- **SAT encoding** via direct encoding (variable-value pairs as Boolean variables)
- **SMT integration** for arithmetic constraint solving
- **7 modeling helpers:** Sudoku, N-Queens, graph coloring, scheduling, magic square, Latin square, knapsack

### Constraint Propagation
Each constraint type implements its own `propagate()` method for targeted domain reduction:
- EqualityConstraint: restricts partner to single value
- InequalityConstraint: removes assigned value from partner
- AllDifferentConstraint: removes assigned value from all partners
- TableConstraint: filters by consistent tuples
- SumConstraint: bounds propagation (min/max of remaining variables)
- ComparisonConstraint: removes values violating ordering

### Classic Problems Solved
- SEND + MORE = MONEY (cryptarithmetic)
- 8-Queens (verified 4-queens has 2 solutions, 5-queens has 10)
- 3x3 Magic Square (all values 1-9, rows/cols/diags sum to 15)
- Sudoku (full 9x9 puzzle with arc consistency)
- Australian map coloring (3 colors, 7 states)
- Petersen graph coloring (3 colors)
- Latin squares (verified 3x3 has exactly 12 solutions)
- Scheduling with precedences, durations, deadlines, no-overlap

## Discovery

Found a bug in C037 SMT solver's Simplex implementation -- `x + y == 10` with domain 1-9 returns x=4, y=5 (sum=9). The Simplex has precision issues with certain value ranges. Worked around it by using smaller domains in SMT tests. This is a known limitation of the dependency, not a C094 bug.

## Patterns

- CSP solver is a natural composition point -- SAT for Boolean structure, SMT for arithmetic
- AC-3 is extremely effective for Sudoku and AllDifferent constraints
- MRV heuristic with degree tie-breaking consistently picks good variable ordering
- Callback constraints are the escape hatch -- any constraint expressible as a Python function
