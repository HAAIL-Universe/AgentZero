## What I Built

**C153: Monte Carlo Methods** -- standalone probabilistic computing, MCMC sampling and integration.

### Components (8):
1. **RandomSampler** -- Base sampling (uniform, normal, exponential, multivariate normal, categorical, beta, gamma, Dirichlet)
2. **MonteCarloIntegrator** -- MC integration with 5 methods (basic, importance sampling, stratified, control variates, antithetic variates)
3. **MetropolisHastings** -- MH sampler (random walk, custom proposals, adaptive scaling)
4. **GibbsSampler** -- Full conditional sampling + block Gibbs
5. **HamiltonianMC** -- HMC with leapfrog integration
6. **NUTS** -- No-U-Turn Sampler (recursive tree-building, auto trajectory length)
7. **ParallelTempering** -- Replica exchange MCMC (temperature ladder, swap proposals)
8. **SliceSampler** -- Auxiliary variable slice sampling (stepping out)
9. **MCMCDiagnostics** -- Convergence diagnostics (ESS, R-hat, split R-hat, Geweke, autocorrelation, summary)

### Key Design Decisions:
- NumPy for all numerical computation (Cholesky, multivariate normal, etc.)
- Samplers return (samples, accept_rate) where applicable
- Unified interface: sample(n_samples, initial, burn_in, thin)
- NUTS uses recursive tree-building with U-turn criterion
- ParallelTempering uses geometric temperature ladder by default
- All samplers support seeded reproducibility

## Reflection

Monte Carlo methods are foundational to probabilistic computing. This challenge covers the major MCMC algorithms:
- MH for general-purpose sampling
- Gibbs for conditionally conjugate models
- HMC/NUTS for efficient high-dimensional sampling
- Parallel tempering for multimodal distributions
- Slice sampling as a tuning-free alternative
- Integration methods with variance reduction

The diagnostic tools (ESS, R-hat, Geweke) complete the picture -- you can't just sample, you need to verify convergence.
