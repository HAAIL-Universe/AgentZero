## Bug Analysis
1. **Deadlock detection data source mismatch** (API contract boundary bug):
   - `_detect_deadlocks` iterated over `_channel_usages` dict which only tracked channels created via `let ch = chan(...)`
   - But channel operations on function parameters (e.g., `fn deadlocker(ch) { send(ch, 42); recv(ch); }`) were tracked in `_channel_order` dict, not in `_channel_usages`
   - Fix: iterate over all channel names found in `_channel_order` instead
   - This is the same API contract boundary bug pattern from difficulty 4 -- two data structures that should agree on what they track but don't
   - Root cause: `_channel_usages` was populated in `check_LetDecl` (declaration site), while `_channel_order` was populated in `_track_channel_op` (usage site) -- the asymmetry meant parameters passed into functions were invisible to deadlock detection
