## What I Did

### 1. Silent Token Truncation Fix (research paper: implemented)
- **Problem:** Local inference path in `agent_zero_inference.py` hardcoded `max_length=2048` in tokenizer calls, silently truncating input to 5% of the model's 40960-token context window.
- **Fix:** Replaced with `input_limit = config.model_context_limit - max_tokens`, using the full context window minus generation budget. Added warning log when truncation occurs.
- **Files:** `agent_zero_inference.py` (lines 259, 405), `test_inference_truncation.py` (10 tests)
- **Impact:** Local inference now uses ~95% of context (38912 tokens) instead of ~5% (2048 tokens).

### 2. Soft-Delete for clear_user_data (research paper: implemented)
- **Problem:** `DELETE /api/user/clear-data` permanently destroyed all user data instantly with no recovery window. A stolen JWT could wipe everything.
- **Fix:** Changed to soft-delete pattern with 48-hour grace period:
  - New `pending_data_deletions` table tracks deletion requests
  - `clear_user_data` now inserts a pending deletion instead of hard DELETE
  - New `POST /api/user/undo-clear-data` endpoint cancels within grace period
  - `purge_user_data()` and `purge_expired_deletions()` handle actual cleanup
  - New `POST /admin/purge-expired-deletions` triggers purge
- **Files:** `database.py` (schema + purge functions), `agent_zero_server.py` (endpoints), `test_soft_delete.py` (17 tests)
