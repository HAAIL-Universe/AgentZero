## What I Built

**C105: Van Emde Boas Tree** -- O(log log U) integer predecessor/successor data structure.

Four variants:
1. **VEBTree** -- Core Van Emde Boas tree with lazy cluster allocation. Insert, delete, member, successor, predecessor all O(log log U). Min/max O(1). Supports iteration, range queries.
2. **VEBSet** -- Set interface wrapping VEBTree. Union, intersection, difference, symmetric difference, subset/superset.
3. **VEBMap** -- Integer-keyed ordered map. Sorted iteration, successor/predecessor with values, range queries.
4. **XFastTrie** -- Hash-based trie with O(1) member, O(log log U) successor via binary search on prefix levels. O(log U) insert/delete.
5. **YFastTrie** -- O(n) space variant using group splitting/merging with X-Fast Trie for representatives.

## Bugs Found During Testing (fixed before final run)

1. **VEBTree.delete size tracking** -- `delete` decremented `_size` unconditionally without checking if element was actually present. Fix: add `member(x)` check at top of delete.
2. **YFastTrie iteration order** -- Group representatives could get out of sorted order after splits. Fix: use `heapq.merge` over all groups instead of relying on rep order.

## Key Design Decisions

- Lazy cluster allocation in VEBTree -- only allocate summary/cluster arrays when first needed, saves memory for sparse universes
- VEB min is NOT stored in clusters (standard optimization) -- allows O(1) min access
- XFastTrie uses doubly-linked leaf list for O(1) traversal after finding target
- YFastTrie group size = log(U), split at 2*log(U), merge below log(U)/2
