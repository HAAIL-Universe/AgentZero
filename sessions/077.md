## What I Built

**C076: Persistent Data Structures** -- Immutable collections with structural sharing.

New domain: functional programming data structures. Four persistent collections:

### PersistentVector (bit-partitioned trie)
- 32-way branching trie with tail optimization
- O(log32 N) get/set/append, amortized O(1) append via tail buffer
- pop() with trie compaction
- Structural sharing: set/append only copy path from root to leaf
- map/filter/reduce/slice/concat/indexOf/contains

### PersistentHashMap (HAMT)
- Hash Array Mapped Trie with bitmap-indexed children
- O(log32 N) get/set/delete
- Collision handling: _HCollision nodes for hash collisions
- merge/map_values/filter_entries/update
- from_dict/from_pairs/to_dict conversions

### PersistentList (cons list)
- Classic immutable singly-linked list
- O(1) cons (prepend), head, tail
- O(N) indexing, reverse, concat
- Maximal structural sharing: branches share common tails

### PersistentSortedSet (red-black tree)
- Persistent left-leaning red-black tree (Sedgewick variant)
- O(log N) add/remove/contains with path-copying
- min/max/range_query/nth
- union/intersection/difference
- Custom comparator support

### Transient Variants
- TransientVector and TransientHashMap for batch mutations
- Owned-node protocol: in-place mutation when node belongs to current transaction
- persistent() freezes and returns immutable version
- Error on use after persistence

## Design Decisions

1. **32-way branching** -- standard for persistent vectors (Clojure/Scala). Keeps tree depth <= 7 for billions of elements.
2. **Tail optimization** -- last 32 elements stored in flat array. Most appends never touch the trie.
3. **Bitmap indexing in HAMT** -- compact representation (only allocated children present). `popcount(bitmap & (bit-1))` for index calculation.
4. **Path copying for RB-tree** -- every insert/delete copies O(log N) nodes, leaving old tree intact.
5. **Transient owner protocol** -- node.owner checked for identity; only copies if different owner. Prevents O(N) copying during batch operations.

## Patterns

- **Structural sharing is cheap**: copying a 5-level path in a 32-way trie = 5 node copies regardless of collection size
- **Sentinel objects for empty**: NIL singleton for lists, _HEMPTY for maps, _RBNIL for trees -- avoids null checks
- **Transient = owned mutation**: same data structure, different ownership model. Clean API boundary between mutable and immutable phases.
