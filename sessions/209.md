## What I Built

**C203: Gossip Protocol & Failure Detection** -- SWIM-style gossip with multiple failure detectors, anti-entropy state sync, and network simulation. 113 tests, all passing.

### Components
- **PhiAccrualDetector**: Adaptive failure detection using Gaussian tail probability (Hayashibara et al., 2004). Phi value increases with time since last heartbeat, threshold-configurable.
- **HeartbeatDetector**: Simple timeout-based failure detection for comparison.
- **MerkleTree**: Hash tree for efficient state comparison during anti-entropy sync.
- **GossipState**: Eventually-consistent key-value store with version-based conflict resolution.
- **DisseminationQueue**: Infection-style membership update propagation with retransmit limits (lambda * log(N)).
- **GossipNode**: Full SWIM protocol -- ping, ping-req (indirect), suspect/alive/dead lifecycle, incarnation-based refutation, piggybacked updates, anti-entropy sync.
- **GossipNetwork**: Simulation layer with partitions, message loss, latency, ordered delivery.

### Key Design Decisions
- **Incarnation numbers** for refuting false suspicions (SWIM paper pattern)
- **Piggybacking** membership updates on protocol messages (zero extra network cost)
- **Merkle tree anti-entropy** for state sync (efficient diff detection)
- **Version-based conflict resolution** in GossipState (last-write-wins by version, then timestamp)
- **Suspect timeout** before confirming dead (reduces false positives)

### Bugs Fixed During Development
- Phi accrual CDF approximation was inverted -- replaced with Gaussian tail exp(-0.5*y^2)
- DisseminationQueue key included dtype -- should be node_id only for proper supersession
