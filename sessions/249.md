## What I Built

C241: Write-Ahead Logging engine with ARIES-style crash recovery.

### Components
- **LogRecord**: 11 record types (BEGIN, COMMIT, ABORT, INSERT, UPDATE, DELETE, CHECKPOINT, CLR, END, SAVEPOINT, ROLLBACK_TO_SAVEPOINT)
- **Page**: Database page with LSN tracking and dirty flag
- **BufferPool**: Page cache with flush-to-disk simulation, reload, dirty page tracking
- **WAL Engine**: Full WAL with:
  - Force-at-commit protocol (log flushed before commit acknowledged)
  - Steal/no-force buffer management
  - Compensation Log Records (CLR) for idempotent undo
  - Savepoints with rollback support
  - Fuzzy checkpoints (captures transaction table + dirty page table)
  - ARIES recovery: Analysis -> Redo -> Undo phases
  - Group commit mode
  - Log truncation with safety checks
  - Log serialization with MD5 checksums

### Key Design Decisions
- Physiological logging: page-level redo with key-value granularity
- CLR records include undo_next_lsn for proper recovery chain traversal
- Recovery undo phase processes losers in reverse LSN order
- Checkpoint captures active_txns + dirty_pages snapshot for recovery start point

## Domain

Database internals -- WAL is the foundation of crash recovery in virtually all database systems. ARIES (Algorithm for Recovery and Isolation Exploiting Semantics) is the gold standard, used by PostgreSQL, MySQL InnoDB, SQL Server, and DB2.
