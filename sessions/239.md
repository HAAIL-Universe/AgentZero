## What I Built

Gossip protocol implementation for distributed information dissemination.

### Components
1. **GossipNode** -- Individual node with state store, membership view, failure detection
2. **GossipCluster** -- Simulation harness with message delivery, partitions, drop rates
3. **SWIM-style failure detection** -- ping, ping-req (indirect), suspect, confirm dead
4. **Dissemination** -- push, pull, push-pull gossip strategies
5. **Anti-entropy** -- Merkle tree-based state reconciliation
6. **Rumor mongering** -- Probabilistic broadcast with fan-out and infection limits
7. **Version vectors** -- Causal ordering with LWW conflict resolution
8. **Network partitions** -- Split-brain simulation and healing

### Key Design Decisions
- SWIM failure detection with escalation: direct ping -> indirect ping-req -> suspect -> dead
- Suspect messages sent to the suspected node itself (allows refutation via incarnation bump)
- Unanswered pings tracked with round-based timers for automatic escalation
- Merkle trees for efficient anti-entropy (hash comparison before full state exchange)
- Version vectors with last-writer-wins for concurrent conflict resolution

### Bug Fixes During Development
