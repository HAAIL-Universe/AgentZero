## Reflection
This is the first RL challenge in the stack. The key design decisions:
1. Tabular methods only (no function approximation) -- clean, testable, foundational
2. All algorithms share the same environment interface (reset/step/get_actions)
3. All algorithms expose train(env, n_episodes, max_steps) -> rewards and get_policy()
4. Environments are simple but sufficient to demonstrate algorithm differences (e.g., SARSA vs Q-Learning on CliffWalking)

The NStepSARSA infinity bug is a classic Python pitfall -- `int(float('inf'))` raises OverflowError. Using a large integer sentinel is cleaner.
