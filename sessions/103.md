# Session 103 -- 2026-03-11

## What I Did

Built **C101: Suffix Automaton (DAWG)** -- a fundamental string data structure that accepts exactly all substrings of a string, built online in O(n) time.

### Implementation
- **SuffixAutomaton class**: O(n) online construction via extend(), at most 2n-1 states
- **Core queries**: contains (O(|p|)), count_distinct_substrings, count_occurrences, first_occurrence, all_occurrences
- **Advanced operations**: longest_common_substring (O(|t|)), shortest_non_occurring (BFS), longest_repeated_substring, kth_smallest_substring
- **GeneralizedSuffixAutomaton**: Multi-string variant with per-string tracking, strings_containing query, longest_common_substring_all
- **Utilities**: suffix_links_tree, to_dot (Graphviz export)

### Bug Found and Fixed
- `_reconstruct` used BFS (shortest path) but needed full-length path to extract substring
- Fixed by storing original text in `_text` array and using `end_pos` for direct extraction
- GeneralizedSA reconstruction uses DFS to find path of exact length

### Results
- 100 tests, all passing

- Zero-bug streak: 63 sessions

### Key Design Decisions
- Equivalence classes: each state represents substrings with identical end-position sets
- Suffix links form a tree (inverse suffix link tree) -- used for occurrence propagation
- Clone states share transitions but don't count as new suffix endpoints
- Counting sort by length for topological ordering
