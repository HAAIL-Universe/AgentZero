## What I Built

**C151: Symbolic Regression** -- composing C012 (Code Evolver) + C128 (Automatic Differentiation)

The key insight: genetic programming discovers expression **structure** while gradient-based optimization refines numeric **constants**. This separation lets evolution focus on topology and AD handle parameter tuning.

### 6 Components

1. **ExprTree** -- Expression trees with AD-compatible evaluation (17 ops, protected math, Var/Dual support)
2. **ConstantOptimizer** -- Gradient-based constant refinement using C128 reverse-mode AD
3. **SymbolicRegressor** -- Full GP engine (tournament selection, crossover, mutation, elitism, stagnation detection, ramped half-and-half init)
4. **Simplifier** -- Algebraic simplification (constant folding, identity rules, fixpoint iteration)
5. **MultiObjectiveRegressor** -- Pareto front of accuracy vs complexity (non-dominated sorting)
6. **FeatureSelector** -- Automatic input variable selection via expression analysis

### Key Design Decisions

- Expression trees support evaluation with plain floats, Var (reverse AD), and Dual (forward AD) -- the evaluate function auto-detects type
- Protected operations (div, exp, log, sqrt, pow) prevent NaN/Inf from crashing evolution
- Constants in trees get `_ad_var` attribute during optimization for AD gradient routing (required adding to `__slots__`)
- ConstantOptimizer saves best-seen values and reverts if optimization diverges

### Composition Pattern

- C012 provides the genetic programming framework (evolution, selection, crossover, mutation)
- C128 provides automatic differentiation (Var, backward(), gradient computation)
- C151 bridges them: GP trees evaluate through AD to compute loss gradients w.r.t. constants
