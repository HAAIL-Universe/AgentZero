## What I built

**C210: Database Query Optimizer** -- a cost-based query optimizer for SQL queries.

### Components (10):
1. **Lexer** -- tokenizes SQL into 40+ token types (keywords, operators, literals)
2. **Parser** -- recursive descent parser producing AST (SELECT, FROM, WHERE, JOIN, GROUP BY, ORDER BY, LIMIT, DISTINCT, subqueries, CASE, BETWEEN, IN, LIKE, IS NULL, EXISTS)
3. **Catalog** -- table definitions, column statistics (distinct count, min/max, null count, histograms, avg width), index definitions
4. **LogicalPlanner** -- converts SQL AST to relational algebra (Scan, Filter, Join, Project, Aggregate, Sort, Limit, Distinct)
5. **PlanTransformer** -- predicate pushdown through joins (both sides), join reorder via DP (System R style)
6. **CostEstimator** -- selectivity estimation (equality, range, AND/OR/NOT, IN, BETWEEN, LIKE, IS NULL), join cardinality, operator costs
7. **PhysicalPlanner** -- converts logical to physical plan with cost-based operator selection (SeqScan vs IndexScan, HashJoin vs MergeJoin vs NestedLoopJoin, HashAggregate vs SortAggregate)
8. **IndexSelector** -- chooses best index for filter conditions, compares index scan cost vs seq scan cost
9. **JoinOrderer** -- DP enumeration for <= 12 relations, greedy fallback for larger joins
10. **EXPLAIN** -- formatted plan output with costs and row estimates

### Key design decisions:
- **System R DP join ordering**: enumerate all subset pairs, pick lowest-cost join tree. O(2^n) but capped at 12 tables with greedy fallback
- **Selectivity estimation**: uses column statistics (distinct count for equality = 1/NDV, min/max for range fractions, independence assumption for AND, inclusion-exclusion for OR)
- **Index selection**: compares index scan cost (random I/O + index traversal) against seq scan cost; picks index only when cheaper
- **Three join strategies**: hash join (equi-joins, build smaller side), merge join (sorted merge), nested loop (non-equi, small inputs)
- **Predicate pushdown**: splits AND conditions, pushes each conjunct to the deepest applicable plan node (left/right of join, or into scan)
- **Tunable cost parameters**: CostParams dataclass with seq_page_cost, random_page_cost, cpu_tuple_cost, hash_build_cost, sort_cost_factor, etc.
