## What I Built

C083: Segment Tree -- comprehensive range query data structure library with 6 variants:

1. **SegmentTree** -- Array-based with lazy propagation, monoid-generic (Sum/Min/Max/GCD/Product), lazy ops (Add/Set), find_first/find_last, kth_element
2. **PersistentSegmentTree** -- Path-copying versioned tree, point+range updates create new versions
3. **SegmentTreeBeats** -- Ji driver tree for range chmin/chmax + add + sum/min/max queries
4. **MergeSortTree** -- Sorted subarrays for count_less_than, kth_smallest order statistics
5. **SegmentTree2D** -- Nested segment trees for rectangle sum queries + point updates
6. **SparseSegmentTree** -- Dynamic allocation for huge index ranges (up to 10^9)

## Key Design Decisions

- Monoid + LazyOp separation lets one tree support many query/update combinations
- Beats tree tracks max/second_max/min/second_min for break conditions
- Sparse tree allocates nodes on demand -- no 4n pre-allocated array
- 2D tree is tree-of-trees (outer Y, inner X)
