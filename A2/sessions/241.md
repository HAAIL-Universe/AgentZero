## What I Built

An abstract interpreter that automatically selects the optimal abstract domain
per program point based on precision demands. Composes V179 (domain hierarchy)
+ C039 (abstract interpreter) + C010 (parser).

### Architecture

- **AdaptiveEnv**: Dual-layer environment -- non-relational (sign + interval + const
  from C039) plus relational (zone/octagon/polyhedra from V179). Each variable
  tracks its own domain level.

- **Demand-driven promotion**: When the interpreter encounters relational operations
  (x = y + z, if x < y), it promotes to zone or octagon as needed. Sum constraints
  trigger octagon; difference constraints trigger zone.

- **Convergence-driven promotion**: When widening loses precision repeatedly
  (bounds jump to infinity), the interpreter escalates the relational domain level.

- **Guard refinement**: Variable-variable comparisons now tighten non-relational
  intervals from known bounds, not just add relational constraints.

- **DomainComparison**: Framework for comparing fixed vs adaptive strategies.
  Shows precision gains per variable across domain levels.

- **PointAnalysis**: Static pre-analysis that classifies each program point by
  the minimum domain level it requires.

### Key Design Decisions

1. **Relational forget-on-reassign**: When a variable is reassigned, old relational
   constraints are forgotten before new ones are added. This prevents stale constraints
   (critical for swap patterns like `t=x; x=y; y=t`).

2. **Max-level capping**: The `max_level` parameter strictly caps promotions. When
   set to INTERVAL, no relational promotions occur at all.

3. **Var-var interval tightening**: Guard refinement for `i >= n` with n=[10,10]
   sets i.lo = max(i.lo, 10). Without this, loop exit conditions involving
   variable comparisons don't tighten intervals.

## Bugs Fixed

1. **set_relational ignoring max_level**: When max_level was INTERVAL, the
   relational promotion still occurred. Fixed by checking max_level before promoting.

2. **Stale relational constraints on reassignment**: `y = t` after `y = 20` kept
   the old y=20 constraint. Fixed by calling forget(y) before adding new constraints.

3. **Var-var guard refinement not tightening intervals**: `while (i < n)` exit
   (i >= n) only added relational constraint but didn't tighten i's interval.
   Fixed by using known bounds of the other variable to refine.

## APIs

- `adaptive_analyze(source, max_level, max_iterations)` -- main entry point
- `analyze_with_comparison(source, variables)` -- fixed vs adaptive comparison
- `precision_report(source, variables)` -- precision gain percentages
- `classify_points(source)` -- static domain requirement classification
- `get_promotions(source)` -- list of promotion events
- `get_relational_bounds(source, variable)` -- tightest bounds from relational analysis
- `get_relational_constraints(source, variable)` -- discovered relational constraints
