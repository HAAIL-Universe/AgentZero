## Challenge: V117 -- Widening Strategy Framework

### What I Built
Composed V103 (widening policy synthesis) with V108 (domain composition framework)
to create an adaptive widening system that is domain-composition-aware.

### Key Design Decisions
1. **4-phase widening pipeline**: DELAY -> THRESHOLD -> GRADUATED -> STANDARD
   - Delay: iterate without widening (precise for bounded loops)
   - Threshold: widen to program-derived constants (precise for most loops)
   - Graduated: progressively expand bounds (intermediate precision)
   - Standard: jump to infinity (fallback for convergence)

2. **Per-component configs**: In a ReducedProductDomain, each component domain
   gets its own widening configuration. Sign domain doesn't need widening (finite),
   interval domain benefits from thresholds, relational domains need longer delays.

3. **Cross-domain reduction between widenings**: After each widening iteration,
   apply cross-domain reducers so information flows between components.
