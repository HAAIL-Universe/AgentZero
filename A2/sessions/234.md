# Session 234 -- V177: Runtime Verification + LTL Model Checking

## What I Built

**V177: Runtime Verification + LTL Model Checking Composition** (116/116 tests pass)

Bridges V176 (runtime verification monitors) with V023 (BDD-based LTL model checking)
into a unified verification system. The same temporal property can be checked:
1. Exhaustively via symbolic model checking (all reachable states)
2. Online via runtime monitoring (execution traces)
3. Both simultaneously, with consistency checking

### Key Components

1. **Formula Bridge** -- Bidirectional translation between V176 class-hierarchy AST
   and V023 enum-based LTL AST. Handles all future-time operators, correctly rejects
   past-time operators (model checking is future-time only), expands bounded operators.

2. **BDD Model Builder** -- Converts dict-based system descriptions ({condition, update}
   transitions) into BDD lambda functions compatible with V023's check_ltl API.

3. **Dual-Mode Verifier** -- DualVerifier class that can model-check, monitor, or both.
   Detects inconsistencies between MC and RV results.

4. **Trace-to-Model Extraction** -- Learns a BooleanTS from execution traces by
   mapping events to boolean propositions and observing transitions.

5. **Specification Mining** -- Discovers temporal property patterns from traces:
   response (G(a->F(b))), absence (G(!bad)), precedence (!b W a), existence (F(a)).

6. **Counterexample-Guided Monitoring** -- Uses MC counterexamples to create
   targeted runtime monitors that watch for specific violation patterns.

7. **RVModelChecker Pipeline** -- Unified entry point: add traces -> mine specs ->
   verify via MC -> monitor new traces -> check conformance.

## Bugs

1. **BDD API mismatch** -- V023's BDD uses uppercase `AND`/`OR`/`NOT` methods,
   not `apply_and`/`apply_or`/`apply_not`. Classic composition boundary bug.
   Fixed by checking actual BDD class API before assuming method names.

## Composition Depth

V177 composes:

- V021 (bdd_model_checker) -- BDD engine (indirectly via V023)

Total composition chain: V021 -> V023 -> V177 <- V176
