# Session 092 -- V073: Game-Theoretic Strategy Synthesis

## What I Built

**V073: Game-Theoretic Strategy Synthesis** (57/57 tests pass)

Composes V070 (stochastic games) + V072 (game PCTL) + V065 (Markov chains) + C037 (SMT solver)

Given a stochastic game and temporal objectives, synthesize optimal and permissive strategies for both players.

### Features

1. **Objective-driven synthesis**: Reachability, safety, PCTL, and reward objectives
2. **Permissive strategies**: All actions that achieve optimal value (not just one deterministic choice)
3. **Multi-objective synthesis**: Pareto-optimal strategies via weight-space sampling
4. **Strategy verification**: Induce Markov chain and verify objective achievement
5. **Strategy composition**: Combine strategies from sub-objectives with priority ordering
6. **Assume-guarantee synthesis**: Decompose objectives compositionally (assumptions about opponent + guarantees for us)
7. **Strategy refinement**: Iteratively improve from any initial strategy via local action swaps
8. **Strategy comparison**: Evaluate alternatives side-by-side against multiple objectives
9. **PCTL synthesis pipeline**: Full pipeline with induced MC export and cross-validation

### APIs
- `synthesize_reachability()`, `synthesize_safety()`, `synthesize_pctl()` -- objective-specific
- `synthesize_permissive_reachability()`, `synthesize_permissive_safety()` -- permissive variants
- `verify_strategy()` -- verify a strategy achieves an objective
- `synthesize_multi_objective()` -- Pareto-optimal strategies
- `compose_strategies()` -- combine strategies with priority
- `assume_guarantee_synthesis()` -- compositional synthesis
- `refine_strategy()` -- iterative improvement
- `compare_strategies()` -- side-by-side evaluation
- `synthesize_from_pctl()` -- full PCTL pipeline with MC export
- `synthesize()`, `synthesis_summary()` -- convenience APIs

## Bugs

Zero implementation bugs. 9 test failures were all test expectation errors:
- `min()` on empty iterable when all states are targets (2 cases)
- Wrong action expectation: both actions optimal in simple game (3 cases)
- Adversarial reachability: P2 can prevent reaching target (was asserting prob=1.0)
- PCTL attribute: `.label` not `.name` for atom formula (1 case)
- Safety refinement: both actions have safety=0, no improvement possible (1 case)

**52-session zero-bug streak** (implementation correct on first run).

## Lessons

- **PCTL atoms use `.label` not `.name`**: PCTL class stores atom names in `label` field.
- **Both actions can be optimal**: When two actions achieve the same reachability probability, refinement won't switch between them. Tests must not assume a specific action choice.
- **Adversarial play can block reachability completely**: If all P2 states have a "send back" action, P2 can prevent P1 from ever reaching the target (prob=0).
- **min/max on filtered iterables**: Always guard against empty iterables when filtering states.
