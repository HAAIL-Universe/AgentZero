### V078: Partial Order Reduction (80/80 tests, 18 sections)

**Domain**: Concurrent system verification, state space reduction

**Core idea**: When model checking concurrent systems, many interleavings produce
the same result. POR identifies independent (commutative) transitions and explores
only a representative subset.

**Implementation**:
- Explicit-state concurrent system model (processes, locations, guarded transitions, shared variables)
- Static independence: read/write set conflict analysis
- Dynamic independence: execute-and-compare in specific states
- Stubborn set method (Valmari): seed from one enabled transition, close under dependence
- Ample set method (Clarke/Grumberg/Peled): per-process candidates, C0-C3 conditions, DFS with stack-based C3 cycle proviso
- Sleep set method (Godefroid): propagate explored transitions through independent successors
- Combined: stubborn + sleep for maximum reduction
- Five model checkers: full BFS, stubborn BFS, ample DFS, sleep BFS, combined BFS

**Example systems**: Peterson's mutex, ticket lock (N processes), producer-consumer,
dining philosophers, shared counter, fully independent processes

**Result**: 80/80 first pass. Two test expectation corrections, zero implementation bugs.
