2. **Almost-sure solver**: Iterative refinement -- Zielonka + RANDOM closure check + Odd attractor removal
3. **Positive-probability solver**: Deterministic Zielonka with RANDOM=EVEN conversion

5. **Game constructors**: Buchi, reachability, safety variants with stochastic vertices
6. **Verification**: Strategy verification, simulation, comparison with deterministic

### Key Insight

Zielonka's recursive decomposition cannot be directly applied to almost-sure stochastic parity games. When Zielonka creates a subgame by removing vertices, it also removes edges from RANDOM vertices that lead outside the subgame. This is incorrect for almost-sure analysis because a RANDOM vertex in a cycle will follow every positive-probability edge infinitely often (Borel-Cantelli lemma). The play WILL escape through any positive-prob edge that leaves the winning region.

**Fix**: Iterative refinement algorithm:
1. Solve deterministic parity on subgame (RANDOM=EVEN)
2. Check RANDOM closure: all positive-prob successors (in ORIGINAL game) in win_even
3. If closure fails: remove bad RANDOM vertices + compute Odd attractor
4. Repeat until stable
