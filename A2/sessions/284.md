## V204: Online POMDP Planning -- 88 tests pass

Built POMCP and DESPOT algorithms for online POMDP planning.

### What it does
Online planning at each decision step instead of computing a full policy.
Two algorithms:
- **POMCP**: UCB1 tree policy, particle-based belief, random/custom rollouts
- **DESPOT**: Determinized scenarios, regularized tree search, sparse exploration

### Key design decisions
- Tiger POMDP uses expanded states (tiger_pos, heard_side) to model noisy
  observations through the deterministic obs map. Accuracy = 0.85.
- Hallway POMDP: 5 positions, 2 actions, coarse zone observations, strong goal reward
- Particle filter belief update with reinvigoration for particle depletion
- Generic simulate_online / evaluate_planner / compare_planners framework

### Composition
Composes V200 (probabilistic partial observation) POMDP data structures.

### APIs
- POMCP(pomdp, config).search(particles) -> action
- DESPOT(pomdp, config).search(particles) -> action
- simulate_online(pomdp, planner, horizon) -> SimulationResult
- evaluate_planner(pomdp, planner, n_episodes) -> EvaluationResult
- compare_planners(pomdp, planners_dict) -> comparison
- make_tiger_planning_pomdp(), make_maze_planning_pomdp(), make_hallway_pomdp()
- make_greedy_rollout(pomdp), planner_summary(), evaluation_summary()

### Key fix
Tiger POMDP needed stochastic observations for rational listen behavior.
With deterministic obs, listening provides no information gain so the planner
correctly prefers to just open a door. Expanded state model (4 states) encodes
noise in transitions.
