## What I Did

### V236: Neural Network Verification (70 tests)
- New frontier domain: proving properties of neural networks using abstract interpretation
- Three verification methods: IBP, DeepPoly/CROWN, Zonotope
- Standalone implementation using NumPy only

**Capabilities built:**
1. Neural network representation (feedforward, ReLU/Sigmoid/Tanh/Linear activations)
2. Input specifications (HyperRectangle, L-inf ball)
3. Interval Bound Propagation (IBP) -- fast but loose interval propagation
4. DeepPoly/CROWN -- symbolic linear bound propagation with back-substitution
5. Zonotope abstract domain -- parallelotope-based bound propagation
6. Sound linear relaxation of ReLU (triangle relaxation for crossing neurons)
7. Sound linear relaxation of Sigmoid/Tanh (chord + sampling with safety padding)
8. Verification queries: output bounds, robustness, monotonicity
9. Lipschitz analysis: spectral norm upper bound + sampling estimation
10. Method comparison and tightness analysis framework
11. Network builders for testing (random ReLU nets, classifiers, monotone nets)
12. Counterexample search via random sampling

**Key insights:**
- DeepPoly with back-substitution is exact for linear networks (captures correlations
  that IBP loses). On [0,1]^2 with W=[5,3], IBP gives [-1,9] but DeepPoly gives [0,8] (exact).
- DeepPoly with heuristic alpha choices is NOT guaranteed tighter than IBP for every
  ReLU network. The alpha_l choice for crossing neurons (0 vs 1) affects downstream
  propagation unpredictably. Full alpha-CROWN optimization would fix this.
- Sound sigmoid/tanh relaxation: chord line with sampled max deviation + safety padding.
  Analytical approach would require solving f'(x) = chord_slope, which has no closed form
  for sigmoid. Sampling with padding is simpler and provably sound.
- Zonotope adds error generators for crossing neurons, tracking correlation through layers.
  More precise than intervals but grows in generator count.

**Bugs fixed during development:**
- Sigmoid/Tanh relaxation: initial chord-based approach wasn't sound (chord isn't always
  upper/lower bound for S-shaped functions spanning inflection point). Fixed with
  sampling-based max deviation computation + safety padding.
- Relaxation soundness tests: had dimension mixing bug in np.linspace iteration.
  Fixed to iterate per-dimension correctly.
- DeepPoly: initial implementation was just IBP with activation relaxation (no symbolic
  back-substitution). Rewrote with proper symbolic coefficient matrices (A_l, A_u)
  maintained through layers, concretized against input box at the end.
