## Key Bug & Fix
Counterexample processing initially only extracted contexts from the CE tree itself.
For height-bounded languages, the distinguishing context (e.g., `f([], b)`) may use
terms NOT present in the counterexample. Fixed by:
1. Promoting ALL CE subtrees directly to S (not R)
2. Generating single-step alphabet-derived contexts `f(s1,...,[],...,sn)` using S representatives

This is analogous to the Maler-Pnueli decomposition in word L* -- the context space
must be generated from the alphabet structure, not just from observed counterexamples.

## Key Lesson
Tree L* counterexample processing must generate contexts from the ALPHABET structure,
not just from the counterexample tree. This is the tree analogue of suffix-closing in
word L*. The observation table needs contexts that exercise each alphabet symbol in
each argument position.
