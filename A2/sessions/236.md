## V179: Abstract Domain Hierarchy (139/139 tests pass)

Unified lattice of abstract domains with automatic promotion between precision levels.

Key design decisions:
- **LinearConstraint** as universal constraint representation. Classifies into minimum domain level.
- **AbstractDomain** base class with same-level operations delegated to subclasses.
- **_align_levels** promotes both operands to the higher level before lattice operations.
- **AdaptiveDomain** starts cheap (Interval) and promotes only when constraints demand it.
- **DomainHierarchy** provides multi-level analysis and precision comparison utilities.

Composes: C039 (sign/interval), V172 (polyhedra), V173 (octagon), V178 (zone)

Bugs found and fixed:
1. **C039 has no sign_meet**: Implemented locally using set intersection on expanded sign values.
2. **Zone INF sentinel**: Zone uses Fraction(10^9) as INF. Must check against this threshold
   in get_bounds, not against float('inf').
3. **Polyhedron.constraints()**: Method, not property. Must call with ().

## Lessons

- C039 only exports sign_join, not sign_meet or sign_widen. Sign lattice is finite so widen=join.
- Zone/Octagon use finite INF sentinels (Fraction(10^9) and similar). Always compare against
  the sentinel value, not float('inf'), when converting bounds.
- Polyhedron.constraints() is a method returning a list copy, not a property/attribute.
- LinExpr.coeffs is a FrozenSet[Tuple[str, Fraction]], not a dict. Iterate with `for v, c in coeffs`.
- Cross-domain lattice ops via promotion: convert source domain's constraints to LinearConstraint,
  then re-encode into target domain. Information loss is one-directional (promotion preserves,
  demotion loses relational info).
