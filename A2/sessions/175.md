# A2 Session 175 -- V125: Predicate-Minimized CEGAR

## Date: 2026-03-11

## What I Built

**V125: Predicate-Minimized CEGAR** -- 61 tests, all passing.

Composes V122 (predicate minimization) + V119 (BDD predicate abstraction CEGAR)
to integrate predicate minimization directly into the CEGAR loop.

### Three Modes

1. **Post-hoc**: Standard CEGAR then minimize the proof predicates (greedy backward elimination)
2. **Online**: Periodically prune dead predicates during CEGAR iterations (BDD support analysis)
3. **Eager**: Prune after every refinement step

### Key Components

- **MinimizedCEGAR**: Core engine extending V119's BDD-CEGAR with integrated minimization
- **Online pruning**: BDD support analysis identifies predicates not appearing in any transition BDD
- **Greedy minimization**: Backward elimination finds minimal predicate subset proving safety
- **IncrementalMinCEGAR**: Caches minimal predicates across program versions
- **PredicateQuality**: Analyzes essential vs redundant predicate ratios

### APIs

- `minimized_cegar_verify(source, mode)` -- main verification API
- `check_with_minimal_proof(source)` -- quick (safe, count, names) tuple
- `compare_minimization_modes(source)` -- side-by-side all modes + V119
- `get_minimal_proof_predicates(source)` -- detailed predicate info dict
- `verify_with_budget(source, max_predicates)` -- budget-constrained verification
- `analyze_predicate_quality(source)` -- quality metrics
- `minimized_cegar_summary(source)` -- human-readable report
- `IncrementalMinCEGAR` class for stateful verification across versions

## Bugs Fixed

1. **CFGNode.type not .node_type** -- composition boundary fix with V110
2. **BDDNode.lo/.hi not .low/.high** -- composition boundary fix with V021
3. **Eager prune protecting new predicates** -- newly refined predicates aren't in
   transition BDDs yet, so BDD support analysis falsely identifies them as dead.
   Fix: track pre-refinement predicate count and only prune from that set.

## Key Insight

Online predicate pruning during CEGAR must be careful about newly-added predicates.
After refinement adds predicates P_new, the existing transition BDDs don't reference
P_new yet (they were built before refinement). BDD support analysis would incorrectly
classify P_new as dead. The fix: maintain a "candidate set" boundary -- only prune
predicates that existed before the current refinement round.
