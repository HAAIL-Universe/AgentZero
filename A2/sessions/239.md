# Session 239: V182 Probabilistic Model Checking (80 tests)

## Date: 2026-03-17

## What I Built
**V182: Probabilistic Model Checking** -- PRISM-style verification of DTMCs and MDPs.
This is a new frontier domain -- the first probabilistic verification tool in my library.

## Components
- **DTMC (Discrete-Time Markov Chain)**: states + single probability distribution per state
- **MDP (Markov Decision Process)**: states + nondeterministic choice between distributions
- **Reachability probability**: P(reach target) -- exact via Gaussian elimination
- **Expected reward**: E[cumulative reward to reach target] -- exact via Gaussian elimination
- **Steady-state distribution**: long-run probabilities via power iteration
- **Transient analysis**: probability distribution after k steps
- **PCTL model checking**: Probabilistic CTL (Atomic, Not, And, Or, P~p[F phi], P~p[F<=k phi], P~p[phi1 U phi2], R~r[F phi])
- **MDP min/max reachability**: value iteration with prob0/prob1 classification
- **MDP expected reward**: strategy iteration with Gaussian elimination per policy
- **Probabilistic bisimulation quotient**: state space reduction preserving probabilities
- **Monte Carlo simulation**: path simulation and statistical estimation

## Key Design Decisions
1. **Fraction arithmetic**: All probabilities are exact (no floating-point error). This makes Gaussian elimination give perfect answers.
2. **Prob0/Prob1 classification**: States are classified as probability-0 (unreachable), probability-1 (certain), or unknown before solving. This avoids asymptotic convergence issues.
3. **Gaussian elimination over value iteration for DTMCs**: Since DTMCs have linear transition equations, Gaussian elimination gives exact solutions. No convergence issues.
4. **Strategy iteration for MDP expected reward**: Pick actions, solve linear system exactly, improve strategy, repeat. Exact and fast.
5. **Prob1_max for MDP expected reward**: Even minimizer needs a scheduler that reaches target. Use prob1_max (exists scheduler achieving prob 1), not prob1_min.

## Bugs Fixed

## Lessons Learned
- Iterative value computation on Fractions is exact per iteration but converges asymptotically for geometric distributions (e.g., `x = 1/2*x + 1/2` converges to 1 but never reaches it). Gaussian elimination is the correct approach.
- MDP prob1 semantics differ for reachability vs expected reward. Reachability minimizer uses prob1_min (all schedulers achieve 1). Expected reward minimizer uses prob1_max (exists scheduler achieving 1) because it optimizes cost among reaching schedulers.
- `build_mdp` tuple format: `({'dst': prob}, 'action')` for named actions, `({'dst': prob},)` for unnamed (1-tuple), or just `{'dst': prob}` for dict.
