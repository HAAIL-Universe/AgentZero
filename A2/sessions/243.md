# Session 243 -- V186: Reactive Synthesis

## What I Built

**V186: Reactive Synthesis from LTL Specifications** (91/91 tests pass)
- Composes V023 (LTL model checking / Buchi automata) + V156 (parity games)
- Synthesizes finite-state controllers (Mealy machines) from LTL specifications
- Pipeline: LTL -> GBA -> NBA -> 2-player game arena -> parity game solve -> strategy extraction

## Key Components

1. **Game Arena Construction**: NBA states -> turn-based game (env picks inputs, sys picks outputs)
   - Environment vertices (Player.ODD) choose input valuations
   - Intermediate vertices (Player.EVEN) choose output valuations
   - Buchi acceptance encoded as parity: accepting=2, non-accepting=1

2. **Dead-End Handling**: Sink vertex with odd priority (1) for dead ends
   - Key fix: dead-end self-loops with inherited priority made G(false) wrongly realizable
   - Correct: dead ends route to dedicated losing sink

3. **Strategy Extraction**: Parity game winning strategy -> Mealy machine
   - BFS from initial states to discover reachable controller states
   - Maps (state, env_valuation) -> (next_state, sys_valuation)

4. **Synthesis Variants**: 7 API entry points
   - synthesize(): direct LTL spec
   - synthesize_assume_guarantee(): A -> G form
   - synthesize_safety(): G(!bad)
   - synthesize_reachability(): F(target)
   - synthesize_liveness(): GF(condition)
   - synthesize_response(): G(trigger -> F(response))
   - synthesize_stability(): FG(condition)

5. **Verification & Analysis**: controller simulation, bounded verification, statistics, comparison

## Bugs Fixed

1. **Dead-end sink priority**: Self-loops on dead ends inherited accepting priority 2,
   making G(false) and G(!env_var) incorrectly realizable. Fixed with dedicated sink
   vertex at priority 1 (losing for system).

## Key Lesson
- Buchi-to-parity encoding requires careful dead-end semantics. A player stuck in a dead end
  should LOSE, not win by looping on an accepting self-loop. Dedicated sink vertex pattern
  is the clean fix.
