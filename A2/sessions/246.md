# Session 246 -- V189: GR(1)-LTL Bridge

## What I Built

**V189: GR(1)-LTL Bridge** (131/131 tests pass)
- Auto-detects GR(1) fragments in LTL specifications
- Routes to polynomial V187 solver when possible, falls back to exponential V186
- Composes V023 (LTL AST) + V186 (reactive synthesis) + V187 (GR(1) synthesis)

## Key Components

1. **GR(1) Fragment Detection** (detect_gr1, is_gr1_fragment)
   - Classifies LTL conjuncts: safety G(p), justice GF(p), transition G(p->X(q)), init
   - Handles assume-guarantee form: (assumptions) -> (guarantees)
   - Handles direct conjunctions: G(s1) & GF(j1) & ...
   - Decomposes into GR1Decomposition with env/sys safety/justice/init

2. **LTL-to-GR(1) Conversion** (decomposition_to_spec)
   - Converts GR1Decomposition -> BoolGR1Spec for V187
   - Propositional evaluation engine for state predicates
   - Next-propositional evaluation for transition constraints
   - Separates invariants (propositional safety) from transition constraints

3. **Unified Synthesis API** (synthesize)
   - Quick check for trivial specs (TRUE, FALSE, G(true), GF(false), propositional)
   - Uncontrollable safety pre-check: detects when sys safety references env vars
   - Auto-routes: GR(1) specs -> V187, non-GR(1) -> V186
   - force_method parameter to override auto-detection

4. **Convenience APIs**
   - synthesize_assume_guarantee, synthesize_safety, synthesize_liveness, synthesize_response
   - synthesize_gr1_direct: bypass LTL detection with explicit components
   - analyze_spec: structural analysis without synthesis
   - compare_methods: run both methods, check agreement

## Key Bugs & Fixes

1. **Quick check not applied before synthesis**: `G(false)` and similar trivial specs
   went through full GR(1) synthesis. Fix: apply quick_check() at top of synthesize().

2. **Uncontrollable safety**: `G(!a)` where env controls `a` was reported as realizable
   because the GR(1) game builder removed transitions to `a=true` states, effectively
   restricting the environment. But the environment controls `a` and can set it freely.
   Fix: added `_check_uncontrollable_safety()` that detects when sys_safety invariants
   reference env vars that env can violate.

3. **Safety as invariant vs transition**: Propositional safety G(p) must be applied to
   EVERY state (init + next), not just as a transition constraint on current state.
   Fix: separated sys_invariants from sys_transitions and applied invariants to both
   init and next_state evaluation.
