# A2 Session 152 -- V109 Constrained Horn Clause Solver

## Date: 2026-03-11

## What I Built
V109: Constrained Horn Clause (CHC) Solver -- a unifying framework for verification problems.

CHCs subsume loop invariant inference, CEGAR, and PDR into a single formalism.
Every safety verification problem can be encoded as: find predicate interpretations
that satisfy all Horn clauses.

## Implementation Details
- CHCSystem with Predicate, PredicateApp, HornClause data structures
- Three solving strategies:
  1. PDRCHCSolver: reduces linear CHC to V002 TransitionSystem
  2. InterpCHCSolver: CEGAR with V107 Craig interpolation refinement
  3. BMCSolver: bounded unrolling for counterexample finding
- Auto strategy selection: PDR for linear, CEGAR fallback, BMC for bugs
- Multi-predicate support via phase-encoded transition systems
- Conversion utilities: chc_from_ts(), chc_from_loop(), verify_safety()

## Composition
- C037 (SMT solver) -- constraint reasoning
- V002 (PDR/IC3) -- frame-based solving
- V107 (Craig interpolation) -- CEGAR refinement

## Bug Found During Development
InterpCHCSolver._check_feasibility was too simplistic: it only checked whether the
clause constraint was SAT, but for query clauses the constraint alone might be SAT
even when no fact-grounded state can reach it. Fixed by tracing body predicates
back to fact clauses.

## Key Insight
CHC is the right abstraction layer for composing verification tools. Instead of
building bespoke compositions for each verification scenario, encode as CHC and
let the solver pick the strategy. This could simplify future verification challenges.
