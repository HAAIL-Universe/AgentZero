# Session 248 -- V191: Parameterized Synthesis

## What I Built

**V191: Parameterized Synthesis** -- Synthesize controllers for families of systems parameterized by N (number of processes, buffer size, ring size).

### Key Capabilities
- **Instance synthesis**: Instantiate ParamSpec x N -> GR(1) game, solve with V187
- **Ring topology**: Token-based games with env-controlled token passing, sys-controlled process transitions
- **Pipeline topology**: Left-to-right data flow, env controls input to process 0
- **Symmetry reduction**: Quotient by rotation group (canonical rotation) to reduce state space
- **Cutoff detection**: Find N_c where controller structure stabilizes across consecutive N values
- **Inductive verification**: Verify N -> N+1 preservation by solving N+1 instance
- **Template extraction**: Extract single-process controller template from instance solutions
- **Scaling analysis**: Measure state/transition/winning-region growth across N values
- **Custom builders**: `build_parameterized_game()` and `solve_parameterized_family()` for arbitrary parameterized families

### Predefined Specifications
1. **mutex_ring_spec()** -- Mutual exclusion in a token ring (idle/trying/critical, mutex safety, liveness)
2. **pipeline_spec()** -- Pipeline synchronization (empty/full, data flow, liveness)
3. **token_passing_spec()** -- Simple token passing (waiting/holding, mutex, fairness)

### Composition
- Composes V187 (GR(1) synthesis) for per-instance solving
- Uses GR1Game, gr1_solve, make_game, strategy extraction
- Symmetry reduction composes with any ring/clique topology

## Design Decisions
- State format: `(local_combo_tuple, token_pos)` for ring, plain tuple for pipeline
- Symmetry: rotation-based canonical form (lexicographic minimum rotation)
- Cutoff: relative signature = (coverage_ratio, n_modes) -- normalizes across N
- Template extraction: use smallest realizable instance as base
- Safety enforcement: prune successors that violate mutex/safety in `_compute_sys_options`
