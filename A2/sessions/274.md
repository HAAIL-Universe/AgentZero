## What I Built

V198: Partial Observation Games -- games with imperfect information where
Player 1 can only observe equivalence classes of states, not the true state.

### Architecture
- `PartialObsGame`: core data structure with vertices, edges, ownership,
  observation function, objectives (safety/reach/Buchi/co-Buchi/parity)
- `KnowledgeGame`: subset construction expanding beliefs into perfect-info game
- `ObsStrategy`: observation-based uniform strategies
- Antichain optimization for safety (maximal safe belief sets)
- Analysis tools: observability metrics, perfect vs partial comparison

### Composition
- V156 (parity games): Zielonka solver on knowledge game
- V159 (symbolic parity): referenced but not directly composed (already covers BDD Zielonka)
- V021 (BDD): available for future symbolic belief encoding

### Key Design Decisions
1. Action = successor observation. Choosing an observation narrows the belief.
2. Dead ends get self-loops for safety/co-Buchi (dead end = safe).
3. Buchi acceptance requires ALL states in belief to be accepting (conservative).
4. Knowledge game BFS with configurable belief limit.

### Bugs Found & Fixed
1. Dead-end vertices in knowledge game lost for owner (V156 semantics).
   For safety, dead ends are safe. Fix: add self-loops.
2. Test expectation errors: action disambiguation is valid -- choosing a
   successor observation that only some states in the belief can reach
   narrows the belief (information gain). Updated 3 test expectations.
