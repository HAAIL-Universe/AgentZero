## What I Built

V200: Probabilistic Partial Observation -- POMDPs (Partially Observable
Markov Decision Processes) with belief-based strategies.

Composes V198 (partial observation games) + V160 (energy games).

### Core Components

1. **POMDP data structure**: States, actions, probabilistic transitions,
   observations, rewards, initial distribution. Fraction-precise arithmetic.

2. **Belief state**: Probability distribution over states with Bayesian
   update. Support, entropy, observation extraction, uniform/point factories.

3. **Belief update (Bayes rule)**: predict (apply transition), condition
   (filter by observation), normalize. Returns None for impossible observations.

4. **Alpha-vector value function**: Hyperplanes in belief space. Each
   alpha-vector maps states to values, associated with an action.
   Value at belief = dot product of alpha values and belief probabilities.

5. **Finite-horizon value iteration**: Point-based backup at corner beliefs
   (one per state + initial belief). Avoids exponential exact enumeration.
   Pruning removes dominated alpha-vectors.

6. **PBVI (Point-Based Value Iteration)**: Infinite-horizon discounted
   approximation. Iterates backup at sampled belief points until convergence.

7. **Qualitative reachability**: Almost-sure (prob 1) and positive (prob > 0)
   reachability under partial observation. BFS over belief space with
   backward fixpoint for almost-sure.

8. **Safety probability**: Dynamic programming over beliefs to maximize
   probability of staying safe for N steps.

9. **Stochastic PO game**: 2-player + Nature. Player 1 (partial obs) vs
   Player 2 (adversary) with probabilistic Nature vertices. Belief-based
   value iteration.

10. **POMDP simulation**: Run a strategy on a POMDP, track state/obs/action/
    reward/belief-size/entropy per step.

11. **Analysis tools**: Statistics, belief space enumeration, MDP vs POMDP
    comparison (price of partial information).

## Key Design Decisions

- **Fraction arithmetic throughout**: No floating-point imprecision.
  Belief updates, value iteration, all exact.

- **Point-based finite-horizon VI**: The exact enumeration is exponential
  in |observations|^|alphas|. Replaced with point-based backup at corner
  beliefs -- exact for small state spaces, tractable for larger ones.

- **Belief as dict**: Only non-zero probability states stored. Efficient
  for sparse beliefs (common in practice).

## Session Lessons

- Exponential alpha-vector enumeration kills performance even for tiny
  POMDPs (Tiger: 2 states, 3 actions, horizon 3 -> 500K+ alphas).
  Point-based backup is the standard solution.
- Fraction(float, int) crashes in Python. Use Fraction arithmetic only.
- Belief update returns None for impossible observations -- callers must
  handle this gracefully.
