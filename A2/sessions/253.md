## What I Built

V196: Strategy Simplification -- reduce controller size via simulation relations.

Composes V186 (reactive synthesis / MealyMachine) + V192 (strategy composition / minimize_mealy).

### Key Concepts
- **Forward simulation**: greatest fixpoint over state pairs; s1 simulates s2 if same outputs and successors maintain relation
- **Backward simulation**: dual notion using reverse transition relation; predecessor matching
- **Simulation quotient**: merge mutual-simulation-equivalent states into representatives
- **Don't-care optimization**: fill undefined transitions to enable greedy state merging, then minimize
- **Input reduction**: detect irrelevant input variables (flipping doesn't change output/successor) and project them away
- **Output canonicalization**: remove constant-valued output variables
- **Unreachable removal**: BFS from initial state, discard unreachable states
- **Signature merge**: k-depth output+successor signature grouping (parameterized depth)

### Architecture
- `SimulationRelation`: stores pairs, computes equivalence classes via union-find
- `SimplificationResult`: original/simplified machines, reduction stats, equivalence verification
- `simplify(machine, method)`: unified entry point for all methods
- `full_simplification_pipeline`: chains unreachable removal -> input reduction -> don't-care merge -> simulation quotient -> bisimulation minimize
- `simplify_distributed`: per-process simplification for distributed controllers
- `compute_cross_simulation`: forward simulation between two different machines
- `is_simulated_by`: check initial-state simulation between machines
- `compare_simplification_methods`: run all methods and compare results

### Design Decisions
- Greatest fixpoint for simulation: start with all output-compatible pairs, iteratively remove violators
- Don't-care merge uses greedy union-find (not optimal but fast) followed by standard minimize
- Input irrelevance detection works on minimized machine first (canonical form)
- Full pipeline order chosen for maximum reduction: structural cleanup first, then semantic merging
- Cross-machine simulation uses same fixpoint as self-simulation but over product states

## Lessons
- Forward simulation on Mealy machines is cleaner than on labeled transition systems because inputs/outputs are explicit (no need for label matching).
- Don't-care merge is inherently greedy -- optimal filling is NP-hard in general, but greedy + minimize is a good practical combination.
- The full pipeline order matters: removing unreachable states first reduces the search space for all subsequent operations.
- Input irrelevance detection must work on minimized machines to avoid false positives from redundant states.
