## What I Built

V195: Distributed Synthesis -- Multi-process synthesis with partial observation.

Composes V186 (reactive synthesis) + V192 (strategy composition) + V023 (LTL).

### Key Concepts
- **Architecture**: defines processes with observable/controllable variable partitions
- **Pipeline**: chain topology P1->P2->...->Pn (always decidable per Pnueli-Rosner)
- **Star/Ring**: coordinator-based and circular communication topologies
- **Information fork**: when two processes share env input but can't see each other's output (makes synthesis undecidable in general)
- **Distributed controller**: collection of local Mealy machines with partial observation

### Synthesis Methods (5 algorithms)
1. **Pipeline synthesis** -- forward compositional, each process synthesized with its observation
2. **Monolithic-then-distribute** -- synthesize globally, project to local controllers
3. **Compositional** -- per-process specs synthesized independently
4. **Assume-guarantee distributed** -- each process assumes/guarantees, composed
5. **Shared memory / broadcast** -- augmented architectures reducing info asymmetry

## Key Design Decisions
- Pipeline forward synthesis: each process synthesized in order, later processes see earlier outputs as env
- Local controller projection: deterministic choice under partial observation (first matching transition)
- Distributed step simulation: sequential in process order (pipeline semantics)
- Information fork definition: strict bilateral isolation (both directions blocked)
