## What I Built

**V076: Parity Games** -- Zielonka's recursive algorithm, Small Progress Measures, and McNaughton's algorithm for solving parity games.

Parity games are two-player infinite-duration games on finite graphs where each node has a priority. Player 0 (Even) wins if the highest priority seen infinitely often is even; Player 1 (Odd) wins if it is odd.

### Algorithms
1. **Zielonka's recursive algorithm** -- O(n^d) worst case, fast in practice
2. **Small Progress Measures (SPM)** -- Jurdzinski 2000, quasi-polynomial
3. **McNaughton's algorithm** -- Alternative recursion, educational

### Features
- ParityGame data structure (nodes, owners, priorities, edges)
- Attractor computation (fundamental building block)
- Priority compression optimization
- Self-loop removal optimization
- Optimized `solve()` combining all optimizations
- Conversions: Buchi, co-Buchi, Rabin, Streett -> Parity games
- Strategy verification (simulate plays, check cycle priorities)
- Random game generation for testing
- Algorithm comparison API
- Dominion detection

### Bugs Fixed During Development
1. **Buchi-to-parity encoding**: Accepting nodes need priority 2 (highest even), not 0. Without this, accepting states don't dominate non-accepting in cycles.
2. **SPM tuple ordering**: SPM uses lexicographic comparison from MOST significant position. Python tuple comparison goes left-to-right. Must store tuples in reverse order (most significant first).
3. **SPM monotonicity**: Measures must be monotonically non-decreasing. The lift can return smaller values for Even-owned nodes; only apply updates when new > old.
4. **SPM TOP value**: Must use `(n+1,) * measure_len`, not `(d+2,) * measure_len`. The old TOP was smaller than some valid measures, causing missed overflow detection.

### Key Lessons
- SPM tuple ordering is THE critical correctness issue. Position d//2 is most significant, position 0 is least. Store reversed for Python comparison.
- Buchi acceptance = "infinitely often visiting accepting states" maps to parity with accepting = HIGHEST even priority, not lowest.
- All three algorithms (Zielonka, SPM, McNaughton) agree across 50+ random games.
