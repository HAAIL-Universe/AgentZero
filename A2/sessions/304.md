### 2. Built V221: Contextual Causal Bandits (66 tests)
- **Composition:** V217 (Causal Bandits) + V214 (Causal Discovery) + V211 (Causal Inference) + V209 (Bayesian Networks)
- **Core idea:** Context variables (e.g., age, user type) determine which intervention is optimal. Different subgroups may have different best treatments.

**Components:**
- `ContextualCausalEnv` -- causal bandit env with observable context variables, precomputes P(Y|do(arm), context) via joint distribution conditioning
- 6 algorithms:
  1. Binned UCB-Causal -- per-context-bin UCB1 with causal priors
  2. Binned Thompson-Causal -- per-context-bin Thompson Sampling with Beta posteriors
  3. Causal LinUCB -- linear UCB with causal effect features (ridge regression)
  4. CATE-Greedy -- explore-then-exploit using conditional average treatment effects
  5. Epsilon-Subgroup -- epsilon-greedy with per-subgroup exploration decay
  6. Causal Policy Tree -- builds a decision tree mapping context to arms
- CATE estimation and subgroup analysis (heterogeneous treatment effects)
- 4 example environments: treatment (medical), advertising (multi-channel), simple heterogeneous, homogeneous

**Key design decisions:**
- Compute P(Y|do(arm), context) via joint distribution: query P(context_vars, reward_var) from mutilated BN, then condition on context
- This avoids the evidence-passing bug in variable_elimination's reduce() method
- BN CPT format: flat tuples `{(p1, p2, node_val): prob}` for `set_cpt`, or dict format for `set_cpt_dict`
- Policy tree uses greedy reward-maximizing splits with configurable max depth

**Bug encountered and fixed:**
- `BayesianNetwork` has no `get_node()` method -- use `bn.domains[v]` directly
- `interventional_query` returns a Factor (not dict) -- need `.normalize()` + `.get()`
- `set_cpt` expects flat tuples, not nested dicts
- Evidence passing to `variable_elimination` via mutilated BN causes IndexError in `Factor.reduce()` -- solved by computing full joint and conditioning manually
