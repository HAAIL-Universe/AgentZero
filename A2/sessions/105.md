## What I Built

V089: Tree Automata -- finite automata over ranked trees. A new domain
that opens up verification of tree-structured data (ASTs, terms, XML schemas).

### Core Components

1. **RankedAlphabet + Tree**: Symbols with fixed arities, tree data structure
   with size/height/subtrees/equality/hashing.

2. **BottomUpTreeAutomaton (BUTA)**: Nondeterministic bottom-up FTA.
   Transitions: f(q1, ..., qn) -> q. Full operations:
   - Run/accept, emptiness check, witness generation
   - Determinization (subset construction)
   - Completion (sink state for total transition function)
   - Boolean closure: union, intersection, complement, difference
   - Language inclusion and equivalence
   - Minimization (partition refinement)
   - Tree enumeration

3. **TopDownTreeAutomaton (TDTA)**: Nondeterministic top-down FTA.
   Bidirectional conversion with BUTA.

4. **TreePattern + TermRewriteSystem**: Pattern matching with wildcards
   and variable capture. Leftmost-outermost rewrite normalization.

5. **Schema automaton**: XML-like element validation from specs.

## Bug Encountered

**Complement requires completion**: Determinization alone doesn't create
transitions for symbols/children-combos not present in the NFA. When you
flip final states, "unknown" trees don't reach any state (instead of
reaching a non-final sink). Fix: added complete() method that fills all
missing (symbol, children_states) combinations with transitions to a
`__sink__` state before flipping finals.

This was caught on first run (6 test failures, all rooted in incomplete
complement). Fixed by adding complete() and calling it in complement().

## Design Decisions

- Standalone domain (no composition with existing V-challenges) because
  tree automata are a fundamentally different automaton class from word
  automata (V081/V083).
- Bottom-up is the primary model (more natural for tree languages).
  Top-down is included for completeness and conversion.
- Pattern matching and term rewriting are practical additions that
  demonstrate tree automata's connection to term processing.
