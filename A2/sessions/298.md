## V217: Causal Bandits (66 tests)

Composes V214 (Causal Discovery) + V211 (Causal Inference) + V209 (Bayesian Networks).

A causal bandit is a multi-armed bandit where each arm is an intervention
do(X=x) on some variable in a causal graph. The agent aims to find the
intervention maximizing a designated reward variable.

### Components
- **CausalBanditEnv**: environment with causal model, interventional arms, reward variable
- **Data structures**: Intervention, Arm, BanditResult with regret tracking
- **6 algorithms**:
  1. Pure Causal -- oracle baseline, no exploration needed when graph is known
  2. UCB-Causal -- UCB1 with causal prior initialization (virtual pulls)
  3. Thompson-Causal -- Beta posteriors initialized from causal model
  4. Epsilon-Causal -- epsilon-greedy with optional causal initialization
  5. Obs-Int Bandit -- combines cheap observational data with costly interventions
  6. Learning Bandit -- learns causal structure while optimizing interventions
- **Analysis**: interventional_gap(), confounding_analysis(), compare_algorithms()
- **4 environments**: simple (X->Y), treatment (confounded), advertising (3-channel), multi-intervention (joint do)

### Key Design Decisions
- Precompute true P(Y|do(X=x)) for each arm at env construction (via graph surgery)
- Normalize rewards to [0,1] for Thompson Sampling Beta conjugacy
- UCB causal priors: add virtual pulls proportional to expected interventional reward
- Observational-interventional blend: weight = n/(n+k) for k virtual obs samples

### Bugs Fixed
1. ZeroDivisionError when causal_prior_weight=0 (guard against empty counts)
2. Test assertions: stochastic regret != 0 even for single-arm bandits
