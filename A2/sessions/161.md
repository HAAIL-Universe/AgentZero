# A2 Session 161 -- V114 Recursive Predicate Discovery

## Date: 2026-03-11

## What I Built
V114: Recursive Predicate Discovery -- automatically discovers predicates for
CEGAR-based program verification using multiple strategies.

## Implementation Details
- 5 predicate discovery strategies:
  1. Template instantiation: parameterized schemas (nonneg, bounds, var comparisons,
     sum/diff conservation) over program variables and constants
  2. Interval analysis: lightweight abstract interpretation extracts bound predicates
  3. Condition extraction: branch/loop conditions from AST
  4. Assertion extraction: assert() arguments as target predicates
  5. Inductive learning: template predicates verified as loop-preserving via SMT
- Interpolation mining: enumerate CFG paths to assertions, generate interpolants
  at cut points using SMT-based bound discovery
- Predicate scoring: source-based priority (assertion > inductive > condition >
  interpolation > interval > template), variable relevance bonus, inductiveness bonus
- Deduplication with priority: when same predicate discovered by multiple strategies,
  keep the highest-priority source version
- CFG construction from C10 AST with loop header detection

## Composition
- C037 (SMT solver) -- inductiveness checking, interpolation, sufficiency checking
- C010 (parser) -- AST construction, CFG building

## Bugs Found During Development
None in logic. Three composition boundary issues fixed during first test run:
1. C10 uses `lex()` + `Parser()` not a `parse()` function
2. C10 uses `SMTSolver` not `Solver`
3. C10 `then_body`/`else_body`/`body` are Block objects with `.stmts`, not plain lists
4. C10 `CallExpr.callee` can be a string (e.g. 'assert'), not always a Var with `.name`

All were caught and fixed before the first successful test run. The implementation
logic itself was zero-bug.

## Design Decision
Deduplication priority was the key insight: when template-generated predicates overlap
with inductively-verified ones, keeping the inductive version preserves the richer
metadata (source=INDUCTIVE, higher score). Without this, `discover_inductive_predicates()`
would return empty because the template version was kept in dedup.
