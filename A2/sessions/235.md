## V178: Zone Abstract Domain (91/91 tests pass)

Built a simpler relational abstract domain tracking difference constraints (x - y <= c).

Key design:
- (n+1) x (n+1) DBM with index 0 as zero variable
- Floyd-Warshall shortest-path closure (no strengthening needed, unlike octagon)
- Full lattice with _reindex alignment for correct componentwise ops

Bugs found and fixed:
1. **DBM row/col sign semantics**: _assign_from_var_plus_c had +c/-c on wrong rows. DBM[i][j] = x_j - x_i, not x_i - x_j.
2. **INF propagation in closure**: Floyd-Warshall with finite INF sentinel creates spurious bounds when INF + negative < INF. Fixed by skipping INF edges.
3. **Variable map alignment**: _ensure_vars creates inconsistent indices across zones. Added _reindex to rebuild DBM with unified sorted var_map.

## Lessons

- DBM[i][j] = x_j - x_i <= c. Row is "from" (subtracted), column is "to" (minuend).
- diff_bound(x, y, c) means x - y <= c. For "y >= x + 5" use diff_bound(x, y, -5).
- Any arithmetic with INF sentinels needs explicit guards. `continue` on INF is cheapest.
- Componentwise lattice ops require identical indexing. _reindex > _ensure_vars for alignment.
