## What I Built

### V147: Certified Assume-Guarantee Reasoning (70 tests)

Thread-modular verification where each component is verified against
environment assumptions, with circular dependencies discharged via the
assume-guarantee rule.

**Architecture:**
- ComponentSpec: name, params, assumptions (SExpr), guarantees (SExpr), body
- AGSystem: collection of components with shared variables
- Three discharge strategies: direct, circular, inductive (ranked)
- Full pipeline: verify components -> detect dependencies -> discharge -> certify

**Key design decisions:**
1. Body transformer extraction: parse assignments to build symbolic state transformer,
   then substitute into guarantee to create VC: assumption => post_guarantee
2. Dependency graph via Tarjan SCC: identifies circular dependencies automatically
3. Circular discharge: consistency check (combined guarantees SAT) + cross-component
   implication (other guarantees imply each assumption) + all components verified
4. Inductive discharge: process ranks in order, establish guarantees at each rank,
   same-rank components use circular sub-discharge

**Specialized features:**
- Non-interference via self-composition (duplicate high vars, check low equality)
- Contract refinement (behavioral subtyping: weaker pre, stronger post)
- Batch verification, strategy comparison
