## What I Built

V087: Abstract Interpretation over Strings -- a framework of abstract domains
for analyzing string-valued programs, composing V081 (Symbolic Automata) and
V086 (String Constraints).

### Five Abstract Domains

1. **LengthDomain** -- interval [lo, hi] over string length with join/meet/widen/concat/slice
2. **PrefixDomain** -- known constant prefix, LCP-based join, prefix-subset ordering
3. **SuffixDomain** -- known constant suffix, LCS-based join, suffix-subset ordering
4. **CharSetDomain** -- per-position character sets + overall alphabet constraint
5. **SFADomain** -- full symbolic automaton tracking via V081 SFA (most precise, optional)

### StringProduct (Reduced Product)

All five domains combined with cross-domain reduction:
- Prefix/suffix length raises length lower bound
- CharSet position count sets exact length
- Length upper bound detects inconsistent prefix/suffix
- Prefix chars intersect charset positions
- Prefix+suffix overlap consistency check at exact lengths

### StringInterpreter

Abstract interpreter for simple string programs:
- SAssign, SConcat, SSlice, SIf, SWhile, SAssert
- 10 condition types: SLenEq, SLenLt, SLenGt, SStartsWith, SEndsWith,
  SEquals, SNotEquals, SContains, SIsEmpty, SNot
- Path-sensitive (if/else refines env per branch)
- Fixed-point with widening for while loops
- Assertion checking: TRUE (definite), FALSE (impossible), UNKNOWN

### APIs

- analyze_string_program() -- main entry point
- get_variable_info() -- detailed info extraction
- compare_domains() -- with/without SFA comparison
- string_domain_from_constraints() -- bridge to V086
- analyze_string_flow() -- data flow graph style
- check_string_property() -- property checking

## Bugs Found and Fixed

1. **CharSet concat with TOP**: Concatenating [34 positions] + [] gave [34 positions],
   falsely constraining length to exactly 34 via reduction. Fix: only concat
   positions when BOTH sides have non-empty position lists.

2. **LengthDomain.slice precision**: When source length is exact, slice result
   should also be exact. Old code was conservative [0, max] for all cases.

## Design Decisions

- SFA domain is optional (expensive) -- product works well without it for most analyses
- CharSetDomain uses empty list `[]` for TOP (no position info), `None` for BOT
- Widening for SFA: go to TOP if state count exceeds 100 (practical threshold)
- Prefix concat extends prefix when self is exact (known full string)
- Suffix concat takes suffix of second operand (sound for all cases)
