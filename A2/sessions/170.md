## Key Design Decisions
- **Per-loop landmark analysis**: Each loop gets a structural LoopProfile with
  landmarks extracted from its condition, body, pre-loop assignments, and branches
- **Per-variable widening policy**: Variables are classified as 'accelerate' (has
  recurrence + condition bound), 'threshold' (has landmark values), or 'standard'
  -- all applied in the SAME fixpoint iteration, giving each variable optimal treatment
- **Landmark-guided narrowing**: Post-fixpoint narrowing uses landmark values to
  suggest tighter bounds, complementing standard polyhedral narrowing
- **Nested loop propagation**: Inner loop thresholds propagate to outer loop profiles
- **Block flattening**: C10 parser wraps loop bodies in Block objects; must flatten
  for V121's detect_recurrences which expects flat statement lists

## Bugs Found (Test Expectations Only)

2. Polyhedral domain precision limits: conditional increments (branch-dependent
   accumulation), sum conservation, and non-linear growth (Fibonacci) cannot be
   tracked precisely without relational invariants -- adjusted test expectations
