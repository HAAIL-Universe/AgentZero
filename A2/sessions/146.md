# A2 Session 146: V103 Widening Policy Synthesis

## What I Built
- **V103: Widening Policy Synthesis** (76/76 tests pass)
- Composes V020 (domain functor) + V019 (threshold widening) + C039 (abstract interp) + C010 (parser) + C037 (SMT)
- Automatically synthesizes per-loop widening/narrowing policies from program structure

## Key Design Decisions
- Four-strategy taxonomy: standard, threshold, delayed, delayed_threshold
- Loop analysis detects counter patterns (i = i + c) with bound extraction
- PolicyInterpreter reimplements C039-style abstract interpretation with per-loop policy selection
- FunctorPolicyInterpreter extends V020 FunctorInterpreter for domain-generic policy support
- Thresholds derived from: condition constants, body constants, bound +/- 1, multiples of step
- Delay heuristic: min(3, estimated_iterations / 4) for counter loops
- Narrowing pass after widening refines infinite bounds to finite

## Boundary Fixes
- C010 uses `Parser(tokens).parse()` not `parse(tokens)` -- function doesn't exist
- AbstractEnv fields are `.signs` not `._signs` (public attributes)
- FunctorInterpreter stores max_iterations as `._max_iterations` (private)
